## 1. 文档概述

### 1.1 背景
"行销方案管理" 是门店灵活获取客源、提升淡季出租率的核心工具。区别于简单的优惠券，它是一套复杂的 **规则引擎 ，负责定义 "什么人、在什么时间、租什么车、能享受什么价格"。

### 1.2 目标
*   **统一管理**: 集中配置所有针对 "门市租车" 业务的促销活动。
*   **场景覆盖**: 支持早鸟优惠、连租特惠、周末促销、淡旺季调价等常见场景。

---

## 2. 核心功能模块

### 2.1 方案管理
一个 "方案" 是一个大的营销主题盒子，包含多条具体规则。
*   **基础属性**:
    *   **方案名称**: 内部管理用 (e.g., "2026 Q3旺季大促")。
    *   **展示名称**: App/官网前端透出文案 (e.g., "暑期特惠")。
    *   **优先级 (Level)**: 决定方案互斥时的生效顺序 (数值越大优先级越高)。
    *   **有效期**: 方案整体的生效时间段。
*   **适用范围**:
    *   **业务线**: 根据用户选择标准门市、24h自助。
    *   **客户群体**: 限制国籍、会员等级。
    *   **支付卡种**: 限制特定信用卡组织 (Visa/Mastercard) 享受特定优惠。

#### 2.1.1 到店人工核验
行销方案仍先按基础适用条件进行配置和匹配，包括会员方案、企业资格、适用门店、适用车组、有效期、下单时间、用车时间、支付卡种等。到店人工核验只是行销方案的一个可选补充项，用于处理系统无法在线判断、必须由门店现场查看证明文件的优惠资格。

| 配置项 | 规则 |
| :--- | :--- |
| 是否需要到店人工核验 | 可选项。关闭时，行销方案只按基础适用条件匹配；开启时，客户取车时必须由门店核验证明文件。 |
| 资格名称 | 开启到店人工核验时必填，例如学生身份、教师身份、医护身份。 |
| 证明文件说明 | 开启到店人工核验时填写文本说明，用于提示门店现场大致需要查看哪些证件或证明，例如学生证、教师证、在职证明。 |
| 取车证明附件 | 开启到店人工核验时，门店取车核验通过至少上传 1 个证明文件附件，最多可上传 3 个证明文件附件，作为订单核验凭证。行销方案不维护固定附件名称。 |
| 适用业务模式 | 开启到店人工核验的行销方案本期仅适用于有人服务的门市租车；24 小时自助租车没有店员现场核验能力，本期不开放选择该类方案。 |
| 订单快照 | 订单创建时保存行销方案版本号、是否需要到店人工核验、资格名称和证明文件说明。后续方案配置调整不反向影响已创建订单。 |

客户预约时可以选择需要到店人工核验的行销方案。系统在下单页面和订单详情中展示“取车时需现场核验证件”的提示；取车时由门店员工完成证件上传和核验结果记录。核验未通过时，原行销方案不得继续生效，必须在取车流程内更换为客户当场符合资格的行销方案或不使用优惠。

### 2.2 规则引擎
一个方案下可挂载多条规则，规则由 "**条件**" 和 "**动作**" 组成。

#### A. 触发条件
1.  **时间限制**:
    *   **下单时间**: 限制用户必须在某段时间内下单。
    *   **用车时间**: 限制订单的取还车时间必须落在特定区间。
    *   **提前预订**: 必须提前 N 天下单 (早鸟逻辑)。
2.  **车型限制**:
    *   全系车型 或 指定某几个 "车型组" (e.g., 仅限 SUV组)。
3.  **租期限制**:
    *   最少租 N 天，最多租 M 天 (连租逻辑)。
4.  **地理限制**:
    *   限制特定的 "取车门店" 或 "还车门店"。

#### B. 优惠动作
1.  **折扣 (Discount)**: 打 N 折 (e.g., 85折)。
2.  **直减 (Minus)**: 立减 N 元 (e.g., 减 200元)。
3.  **一口价 (Fixed)**: 无论原价多少，强制按 N 元/天 计算。
4.  **定价模式**:
    *   **统一价**: 全周执行同一优惠力度。
    *   **平假分开**: 在车组标准价基础上，周一至周四平日执行折扣价 A，周五至周日假日执行折扣价 B (支持法定节假日单独配置)。

### 2.3 营销日历
*   **可视化视图**: 以日历形式展示当前正在运行的所有营销方案。
*   **颜色编码**: 不同类型的方案使用不同颜色色块标记，方便运营人员一眼识别 "促销撞车" 或 "优惠空窗期"。

## 3. 业务规则
*   **互斥与叠加**:
    *   默认原则: **不叠加**。一个订单只能命中一个最优方案。
    *   优先级判断: 系统优先匹配权重 (Level) 最高的方案；若权重相同，匹配优惠力度最大的。（用户可再手动更改选择），此功能需可在代码配置开关，关闭后没有默认方案给用户，需用户手动选择或默认第一个顺序方案
*   **价格计算顺序**: 个人订单为标准原价 -> *行销方案* -> 积分点数 -> *优惠券* -> 最终价；企业身份订单为标准原价 -> 企业资格可用*行销方案* -> 最终价，不支持优惠券和积分点数。
    *   *注*: 门市租车行销方案作用于车组标准价。标准价不区分平日/假日，行销方案可通过平假分开模式配置不同折扣价。
*   **订单适配**:
    *   新建订单页的“行销优惠方案”读取门市租车 Tab 下的有效方案。
    *   个人预订和企业资格预订分别匹配对应可用方案。非月结企业预订使用企业资格可用方案后，资金流程仍按客户自付订单处理；企业身份订单不支持优惠券和积分点数；企业月结订单不发起客户支付或预授权，最终应收金额进入企业月结账单。
    *   跨平假日订单按每一轮 24 小时的起始时间判断该轮适用平日折扣或假日折扣；超过完整 24 小时后的剩余时长视为新一轮 24 小时的开始。
    *   订单创建成功时保存行销优惠方案基线快照，包含方案 ID、方案版本号、方案名称、适用客户资格、平日折扣、假日折扣和抵扣金额。订单创建后，客户企业关系、方案上下架、折扣调整或资格变化不反向改写已创建订单的基线快照；若订单后续发生取车现场改行销方案、续租重算或其他商业条件变化，则按事件发生时命中的方案版本生成事件级增量快照，仅重算该事件影响的差额。

### 3.1 行销方案版本记录
行销方案新增、编辑、启用、停用、适用资格调整、折扣规则调整、有效期调整和优先级调整时，系统生成新的方案版本。历史版本只读保留，订单已引用的版本不允许覆盖修改。

| 版本字段 | 规则 |
| :--- | :--- |
| 方案版本号 | 系统生成唯一 `marketingPlanVersionId`，用于订单计费快照追溯。 |
| 方案基础信息 | 记录方案 ID、方案名称、展示名称、业务线、适用范围、有效期和优先级。 |
| 适用资格 | 记录个人会员、企业资格、门店、车组、租期、下单时间和用车时间等命中条件。 |
| 优惠动作 | 记录折扣、直减、一口价、平日折扣、假日折扣、法定假日配置和抵扣上限。 |
| 操作信息 | 记录保存人、保存时间、变更说明和版本状态。 |

新建订单命中行销方案时，系统将 `marketingPlanVersionId` 和该版本的关键优惠字段复制到订单级基线快照。订单创建后，后台营运查看订单时按快照版本展示当时的方案内容，不按当前行销方案状态重新解释历史订单；如订单后续因取车现场改方案、续租重算或其他商业条件变化触发方案重算，则按事件发生时命中的 `marketingPlanVersionId` 生成事件级增量快照，并与原订单基线方案快照并存保留。

行销方案列表和方案配置详情页需要提供历史版本入口。点击方案版本号或“历史版本”时，打开版本详情弹窗，版本详情沿用方案详情页和规则详情页的排版结构，以只读方式展示该版本内的完整设定值，包括方案基础信息、适用客户资格、适用业务线、适用车组、有效期、优先级、优惠动作、平日/假日折扣、规则明细、保存人、保存时间和变更说明。历史版本只读展示；订单详情页点击 `marketingPlanVersionId` 时，展示订单下单时命中的方案版本详情。

## 4. UI/UX 细节
*   **向导式创建**: 采用 Step-by-Step 弹窗向导 (场景选择 -> 基础信息 -> 规则配置)，降低配置复杂度。
*   **实时预览**: 在配置规则折扣时，右侧实时显示 "模拟计算结果" (e.g., 配置8折，原价500 -> 预览价400)，防止配置失误导致亏损。
