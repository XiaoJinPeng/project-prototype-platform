## 1. 文档范围

本文件定义企业端公务车后台功能的共用业务口径、角色权限、数据范围和跨页面规则。各页面的具体字段、筛选、按钮和交互规则写入对应页面 PRD。

适用客户端：企业端后台。

适用业务线：公务车。

适用角色：企业总管理员、分公司管理员。

## 1.1 文档版本维护规则

1. 公务车 PRD 文档必须保留“版本记录”区块，放在文档标题后、正文前。
2. 每次调整 PRD 内容时，必须在版本记录中新增一行，记录版本号、日期和调整概括。
3. 版本号按 `v主版本.次版本` 记录；小范围字段、文案、交互补充按次版本递增，重大业务口径调整按主版本递增。
4. 调整概括需要写清本次影响的功能范围、业务规则或页面行为，不写泛泛的“优化”。
5. 如果调整涉及状态流、权限、数据范围、账单、车辆套用、提醒规则等关键业务口径，还需要同步更新业务决策台账。

## 2. 核心业务口径

1. 公务车用于企业内部员工预约用车，不面向个人会员租赁。
2. 公务车订单不发起个人支付、个人预授权、个人退款。
3. 公务车车辆的资产归属由“签约企业 / 分公司”决定，车辆所在据点表示车辆实际停放位置。
4. 车辆资产归属分公司负责管理车辆资料、绑定合约配额、处理清洁保养提醒。
5. 员工所属分公司决定订单归属、审批归属、统计归属。
6. 其他分公司员工只要具备对应据点用车权限，可以预约停放在该据点的可用车辆；但不能管理不属于本分公司的车辆。
7. 车辆调拨到其他分公司后，原有未开始和进行中的预约继续按原订单执行；调拨后的新预约按新资产归属和新合约配额处理。
8. 车辆合约到期或配额失效后，不开放新预约；已预约订单保留，由企业内部自行处理。

## 3. 角色与权限

| 角色 | 可见范围 | 可执行操作 | 数据隔离 |
| --- | --- | --- | --- |
| 企业总管理员 | 企业全量分公司、部门、员工、车辆、订单、合约、账单和统计数据 | 建立分公司、配置分公司可用据点、管理车辆、调拨车辆资产归属、分配合约配额、查看全量账单和统计 | 可跨分公司查看和管理企业内数据 |
| 分公司管理员 | 本分公司员工、订单、部门、资产归属本分公司的车辆、分公司可用据点内车辆的可用性数据 | 管理本分公司员工、部门、订单审核、车辆基础资料、车辆提醒、用车范围 | 不可管理其他分公司资产车辆，不可把车辆调出本分公司 |

## 4. 数据归属规则

### 4.1 车辆归属

1. 车辆资产归属以车辆的签约企业 / 分公司为准。
2. 只有资产归属分公司及企业总管理员可以管理该车辆。
3. 车辆所在据点只代表停放位置，不改变资产归属。
4. 车辆可被其他分公司员工预约时，只改变订单使用方，不改变车辆资产归属。

### 4.2 订单归属

1. 公务车订单按用车员工所属分公司归属。
2. 员工跨据点使用其他分公司资产车辆时，订单仍归属于员工所属分公司。
3. 订单历史展示读取订单创建时保存的员工、车辆、据点和审批快照。

### 4.3 统计归属

1. 员工、部门、订单、用车时长统计按员工所属分公司统计。
2. 车辆使用率、车辆排名按页面查看范围统计。
3. 车辆资产维度统计按资产归属分公司统计。

## 6. 共用页面规则

1. 企业端页面不展示“会员”“会员方案”“个人支付”“预授权”等个人租赁概念。
2. 与公务车员工相关的页面统一使用“员工”“用车员工”“公务订单”。
3. 总公司 / 分公司角色视图切换仅保留在需要演示数据范围差异的页面；已确认去除的页面不显示该组件。
4. 页面内容区背景以企业端现有页面风格为准，不使用异常的 `#F3F4F6` 或 `#F2F4F5` 背景块。
5. 历史订单、账单、提醒和调拨记录按快照展示，不随当前主数据变化反向覆盖历史。

## 7. 本期不做

1. 不做车机开关门、车机里程回传和车机故障闭环。
2. 不做个人支付、个人退款和个人预授权。
3. 不做自动派发维修、清洁、保养工单。
4. 不做跨企业车辆共享。
5. 不做通用可配置审批流，仅支持公务车预约审批开关。
