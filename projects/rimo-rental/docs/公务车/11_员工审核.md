## 1. 页面范围

原型页面：`02_原型和PRD/原型/企业端/employee_review.html`

页面用途：审核员工加入企业的申请。

## 2. 数据范围

1. 企业总管理员可查看企业全部员工加入申请。
2. 分公司管理员可查看本分公司员工加入申请。
3. 审核通过后，员工成为企业员工，可进入公务车用车流程。

## 3. 列表字段

| 字段 | 说明 |
| --- | --- |
| 申请人 | 姓名、身份证号 |
| 联系方式 | 手机号、Email |
| 申请组织 | 申请加入的分公司、部门 |
| 申请时间 | 员工提交申请时间 |
| 审核状态 | 待审核、已通过、已驳回 |
| 操作 | 审核、详情 |

## 4. 审核规则

1. 审核通过后，员工加入对应分公司和部门。
2. 审核通过后，员工可按分公司可用据点、车辆状态、用车范围、排程冲突进行公务车预约。
3. 审核驳回后，员工不成为企业员工，不能使用该企业公务车。
4. 审核驳回必须记录驳回原因。

## 5. 批量审核

1. 支持批量通过。
2. 支持批量驳回。
3. 批量驳回时必须填写统一驳回原因。
4. 批量操作只对当前用户有权限的数据生效。

## 6. 验收标准

1. 待审核员工通过后出现在员工管理列表。
2. 被驳回员工不能进入公务车预约流程。
3. 分公司管理员只能审核本分公司申请。
4. 审核操作写入操作记录。
