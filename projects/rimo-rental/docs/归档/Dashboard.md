# 

## 1. 产品概述
### 1.1 项目背景
RIMO 租车系统的后台包含订单、会员、车辆资产、调度、财务等多条业务线。目前营运人员缺乏一个集中的全局视图来监控各项指标、资源的流转状态以及紧急的业务异常。为提升调度效率和运营监管能力，需重构首屏（`index.html`）为综合业务大盘。

### 1.2 目标定位
提供租车分时/门市业务的核心运营数据概览，实时暴露系统运转中的业务卡点与异常工单，辅助主理人或调度员进行每日的宏观决策和即时干预。

### 1.3 用户角色
- 平台主理人（关注营收、整体出租率）
- 城市/区域站长（关注可用车辆数、异常工单与维修保养池）

## 2. 核心功能需求

### 2.1 顶部核心指标 (Stats Board)
需直观呈现系统四大底层数据的今日/总累计状态，支持高频（如 5~10 分钟）刷新。

| 指标维度 | 显示数据 | 释义与计算逻辑 |
| --- | --- | --- |
| **会员与客户** | 会员总规模 | `Count()` 所有个人会员与企业客户总数。<br/> 附加显示本周新增数和企业客户下钻数量。 |
| **运力与状态** | 车辆设备总数 | `Count()` 名下可控的车辆档案。<br/>必须拆解当前三态：**使用中**（生成进行中订单）、**待租中**（站点可用）、**维保中**（封控/整备中）。|
| **今日订单** | 今日新增订单 | 当日支付或预约成功的新订单数。<br/>须分业务线拆解：**门市租赁**日增量与**分时租赁**日增量。 |
| **今日营收** | 今日预估营收 | 当前结算账单金额 + 今日进行中订单的预估租金。<br/>附加显示较昨日同时段百分比环比（YoY/MoM）。|

### 2.2 门市与车辆分布区域 (Space Monitor)
通过可视化的地图（ECharts Map/Scatter）形式展示车辆驻网情况。

- **功能点**：
  - 显示 RIMO 各服务站点的可用车辆密度（热力散点越大表示滞留车辆/可用池越大）。
  - 支持 **热力模式** 和 **站点分布模式** 切换。
  - Hover 提示 tooltip 须标明网点名称与可用库存。
- *交互说明*：在实际接入中，此区块可支持点击下钻到具体的站群或城市地图。

### 2.3 异常工单与预警中心 (Exception Center)
Dashboard 的**核心行动触发区**。必须显式并分类列出正在发生的业务异常情况，方便运维人员直接跳转干预。

#### 2.3.1 订单追踪异常
- **触发条件**：
  - 逾期未还：当前时间 > 订单预计还车时间 + 容忍时长（如 30 min）。并红字标记超时时效。
  - 未准时取车：当前时间 > 客户预订取车时间 + 容忍时长（如 15 min）。
  - 事故工单：司机上报或系统标记进入事故定损流程的异常单。
- **展示要素**：异常标签、订单编号、超期时长、简要描述、客户姓名与加密电话。

#### 2.3.2 资产运维异常
- **触发条件**：
  - 需保养：车辆 ODO (总里程) 达到系统设置的保养阈值。
  - 待清洁：车辆刚结束长程租用或异地还车，传感器上报或系统按规则压入待整备池。
  - 低电量/低油量：针对分时租赁业务，如果车辆归还后 SOC < 15% 或剩余油量低于报警线，需发出寻车补能任务单。
- **展示要素**：工单类型、车牌号、当前停放站点、简要告警理由。

### 2.4 底部商业智能分析区 (BI Charts)
通过 ECharts 渲染动态数据的走势图，分析最近 7 天的核心业务动向。

- **车队出租率趋势图 (Fleet Utilization)**
  - 图表类型：平滑折线图 (带渐变 Area)。
  - 定义：出租率 = 当日有租赁计费时长的车辆 / 运营池可用车辆总数。
  - 目标：展示周一至周日的出租率波峰波谷，帮助站长判断闲时和节假日的运力短缺情况。
- **营收与订单结构图 (Revenue & Orders)**
  - 图表类型：堆叠柱状图 (Stacked Bar Chart)。
  - 定义：展示每日核心租金总流水与增值服务（如基础保险费、超时滞纳金、异地还车费等）层叠展示总流水。

## 3. 非功能性与 UI/UX 规范
1. **自动刷新机制**：异常预警中心需要实现局部轮询（Polling）或 Server-Sent Events (SSE) 接入，无需刷新页面即可感知新告警。
2. **渐进增强的视觉 (Progressive UI)**：
   - 使用卡片式设计 (Tailwind 驱动)，Stats 卡片需包含背景微动效及清晰的 Icon，采用强对比字重区分数据。
   - 告警中心的颜色符合心理学：红（订单逾期/事故等高危）、黄（低电量/待保养的中危）。
3. **响应式 (Responsive Design)**：
   - 大屏下，地图为主辅以右侧预警；BI 图表平分底部。
   - 窄屏（如 iPad 竖屏）下，统计卡片调整为一排 2 个，右侧异常区流动到下方或做弹层隐入。
