## 1. 文档概述

### 1.1 背景
门市租车订单列表是门店日常运营的指挥中心，承载订单查询、状态监控、取还车处理、费用处理和异常跟进等高频动作。列表需要在一屏内同时呈现订单主状态、派车状态、支付状态、追缴状态和风险标记，帮助门店员工快速判断下一步处理动作。

### 1.2 目标
* **全景视图**：统一管理门市租车订单，并在有业务线授权时支持查看 24h 自助订单。
* **高效流转**：为已预订、逾期未取、用车中、续租中、逾期未还、待补缴、待退款、追缴待通知等订单提供快捷入口。
* **风险识别**：在列表中直接暴露逾期、事故、黑名单、证件异常、追缴等风险信息。
* **状态一致**：订单列表使用《门市租车订单状态》中定义的主状态和辅助标签口径。

### 1.3 页面入口
* 后台侧边栏：订单中心 -> 门市租赁订单
* 对应原型文件：`store_orders.html`

## 2. 页面结构

页面由以下区域组成：

1. **页面标题区**
   * 标题：门市租赁订单
   * 副标题：管理所有门市租赁订单，支持快速取还车、订单查询及状态跟踪。
   * 主按钮：新建门市订单
   * 分裂按钮：支持按当前业务线进入新建订单。

2. **运营统计区**
   * 展示全部订单、待处理、履约中、结算处理、异常风险、企业月结、完成后追缴、待收/待退等快捷统计。
   * 点击统计卡片后，列表按对应业务口径筛选。

3. **筛选区**
   * 承载关键词、客户类型、订单状态、日期范围、取还门店、车型/车组等筛选项。

4. **业务线 Tab**
   * 门市租赁订单
   * 24h 自助订单
   * 当前账号无对应业务线授权时，不展示该业务线订单。

5. **订单表格**
   * 展示订单信息、客户信息、车辆信息、行程时间、取还门店、金额、主状态和操作按钮。

6. **分页区**
   * 支持页码切换和每页条数切换。

## 3. 筛选与搜索

### 3.1 关键词搜索

关键词输入框支持模糊搜索以下字段：

| 搜索字段 | 说明 |
| :--- | :--- |
| 订单号 | 支持完整订单号和部分订单号 |
| 客户姓名 | 支持个人客户姓名 |
| 手机号 | 支持客户手机号 |
| 身份证号 | 支持证件号搜索 |
| 企业名称 | 支持企业客户名称和简称 |
| 车牌号 | 支持已派车订单的车牌号 |

### 3.2 客户类型筛选

| 选项 | 说明 |
| :--- | :--- |
| 全部客户 | 不限制客户类型 |
| 个人客户 | 仅展示个人客户订单 |
| 企业客户 | 仅展示企业客户订单 |

### 3.3 订单状态筛选

订单状态筛选支持多选。筛选器按业务场景分组展示，可选择分组，也可选择具体状态。选择分组时，系统按分组内全部状态查询。

| 分组 | 包含状态 |
| :--- | :--- |
| 待处理 | 待支付、已预订、逾期未取 |
| 进行中 | 验车中、用车中、续租中 |
| 异常/风险 | 逾期未还、出险处理中 |
| 结算处理 | 待结算、待补缴、待退款 |
| 已结束 | 已完成、已取消、爽约、已拒绝、已关闭 |

### 3.4 日期范围筛选

日期范围筛选支持以下时间口径：

| 时间口径 | 说明 |
| :--- | :--- |
| 取车时间 | 按预计取车时间筛选 |
| 下单时间 | 按订单创建时间筛选 |

日期范围控件需要提供时间口径切换，支持按取车时间或下单时间查询。

### 3.5 门店筛选

门店筛选支持分别筛选：

| 筛选项 | 说明 |
| :--- | :--- |
| 取车门店 | 按订单预计取车门店筛选 |
| 还车门店 | 按订单当前还车门店筛选 |

取车门店与订单当前还车门店不一致时，列表中使用异地还车样式高亮。异地还车样式和异地还车费均按门店判断，不按车辆所在据点或归还据点判断。改还车门市/转单后，列表展示新的订单当前还车门店。

### 3.6 车型筛选

车型筛选按车组和车型两级展示：

| 层级 | 说明 |
| :--- | :--- |
| 车组 | 如经济型、SUV组、商务车组 |
| 车型 | 如 Toyota RAV4、Honda Odyssey |

筛选器支持选择车组或具体车型。选择车组时展示该车组下全部订单；选择车型时仅展示该车型订单。

### 3.7 时间排序

行程时间列支持以下排序：

| 排序项 | 说明 |
| :--- | :--- |
| 取车时间正序 | 按预计取车时间从早到晚排序 |
| 取车时间倒序 | 按预计取车时间从晚到早排序 |
| 还车时间正序 | 按预计还车时间从早到晚排序 |
| 还车时间倒序 | 按预计还车时间从晚到早排序 |
| 默认排序 | 按创建时间倒序展示 |

### 3.8 结算方式、追缴和风险筛选

| 筛选项 | 选项 | 说明 |
| :--- | :--- | :--- |
| 结算方式 | 全部结算、客户自付、企业月结 | 客户自付包含个人订单和非月结企业预订；企业月结仅展示 `billingMode=monthly` 或 `paymentStatus=monthly_billing` 的订单。 |
| 追缴状态 | 全部追缴、存在追缴、追缴待通知、追缴已通知、追缴待支付、企业月结追缴、追缴已结清 | 追缴筛选只作用于 `postCharge` 子状态，不改变订单主状态筛选口径。 |
| 风险标记 | 黑名单、证件异常、需联系客户、事故上报、违章待处理、自助异常 | 风险筛选作用于客户风险标记和状态风险标记。 |

### 3.9 运营统计快捷筛选

运营统计卡片展示当前业务线下的订单数量，点击后作为独立快捷筛选条件，不与普通筛选项互相覆盖。

| 统计项 | 统计口径 |
| :--- | :--- |
| 全部订单 | 当前业务线全部订单。 |
| 待处理 | `pending_payment`、`reserved`、`pickup_overdue`。 |
| 履约中 | `inspecting`、`renting`、`renewing`。 |
| 结算处理 | `settlement_pending`、`payment_due`、`refund_pending`。 |
| 异常风险 | `overdue`、`accident_processing` 或存在风险标记。 |
| 企业月结 | 企业月结订单。 |
| 完成后追缴 | 存在 `postCharge` 的订单。 |
| 待收/待退 | 主状态为 `payment_due`、`refund_pending`，或完成后追缴处于待通知、已通知、待支付。 |

## 4. 列表字段定义

| 字段模块 | 包含信息 | 展示规则 |
| :--- | :--- | :--- |
| 订单信息 | 订单号、下单时间 | 订单号使用可点击链接，点击进入订单详情页。 |
| 客户信息 | 客户姓名、手机号、会员等级、企业简称、风险标记 | 企业客户展示企业简称标签；黑名单、证件异常、需联系客户等风险标记展示在客户信息下方。 |
| 车辆信息 | 车组、分派车型、车牌号、派车状态 | 第一行展示客户预订的车组。已分派车辆时，第二行展示分派车型，第三行展示车牌号；派车状态使用轻量图标表达，并通过鼠标悬停查看完整状态文字。未分派车辆时，只展示车组和“待派车”提示，不展示车型和车牌占位。 |
| 行程时间 | 取车时间、还车时间、今日标记 | 今日取车或今日还车展示“今日”标签；逾期未取高亮取车时间，逾期未还高亮还车时间。 |
| 取还站点 | 取车门店、车辆所在据点、订单当前还车门店、归还据点 | 展示格式为“门店 - 据点”；同店取还使用普通样式；取车门店与订单当前还车门店不一致时使用警示色。不同据点但门店一致时不按异地还车处理。 |
| 金额 | 订单金额、资金状态 | 金额使用等宽数字样式；金额下方最多展示一个资金状态标签。普通客户自付订单展示支付状态；企业月结订单只展示“企业月结”；非月结订单存在追缴时只展示追缴状态和追缴金额，不再展示原订单支付状态。 |
| 状态 | 订单主状态 | 主状态只展示订单生命周期状态，不在主状态下方叠加辅助标签。 |
| 操作 | 详情、支付、取车、继续取车、延长、补办续租、还车、继续结算、收款、退款、通知客户、追缴收款、查看费用 | 操作按钮根据主状态和辅助标签动态展示。 |

列表中客户信息、企业简称、车型、车组、车牌、取还门店名称、门店地址、订单来源和金额口径均读取订单历史信息快照。筛选条件可使用主数据 ID 或快照 ID，但列表展示不得直接读取当前主数据名称覆盖历史订单。门店或车组后续改名时，历史订单列表仍展示下单时的名称。

## 6. 辅助标签展示

订单列表采用“一个主状态 + 必要辅助标签”的展示方式。主状态用于表达订单生命周期阶段，辅助标签用于表达派车、支付、追缴和风险信息。辅助标签不叠加在状态列下方，金额列最多只展示一个资金状态标签，避免列表过重。

### 6.1 派车状态 `dispatchStatus`

| 字段值 | 页面显示 | 展示位置 |
| :--- | :--- | :--- |
| `pending` | 待派车 | 车辆信息列第二行文字提示 |
| `pre_assigned` | 已预分配 | 车辆信息列车型后图标，悬停显示完整文字 |
| `locked` | 已硬锁定 | 车辆信息列车型后图标，悬停显示完整文字 |
| `manual_reassigned` | 人工改派 | 车辆信息列车型后图标，悬停显示完整文字 |
| `upgraded` | 免费升等 | 车辆信息列车型后图标，悬停显示完整文字 |

### 6.2 调度风险标记

| 字段值 | 页面显示 | 展示位置 |
| :--- | :--- | :--- |
| `dispatch_manual_required` | 人工调度 | 待派车提示旁图标/风险筛选 |

### 6.3 支付状态 `paymentStatus`

| 字段值 | 页面显示 | 展示位置 |
| :--- | :--- | :--- |
| `unpaid` | 未支付 | 金额列 |
| `deposit_paid` | 已收预授权 | 金额列 |
| `monthly_billing` | 企业月结 | 金额列 |
| `partial_paid` | 部分支付 | 金额列 |
| `paid` | 已结清 | 金额列 |
| `payment_due` | 待补缴 | 金额列 |
| `refund_pending` | 待退款 | 金额列 |
| `refunded` | 已退款 | 金额列 |

### 6.4 追缴状态 `postCharge.status`

| 字段值 | 页面显示 | 展示位置 |
| :--- | :--- | :--- |
| `pending_notify` | 追缴待通知 | 金额列 |
| `notified` | 追缴已通知 | 金额列 |
| `payment_due` | 追缴待支付 | 金额列 |
| `monthly_billing` | 企业月结追缴 | 追缴筛选/详情页/企业账单 |
| `paid` | 追缴已结清 | 金额列 |

追缴费用不改变订单主状态。订单主状态为 `已完成`、`已取消`、`爽约`、`已拒绝` 或 `已关闭` 且存在追缴费用时，状态列继续展示原主状态。非月结订单金额列只展示追缴标签和追缴金额，不再展示原订单支付状态；原订单费用在还车完成或终态形成前已完成结清、退款或取消处理。企业月结订单终态后产生追缴费用时，费用进入企业账单，金额列只展示“企业月结”，不展示客户待支付状态，追缴状态通过筛选、详情页费用管理和企业账单明细追踪。

### 6.5 风险标记 `riskFlags`

| :--- | :--- | :--- |
| `blacklist` | 黑名单 | 客户列 |

| `pickup_overdue_contact` | 需联系客户 | 客户列 |
| `accident` | 事故上报 | 风险筛选/详情页 |
| `violation_pending` | 违章待处理 | 风险筛选/详情页 |
| `self_service_exception` | 自助异常 | 风险筛选/详情页 |

## 7. 操作列规则

| 条件 | 展示按钮 | 点击后动作 |
| :--- | :--- | :--- |
| 任意状态 | 详情 | 进入订单详情页 |
| 主状态为 `pending_payment` | 支付 | 进入订单详情页并打开下单预授权押金确认弹窗 |
| 主状态为 `pending_payment`、`reserved`、`pickup_overdue` | 取消 | 进入订单详情页并打开取消订单入口 |
| 主状态为 `reserved`、`pickup_overdue` | 取车 | 进入订单详情页并打开取车作业入口 |
| 主状态为 `pickup_overdue`，且当前时间超过预计取车时间 + 保留时长 | 标记爽约 | 进入订单详情页并打开爽约处理入口 |
| 主状态为 `inspecting` | 继续取车 | 进入订单详情页并打开取车作业入口；存在取车中断或操作超时时，详情页展示操作锁处理入口 |
| 主状态为 `renting`、`renewing` | 延长 | 进入订单详情页并打开延长用车入口 |
| 主状态为 `overdue` | 补办续租 | 进入订单详情页并打开补办续租入口，需通过逾期续租风控校验 |
| 主状态为 `accident_processing` | 处理事故 | 进入订单详情页并打开事故处理入口 |
| 主状态为 `renting`、`renewing`、`overdue`、`accident_processing` | 还车 | 进入订单详情页并打开还车作业入口 |
| 主状态为 `settlement_pending` | 继续结算 | 进入订单详情页并打开还车费用结算入口 |
| 主状态为 `payment_due` | 收款 | 进入订单详情页并打开还车费用结算入口 |
| 主状态为 `refund_pending` | 退款 | 进入订单详情页并打开还车费用结算入口 |
| 主状态为 `completed`、`cancelled`、`no_show`、`rejected`、`closed` 且 `postCharge.status=pending_notify`，且非企业月结 | 通知客户 | 进入订单详情页费用管理入口 |
| 主状态为 `completed`、`cancelled`、`no_show`、`rejected`、`closed` 且 `postCharge.status=notified/payment_due`，且非企业月结 | 追缴收款 | 进入订单详情页费用管理入口 |
| 主状态为 `completed`、`cancelled`、`no_show`、`rejected`、`closed` 且 `postCharge.status=monthly_billing` | 查看费用 | 进入订单详情页费用管理入口 |
| 主状态为 `cancelled`、`no_show`、`rejected`、`closed` 且不存在追缴状态 | 仅详情 | 终态订单不展示业务处理按钮 |

### 7.1 列表跳转参数

订单列表进入详情页时，通过 `action` 参数控制详情页默认打开的入口。

| 列表按钮 | `action` | 详情页打开入口 |
| :--- | :--- | :--- |
| 支付 | `pay` | 下单预授权押金确认弹窗 |
| 取车/继续取车 | `pickup` | 取车作业弹窗 |
| 还车/继续结算/收款/退款/确认月结 | `return` | 还车作业弹窗或还车费用结算步骤 |
| 延长/补办续租 | `extend` | 延长用车弹窗 |
| 取消 | `cancel` | 取消订单弹窗 |
| 标记爽约 | `no_show` | 爽约处理弹窗 |
| 处理事故 | `accident` | 事故/出险登记弹窗 |
| 通知客户 | `notify` | 费用管理弹窗 |
| 追缴收款 | `collect` | 费用管理弹窗 |
| 查看费用 | `fees` | 费用管理弹窗 |

列表跳转详情页时，需要同时携带订单号、主状态、支付状态、企业结算方式、客户类型、客户信息、企业信息、车辆信息、行程信息、门店信息和追缴信息。详情页根据这些参数恢复当前订单的头部信息、客户信息、金额标签和默认入口。

## 8. 业务规则

### 8.1 默认排序
订单列表默认按创建时间倒序展示，最新创建的订单显示在最前。

### 8.2 待支付订单
`pending_payment` 订单仅出现在客户自付订单中。列表金额列展示 `unpaid`，操作列展示“支付”和“取消”。待支付订单在资源保留时长内占用车组库存，不参与具体车辆自动派车和硬锁定；资源保留超时仍未完成预授权押金确认时，订单自动进入 `cancelled` 并释放库存。点击“支付”后进入订单详情页并自动打开下单预授权押金确认弹窗；弹窗仅支持线上预授权，不支持现金押金、线下押金或仅登记线下凭证；弹窗关闭、支付失败或未完成后续操作时，列表状态仍保持“待支付”。

企业月结订单不展示“支付”按钮。企业月结订单创建后直接进入“已预订”，金额列展示“企业月结”。企业月结订单完成后产生追缴费用时，费用进入企业账单，金额列仍只展示“企业月结”，操作列展示“查看费用”，不展示“通知客户”“追缴收款”“收款”或“退款”。

### 8.3 模拟数据覆盖

订单列表原型数据需要覆盖以下资金和操作场景：

| 场景 | 原型表现 |
| :--- | :--- |
| 个人待支付 | 主状态为 `pending_payment`，金额列展示未支付，操作列展示支付和取消。 |
| 非月结企业待支付 | 主状态为 `pending_payment`，客户列展示企业简称，金额列展示未支付，操作列展示支付和取消。 |
| 企业月结已预订 | 主状态为 `reserved`，金额列展示企业月结，操作列不展示支付。 |
| 客户自付待补缴 | 主状态为 `payment_due`，金额列展示待补缴，操作列展示收款。 |
| 客户自付待退款 | 主状态为 `refund_pending`，金额列展示待退款，操作列展示退款。 |
| 客户自付追缴待通知 | 主状态保持原终态，金额列展示追缴待通知，操作列展示通知客户。 |
| 客户自付追缴已通知/待支付 | 主状态保持原终态，金额列展示追缴已通知或追缴待支付，操作列展示追缴收款。 |
| 客户自付追缴已结清 | 主状态保持原终态，金额列展示追缴已结清，操作列不展示追缴收款。 |
| 企业月结追缴 | 主状态保持原终态，金额列展示企业月结，操作列展示查看费用。 |

### 8.4 权限隔离
* 普通店员仅能查看所属门店相关订单。
* 店长可查看本门店全部订单。
* 管理员可查看全部门店订单。
* 用户无某业务线授权时，不能搜索到该业务线订单。

### 8.5 跨业务线搜索
筛选区为门市租车订单的全局筛选区。账号同时拥有门市租车和 24h 自助业务线授权时，搜索结果可按业务线 Tab 展示；账号无对应业务线授权时，不展示该业务线订单。

### 8.6 数据导出
导出按钮导出当前筛选结果。导出文件字段与列表字段保持一致，并包含主状态、派车状态、支付状态、追缴状态和风险标记。

## 9. UI/UX 规则

* `已取消`、`已关闭` 的行整行灰置，降低历史终态订单的视觉权重。
* 今日取车或今日还车的订单展示“今日”标签。
* `逾期未取` 的取车时间使用橙色高亮，并展示警示图标。
* `逾期未还` 的还车时间使用红色高亮，并展示警示图标。
* 异地还车订单在取还门店列使用警示色展示。
* 金额列最多展示一个资金状态标签。普通客户自付订单展示支付状态；非月结追缴订单展示追缴状态和追缴金额；企业月结订单统一展示“企业月结”。
* 状态列只展示主状态；事故、违章、取车中断、操作超时等风险信息通过筛选项、客户列风险标记、金额列追缴标签或详情页呈现。

## 10. 验收标准

| 场景 | 验收标准 |
| :--- | :--- |
| 状态筛选 | 支持多选具体状态和状态分组；选择分组后展示该分组内全部状态订单。 |
| 运营统计 | 统计卡片按当前业务线计算数量，点击后按对应业务口径筛选；“待收/待退”必须同时覆盖主状态待补缴/待退款和已完成追缴待处理订单。 |
| 结算方式筛选 | 支持客户自付和企业月结筛选；非月结企业预订归入客户自付。 |
| 追缴筛选 | 支持存在追缴、追缴待通知、已通知、待支付、企业月结追缴、已结清筛选。 |
| 风险筛选 | 支持按客户风险和状态风险标记筛选。 |
| 逾期未取 | 订单主状态显示“逾期未取”，取车时间橙色高亮，操作列展示“取车”；超过保留时长后展示“标记爽约”。 |
| 爽约 | 订单主状态显示“爽约”，操作列仅展示“详情”。 |
| 验车中 | 订单主状态显示“验车中”，操作列展示“继续取车”。 |
| 取车中断 | 取车窗口关闭且未确认发车时，订单保持“验车中”，操作列展示“继续取车”，详情页展示操作锁信息。 |
| 操作超时 | 操作锁超过有效期时，订单保持“验车中”，详情页允许具备操作锁释放权限的人员释放操作锁；释放后按当前时间重新判定“已预订”“逾期未取”或“爽约”。 |
| 逾期未还 | 订单主状态显示“逾期未还”，还车时间红色高亮，操作列展示“还车”。 |
| 出险处理中 | 订单主状态显示“出险处理中”，操作列展示“处理事故”和“还车”。 |
| 待支付 | 订单主状态显示“待支付”，金额列展示未支付状态，操作列展示“支付”。 |
| 待支付支付 | 点击“支付”后进入订单详情页并打开下单预授权押金确认弹窗；预授权成功后订单更新为“已预订”，金额列更新为“已收预授权”。 |
| 企业月结 | 企业月结订单主状态按生命周期展示，金额列展示“企业月结”；待结算时操作列展示“确认月结”，不展示支付、收款或退款入口。 |
| 终态追缴 | 主状态保持原终态，非月结订单金额列展示追缴状态和追缴金额；客户自付订单待通知时展示“通知客户”，追缴已通知或待支付时展示“追缴收款”；企业月结订单金额列展示“企业月结”，操作列展示“查看费用”。 |
| 已取消/爽约/已拒绝/已关闭 | 无追缴状态时，终态订单整行灰置或降低视觉权重，操作列仅展示“详情”。 |
| 待结算 | 验车数据已提交但未完成还车确认，状态列展示“待结算”；客户自付订单操作列展示“继续结算”，企业月结订单展示“确认月结”。 |
| 待补缴 | 金额列展示“待补缴”，状态列展示“待补缴”，客户自付订单操作列展示“收款”。 |
| 待退款 | 金额列展示“待退款”，状态列展示“待退款”，客户自付订单操作列展示“退款”。 |
| 派车状态 | 车辆信息列不使用文字标签堆叠。待派车以文字提示展示；已预分配、已硬锁定、人工改派、免费升等等状态使用车型后的轻量图标表达，悬停显示完整状态文字。 |
| 风险标记 | 客户列展示黑名单、证件异常、需联系客户；事故上报、违章待处理等风险信息通过筛选、详情页和费用/事故入口呈现。 |
| 异地还车 | 取车门店和订单当前还车门店不一致时，取还站点列使用警示样式；同一门店下不同据点不使用异地还车样式。 |
| 权限隔离 | 无业务线授权时，筛选和列表均不展示该业务线订单。 |
