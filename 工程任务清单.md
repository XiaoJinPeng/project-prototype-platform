## 一、工程目标

将当前项目分散的 HTML 原型统一接入 Vue 3 工程，形成可持续维护、统一预览和交付前端开发参考的业务原型平台。

工程只承载原型页面和模拟数据，不连接真实后端，不实现生产身份认证或数据持久化。

## 四、页面迁移验收

以下问题已纳入全工程检查并修复：

- 空路由、刷新白屏和未返回模板变量。
- 页面内残留旧侧栏、旧顶栏、客户端门户脚本和 `h-screen` 外壳。
- 原 HTML 的弹窗、抽屉、表格、标签页和表单遗漏。
- 行销方案新增抽屉与弹窗遗漏。
- 动态调价弹窗遗漏。
- 平台账户管理旧 Element Plus 全局 API 导致页面异常。
- 订单详情、客户等级方案、基础产品定价等页面边框、圆角和图标基础样式丢失。
- 页面链接跳回旧 `.html` 文件。
- 营运端和企业端菜单层级、顺序与路由不一致。

自动审计命令：

```powershell
npm run audit:views
```

审计范围包括 51 个营运端、企业端及登录组件，并对照源 HTML 检查主要 Element Plus 业务节点数量。

## 六、代码与数据规范

- 页面只保留业务内容区及同级弹窗、抽屉，不包含统一外壳。
- 业务页面统一登记在所属项目包的 `page-definitions.js`，由项目注册表生成动态路由和菜单。
- 已完成抽离的共享模拟数据随业务页面迁入所属项目包的 `data`；后续历史页面的模拟数据默认保留在各页面内部，不再继续抽离。
- 页面专属表单、筛选、弹窗和临时演示状态保留在当前组件。
- Element Plus 消息组件必须显式导入，禁止使用 `ElementPlus.ElMessage` 全局命名空间。
- 不允许保留外部 CDN、旧 `.html` 跳转、`window.location` 或 `app.mount('#app')`。
- 新增页面必须执行 [MIGRATION_GUIDE.md](./MIGRATION_GUIDE.md)。
- 规范 HTML 和平台可回导 HTML 均可通过 [PAGE_TRANSFER_GUIDE.md](./PAGE_TRANSFER_GUIDE.md) 导入；导出 HTML 的运行区仍是演示快照，回导读取其源文件区。

## 七、交付检查

每次批量修改或新增页面后执行：

```powershell
npm run i18n:sync
npm run i18n:traditional
npm run audit:i18n
npm run audit:views
npm run build
```

默认新增页面使用 npm run generate:page 创建，并执行 npm run quality:check。多语言命令仅在任务明确包含翻译时执行。

新增或修改页面还应执行 `npm run test:smoke`；涉及高频演示页面或公共样式时执行 `npm run test:visual`。视觉基线不得在未确认差异原因时直接覆盖。

代码检查通过后，由产品负责人进行浏览器视觉验收，重点确认页面间距、边框、圆角、图标位置、表格高度、滚动区域和弹窗内容。

处理边界：

1. 历史页面发现功能缺失、页面打不开、弹窗遗漏或明显视觉失真时，必须修复。
2. 历史页面已经稳定且没有明确问题时，不因公共组件、样式规范或新模板升级而主动重写。
3. 新页面不得复制上述历史兼容方式，不得新增 `legacy: true`、非 scoped 样式或页面自有的标准标题结构。
4. HTML 导入功能面向专用模板和平台可回导 HTML，不承担历史页面自动清理或批量重构。
