# 企业端公务车后台功能 PRD - 公务车据点管理

## 版本记录

| 版本 | 日期 | 调整概括 |
| --- | --- | --- |
| v1.0 | 2026-06-25 | 补建版本记录，作为当前企业端公务车 PRD 基线版本。 |

## 1. 页面范围

页面用途：企业端管理公务车用车据点，支持企业自建据点和查看平台授权据点，作为公务车车辆停放、员工预约查询和取还车展示的据点来源。

适用客户端：企业端。

适用角色：企业总管理员、分公司管理员。

对应营运端：`02_原型和PRD/原型/营运端/stations.html`。营运端服务站点管理仍作为平台据点主数据底座，本 PRD 不改造营运端服务站点管理页面。

## 2. 业务口径

公务车据点分为两类：

| 据点类型 | 字段值 | 来源 | 据点管理方 | 车辆管理方 | 企业端操作 |
| --- | --- | --- | --- | --- | --- |
| 平台授权据点 | `PLATFORM` | 营运方建立并授权给企业使用 | 营运端 | 营运端 | 只读查看 |
| 企业自建据点 | `ENTERPRISE` | 企业自行建立的内部停车场、办公园区、厂区等 | 企业端 | 企业端 | 新增、编辑、停用、启用 |

本期使用同一套据点数据底座，不拆成两套据点表。系统通过据点来源字段区分平台授权据点和企业自建据点。

## 3. 数据模型

据点实体复用平台服务站点模型，新增或明确以下字段。

| 字段 | code | 类型 | 必填 | 说明 |
| --- | --- | --- | --- | --- |
| 据点来源 | `stationSourceType` | enum | 是 | `PLATFORM` 平台授权据点；`ENTERPRISE` 企业自建据点 |
| 企业客户 | `corporateId` | id | `ENTERPRISE` 必填 | 据点所属企业客户；用于企业级数据隔离 |
| 管理归属分公司 | `ownerBranchId` | id | `ENTERPRISE` 必填 | 负责管理该企业自建据点的分公司 |
| 公务车服务能力 | `officialVehicleEnabled` | boolean | 是 | 是否作为公务车据点使用 |
| 据点用途 | `stationPurpose` | enum | 是 | 企业自建公务车据点固定为 `DEFAULT`，不可配置为团体票券专用 |
| 据点状态 | `stationStatus` | enum | 是 | `enabled` 启用；`disabled` 停用 |

枚举规则：

| 字段 | 枚举值 | 中文名称 | 说明 |
| --- | --- | --- | --- |
| `stationSourceType` | `PLATFORM` | 平台授权据点 | 由营运端建立和维护，企业端只读 |
| `stationSourceType` | `ENTERPRISE` | 企业自建据点 | 由企业端建立和维护，只归属当前企业 |
| `stationPurpose` | `DEFAULT` | 默认 | 可用于普通公务车预约 |

## 4. 数据隔离与可用范围

### 4.1 企业级隔离

1. `ENTERPRISE` 据点必须写入 `corporateId`。
2. 企业端只能读取本企业 `corporateId` 下的 `ENTERPRISE` 据点。
3. `ENTERPRISE` 据点不得被其他企业查看、选择或使用。
4. `PLATFORM` 据点只有在营运端公务车服务能力中授权给当前企业后，才在该企业端展示。

### 4.2 分公司管理归属

1. `ownerBranchId` 表示企业自建据点的管理归属分公司。
2. 企业总管理员可以查看和管理本企业全部 `ENTERPRISE` 据点。
3. 分公司管理员只能查看和管理 `ownerBranchId` 为本分公司的 `ENTERPRISE` 据点。
4. `ownerBranchId` 不等同于员工可用范围。员工能否使用据点，以分公司可用据点关系为准。

### 4.3 分公司可用据点

1. 分公司可用据点仍在分公司管理中配置。
2. 新增 `ENTERPRISE` 据点保存成功后，系统自动将该据点加入 `ownerBranchId` 对应分公司的可用据点。
3. 企业总管理员可以在分公司管理中，将本企业 `ENTERPRISE` 据点分配给其他分公司使用。
4. 分公司管理员新增本分公司据点后，只能自动加入本分公司可用据点，不能直接分配给其他分公司。
5. `PLATFORM` 据点被授权给企业后，不自动开放给所有分公司；仍需企业总管理员在分公司管理中分配给具体分公司。

员工预约公务车时，系统只查询员工所属分公司可用据点内、已上架、未被管控且时间不冲突的车辆。

## 5. 页面结构

企业端公务车据点管理使用两个 tab。

| Tab | 数据范围 | 操作权限 |
| --- | --- | --- |
| 我的据点 | 当前企业的 `ENTERPRISE` 据点 | 新增、编辑、停用、启用 |
| 平台授权据点 | 营运端授权给当前企业的 `PLATFORM` 据点 | 只读查看 |

## 6. 我的据点列表

| 字段 | 说明 |
| --- | --- |
| 据点名称 | 据点名称、地址 |
| 管理归属分公司 | `ownerBranchId` 对应分公司 |
| 可用分公司 | 已配置可使用该据点的分公司数量或名称 |
| 据点类型 | 办公园区、厂区、仓库、停车场、其他 |
| 车位概况 | 总车位、当前停放车辆数、剩余空位 |
| 据点状态 | 启用、停用 |
| 操作 | 编辑、停用 / 启用 |

列表规则：

1. 企业总管理员展示本企业全部企业自建据点。
2. 分公司管理员只展示管理归属为本分公司的企业自建据点。
3. 停用据点仍保留在列表中，不再进入新预约可用范围。
4. 本期不提供删除入口。

## 7. 平台授权据点列表

| 字段 | 说明 |
| --- | --- |
| 据点名称 | 据点名称、地址 |
| 所属车行 / 门市 | 营运端组织归属 |
| 可用分公司 | 当前企业已分配可使用该据点的分公司 |
| 车位概况 | 总车位、当前停放车辆数、剩余空位 |
| 公务车服务状态 | 启用、暂停 |
| 据点状态 | 启用、停用 |
| 操作 | 详情 |

列表规则：

1. 企业端对平台授权据点只读。
2. 平台授权据点不展示编辑、停用、启用、删除入口。
3. 营运端取消企业授权后，该据点从企业端平台授权据点列表移除。
4. 平台授权据点的地址、车位、服务状态读取营运端最新主数据；历史订单仍读取订单快照。

## 8. 新增企业自建据点

入口：我的据点 tab 的“新增据点”按钮。

新增方式：弹窗。

新增字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 据点名称 | 是 | 用于企业端列表、车辆所在位置、预约取还车说明 |
| 管理归属分公司 | 是 | 企业总管理员可选择分公司；分公司管理员固定为本分公司 |
| 据点类型 | 是 | 办公园区、厂区、仓库、停车场、其他 |
| 地址 | 是 | 用于 APP 展示、导航和取还车指引 |
| 经纬度 | 否 | 用于地图定位；未填写时不做地图定位 |
| 电子围栏 | 否 | 用于取还车位置校验；未配置时不做围栏校验 |
| 总车位数 | 是 | 用于车位容量展示和提醒 |
| 进出方式 | 否 | 门禁卡、车牌辨识、人工管理、其他 |
| 固定车位号 | 否 | 企业自有车位编号 |
| 联络人 | 否 | 现场联系人 |
| 联络电话 | 否 | 现场联系电话 |
| 服务时间 | 否 | 预约参考信息，不作为本期强制预约校验 |
| 取还车说明 | 否 | APP 或后台展示的现场指引 |
| 内部备注 | 否 | 企业后台内部备注，不对员工展示 |

保存规则：

1. 保存时写入 `stationSourceType = ENTERPRISE`。
2. 保存时写入当前企业 `corporateId`。
3. 保存时写入 `ownerBranchId`。
4. 保存时默认 `officialVehicleEnabled = true`。
5. 保存时默认 `stationPurpose = DEFAULT`。
6. 保存时默认 `stationStatus = enabled`。
7. 保存成功后，系统自动将该据点加入 `ownerBranchId` 对应分公司的可用据点。
8. 新增企业自建据点不需要营运端审批。
9. 新增企业自建据点不选择所属车行、所属门市，不写入营运端组织归属。
10. 新增企业自建据点不填写租赁押金、月租成本、水电成本、人事成本等营运字段。

## 9. 编辑企业自建据点

1. 企业总管理员可以编辑本企业全部企业自建据点。
2. 分公司管理员只能编辑管理归属为本分公司的企业自建据点。
3. 可编辑字段与新增字段一致。
4. `stationSourceType` 不可编辑。
5. `corporateId` 不可编辑。
6. 企业总管理员可以调整 `ownerBranchId`。调整后，据点管理权转移给新分公司；原分公司的可用据点关系不自动删除，是否继续可用由企业总管理员在分公司管理中调整。
7. 分公司管理员不可调整 `ownerBranchId`。
8. 编辑保存后写入业务操作记录，记录变更前值、变更后值、操作人和操作时间。

## 10. 停用与启用

1. 停用只适用于 `ENTERPRISE` 据点。
2. 停用后，该据点不再对新预约可见。
3. 停用后，车辆仍可保留在该据点资料中，但不得作为新预约可用车辆展示。
4. 停用不自动取消已创建预约。
5. 已创建预约继续按订单快照展示原据点信息。
6. 启用后，该据点恢复进入分公司可用据点范围。
7. 停用和启用必须写入业务操作记录。

## 11. 平台授权据点来源

平台授权据点由营运端维护。

授权规则：

1. 营运端在服务站点管理建立 `PLATFORM` 据点。
2. 营运端为据点开启公务车服务能力。
3. 营运端在公务车服务能力配置中关联企业客户 `corporateIds`。
4. 被关联企业可在企业端“平台授权据点”tab 查看该据点。
5. 授权关系复用营运端公务车服务能力配置，不新建企业端授权表。

分公司使用规则：

1. `corporateIds` 只表示该平台据点授权给企业客户。
2. 平台授权据点是否开放给某个分公司，仍由企业总管理员在分公司管理中配置。
3. 分公司管理员不可自行把平台授权据点分配给本分公司。

## 12. 车辆入驻规则

本期车辆与据点来源严格匹配。

| 据点来源 | 可停放车辆 | 车辆管理方 | 说明 |
| --- | --- | --- | --- |
| `ENTERPRISE` | 企业自有公务车辆 | 企业端 | 企业自建据点只接企业自有车辆 |
| `PLATFORM` | 营运方提供公务车辆 | 营运端 | 平台授权据点只接营运方车辆 |

规则：

1. 企业自有车辆选择所在据点时，只能选择 `ENTERPRISE` 据点。
2. 营运方提供车辆选择所在据点时，只能选择 `PLATFORM` 据点。
3. 企业端新增或编辑企业自有车辆时，所在据点下拉只展示当前企业可用的 `ENTERPRISE` 据点。
4. 企业端不可编辑营运方车辆所在据点。
5. 本期不支持企业自有车辆停放到平台授权据点。
6. 本期不支持营运方车辆停放到企业自建据点。
7. 本期不支持同一个物理停车场同时以一个据点记录承载 `ENTERPRISE` 和 `PLATFORM` 两种来源；如同一物理停车场确实存在两种运营模式，系统按两个据点记录管理。

## 13. 预约与查询规则

员工发起公务车预约时：

1. 系统读取员工所属分公司的可用据点。
2. 系统在可用据点内查询车辆。
3. 车辆必须已上架、未被管控、时间不冲突。
4. 车辆必须符合用车管理中配置的员工或部门使用范围。
5. `ENTERPRISE` 据点与 `PLATFORM` 据点在员工端不需要区分展示；员工只看到可预约车辆和取还车地点。
6. 据点停用、服务暂停或未分配给员工所属分公司时，不进入新预约查询范围。

## 14. 存量数据迁移

上线前需要对现有公务车据点进行一次来源分类。

迁移规则：

1. 现有由营运端建立、继续由营运方管理并提供车辆的公务车据点，标记为 `PLATFORM`。
2. 现有由营运端代建、实际属于企业内部停车场并由企业管理的公务车据点，迁移为 `ENTERPRISE`。
3. 迁移为 `ENTERPRISE` 的据点必须补齐 `corporateId` 和 `ownerBranchId`。
4. 迁移为 `ENTERPRISE` 的据点自动加入 `ownerBranchId` 对应分公司的可用据点。
5. 已创建预约不重新解释据点来源，继续读取订单创建时的据点快照。
6. 未分类完成的存量据点不得开放企业端编辑入口。
7. 迁移过程写入系统迁移记录和业务操作记录。

## 15. 与营运端服务站点管理的关系

1. 营运端服务站点管理默认只展示 `PLATFORM` 据点。
2. `ENTERPRISE` 据点不进入营运端服务站点管理默认列表。
3. 营运端如需查看企业自建据点，只通过只读视图或报表入口查看。
4. `ENTERPRISE` 据点不参与分时租赁、甲租乙还、门市租赁、24 小时自助租车和团体票券据点选择。
5. `ENTERPRISE` 据点的操作记录与 `PLATFORM` 据点操作记录分开记录。

## 16. 数据快照

公务车预约创建时保存据点快照：

| 快照字段 | 说明 |
| --- | --- |
| 据点 ID | 创建订单时的据点 ID |
| 据点名称 | 创建订单时的据点名称 |
| 据点地址 | 创建订单时的地址 |
| 据点来源 | `PLATFORM` 或 `ENTERPRISE` |
| 管理归属分公司 | `ENTERPRISE` 据点适用 |
| 营运端组织归属 | `PLATFORM` 据点适用 |

规则：

1. 据点后续改名、改地址、停用、迁移来源，不影响历史预约展示。
2. 已创建预约按订单快照履约，系统不自动取消、不自动改派。
3. 历史统计按订单归属员工分公司统计，不按当前据点管理归属重新计算。

## 17. 异常处理

| 场景 | 系统处理 |
| --- | --- |
| 新增据点缺少必填字段 | 阻止保存并提示缺少字段 |
| 企业自建据点未配置经纬度 | 保存成功，但不做地图定位和围栏校验 |
| 企业自建据点停用后仍有车辆停放 | 允许停用，车辆不进入新预约查询范围 |
| 企业自建据点停用后存在未来预约 | 预约保留，详情按快照展示，由企业内部处理 |
| 所在据点与车辆来源不匹配 | 阻止保存车辆所在据点 |
| 平台授权据点被营运端取消授权 | 企业端不再展示该据点，不影响历史订单快照 |
| 存量据点未完成来源分类 | 不开放企业端编辑入口 |

## 18. 本期不做

1. 不做企业自有车辆入驻 `PLATFORM` 据点。
2. 不做营运方车辆入驻 `ENTERPRISE` 据点。
3. 不做一个据点同时作为 `ENTERPRISE` 和 `PLATFORM` 的混合模式。
4. 不做企业端向营运端申请据点审批的工单流程。
5. 不做跨企业共享 `ENTERPRISE` 据点。
6. 不做营运端对企业自建据点的编辑、停用或审核。
7. 不做据点电子围栏的可视化绘制工具。
8. 不做服务时间对预约时间的强制拦截。
9. 不做据点删除。

## 19. 验收标准

1. 企业端公务车据点管理页面展示“我的据点”和“平台授权据点”两个 tab。
2. 我的据点只展示当前企业 `ENTERPRISE` 据点。
3. 平台授权据点只展示营运端授权给当前企业的 `PLATFORM` 据点。
4. 企业总管理员可以新增、编辑、停用、启用本企业自建据点。
5. 分公司管理员只能新增、编辑、停用、启用本分公司管理归属的自建据点。
6. 新增企业自建据点保存后，自动加入管理归属分公司的可用据点。
7. 平台授权据点在企业端只读，无编辑、停用、启用、删除入口。
8. 分公司可用据点可同时选择平台授权据点和企业自建据点。
9. 员工预约只查询员工所属分公司可用据点内的可预约车辆。
10. 企业自有车辆所在据点只能选择企业自建据点。
11. 营运方车辆所在据点只能选择平台授权据点。
12. 据点停用后不进入新预约查询范围，已创建预约保留。
13. 历史预约展示读取据点快照，不被据点后续修改影响。
14. 未完成来源分类的存量据点不开放企业端编辑入口。
