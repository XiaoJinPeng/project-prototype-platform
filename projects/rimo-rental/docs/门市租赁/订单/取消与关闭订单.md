## 1. 文档概述

### 1.1 背景
取消、爽约、拒绝履约和关闭订单都是订单未按正常“取车 - 用车 - 还车”链路完成时的终止处理。不同终止原因对应不同责任归属、费用规则、预授权处理、车辆释放方式和权限要求，需要在订单状态、费用中心、订单列表和详情页中保持一致。

### 1.2 目标
* **责任清晰**：区分客户主动取消、逾期未取爽约、取车资质拒绝、门店责任关闭。
* **资金清晰**：根据责任归属处理取消费、爽约费、拒绝履约费用、预授权释放、预授权扣款和退款。
* **库存释放**：终止类操作完成后释放车组库存、具体车辆和派车关系。
* **权限可控**：关闭订单、拒绝履约、特殊减免和预授权扣款必须记录权限与审计日志。
* **状态闭环**：订单进入 `cancelled`、`no_show`、`rejected`、`closed` 后不再回到履约链路。

### 1.3 功能范围

| 功能 | 目标状态 | 业务含义 | 责任归属 |
| :--- | :--- | :--- | :--- |
| 取消订单 | `cancelled` | 未取车前的标准取消。 | 客户责任或门店同意取消。 |
| 标记爽约 | `no_show` | 逾期未取且超过保留时长后，客户未到店履约。 | 客户责任。 |
| 拒绝履约 | `rejected` | 取车办理中因风控、证件、资质原因拒绝交车。 | 客户责任或风控责任。 |
| 关闭订单 | `closed` | 门店责任、不可抗力或重大客诉导致订单异常作废。 | 客户无责。 |

### 1.4 页面入口
* 订单详情页更多操作：展示“取消订单”“关闭订单”。
* 取车向导：`inspecting` 状态下展示“拒绝履约”入口。
* 订单列表：`pending_payment`、`reserved`、`pickup_overdue` 展示“取消”；`pickup_overdue` 展示“标记爽约”。
* 订单列表：终态 `cancelled`、`no_show`、`rejected`、`closed` 仅保留“详情”。
* 对应原型文件：`order_detail.html`、`store_orders.html`。

## 2. 状态与入口规则

| 场景 | 规则 |
| :--- | :--- |
| 允许取消 | `pending_payment`、`reserved`、`pickup_overdue`。 |
| 允许标记爽约 | `pickup_overdue`，且超过门店配置的保留时长。 |
| 允许拒绝履约 | `inspecting`，且取车资质校验、证件校验、黑名单或风控校验不通过。 |
| 允许关闭订单 | `pending_payment`、`reserved`、`pickup_overdue`、`inspecting`、`renting`、`renewing`、`overdue`、`accident_processing`。 |
| 禁止终止操作 | `settlement_pending`、`payment_due`、`refund_pending` 需先完成还车结算；`completed`、`cancelled`、`no_show`、`rejected`、`closed` 为终态。 |
| 终态处理 | 进入 `cancelled`、`no_show`、`rejected`、`closed` 后，订单不再展示取车、续租、还车入口。 |

### 2.1 状态流转口径

| 流转 | 触发时机 | 处理规则 |
| :--- | :--- | :--- |
| `pending_payment` -> `cancelled` | 待支付资源保留超时、支付超时或客户取消。 | 未产生有效预授权时直接取消并释放车组库存和派车关系；已产生预授权时释放预授权。 |
| `reserved` -> `cancelled` | 客户未取车前取消。 | 根据取消规则计算取消费，释放库存和预授权剩余额度。 |
| `pickup_overdue` -> `cancelled` | 门店与客户确认取消。 | 按取消规则计算费用，释放库存。 |
| `pickup_overdue` -> `no_show` | 超过保留时长后系统自动处理或门店手动标记爽约。 | 生成爽约费，释放库存，按规则扣取预授权或生成待补缴。 |
| `inspecting` -> `rejected` | 取车办理中资质或风控不通过。 | 释放车辆，保留拒绝原因和取车草稿记录，按责任处理费用。 |
| 可关闭状态 -> `closed` | 门店责任、不可抗力或重大客诉异常作废。 | 客户无责，全额退款，释放库存，写入高权限审计日志。 |
| 企业月结终止 | 企业月结订单取消、爽约、拒绝履约或关闭。 | 不处理客户支付、退款和预授权；客户有责费用生成企业月结挂账，客户无责关闭不生成应收挂账。 |

终止类订单确认后，系统必须释放当前订单的车组库存、具体车辆绑定和派车关系。释放范围包含 `pending`、`pre_assigned`、`locked`、`manual_reassigned`、`upgraded` 等派车状态；历史派车记录和人工改派记录保留在订单操作记录中，不作为后续可用车辆占用。

## 3. 取消订单

### 3.1 适用状态
取消订单适用于未取车前的标准取消：
* `pending_payment`
* `reserved`
* `pickup_overdue`

`inspecting` 状态不走普通取消。取车办理已开始时，需要先释放取车操作锁回到取车前状态，或按拒绝履约、关闭订单处理。

### 3.2 取消费用规则

```text
固定金额模式：取消费 = 规则快照中的固定取消费或当前阶梯固定金额
预授权比例模式：取消费 = 订单预授权押金金额 × 规则快照中的预授权扣取比例或当前阶梯比例
应退款金额 = 已实收金额 + 已扣款金额 - 取消费 - 已退款金额
```

| 场景 | 规则 |
| :--- | :--- |
| 支付超时取消 | 未产生有效预授权或收款时，取消费为 0。 |
| 冷静期取消 | 下单后配置时间内取消，取消费为 0。 |
| 免费取消 | 距预计取车时间大于免费取消阈值，取消费为 0。 |
| 有责取消 | 距预计取车时间小于免费取消阈值，按订单规则快照扣取取消费。 |
| 逾期未取后取消 | 客户已超过取车时间但未达到爽约保留时长，按有责取消处理。 |
| 特殊减免 | 店长或管理员可将取消费减免为 0，必须填写减免原因和备注。 |
| 固定金额 | 按规则快照中的固定金额收取取消费；开启阶梯扣费时，按命中的阶梯固定金额收取。 |
| 预授权比例 | 按规则快照中的预授权比例收取取消费；开启阶梯扣费时，按命中的阶梯预授权比例收取。 |

### 3.3 预授权处理
取消订单涉及下单阶段预授权押金，处理规则如下：

| 资金状态 | 处理规则 |
| :--- | :--- |
| 未预授权 | 直接取消，无资金动作。 |
| 预授权冻结中，取消费为 0 | 释放预授权，生成 `pre_auth_release` 资金记录。 |
| 预授权冻结中，取消费 > 0 | 按取消费执行预授权扣款 `pre_auth_capture`，剩余冻结额度释放。 |
| 已存在实收款 | 按应退款金额生成退款记录。 |
| 取消费大于可扣金额 | 订单进入已取消，费用中心保留待收取消费；金额列展示待补缴。 |

### 3.4 企业月结取消规则
企业月结订单取消时不处理预授权、客户补缴和门店退款。取消费为 0 时仅释放库存并取消预计月结；取消费大于 0 时生成取消费，费用项状态为 `monthly_billing`，并同步写入 `monthly_bill` 挂账记录，费用统一进入企业月结账单，订单主状态更新为 `cancelled`。

### 3.5 操作流程
1. 门店点击“取消订单”。
2. 系统根据当前状态、预计取车时间、下单时间、已收款和预授权状态试算取消费。
3. 门店选择取消原因，填写备注。
4. 需要特殊减免时，勾选“特殊减免”，选择减免原因并填写备注。
5. 点击“确认取消订单”。
6. 系统释放库存，处理预授权/退款/取消费，写入费用中心和审计日志。
7. 订单主状态更新为 `cancelled`。

## 4. 逾期未取与爽约

### 4.1 逾期未取 `pickup_overdue`
`pickup_overdue` 不是终态。客户超过预计取车时间且仍未到店时，订单进入逾期未取。门店需要联系客户并选择后续处理方式。

| 处理方式 | 规则 |
| :--- | :--- |
| 继续取车 | 客户到店后可继续办理取车，订单进入 `inspecting`。 |
| 取消订单 | 客户确认不取车时，按取消订单处理。 |
| 标记爽约 | 超过保留时长且客户未到店，按爽约处理。 |

### 4.2 爽约 `no_show`
爽约是客户未在保留时长内到店履约的违约终态。

| 项目 | 规则 |
| :--- | :--- |
| 触发状态 | 仅允许从 `pickup_overdue` 标记。 |
| 触发条件 | 当前时间超过预计取车时间 + 保留时长。 |
| 操作方式 | 订单规则快照中开启自动释放资源时，系统在达到爽约条件后自动释放车辆资源并将订单置为 `no_show`；自动释放关闭或自动任务未执行前，门店可手动标记爽约。 |
| 费用规则 | 生成爽约费，金额按规则快照中的固定金额或预授权比例计算。 |
| 资金处理 | 客户自付订单优先从预授权扣取爽约费，剩余预授权释放，不足部分进入待补缴；企业月结订单生成 `monthly_billing` 费用项和 `monthly_bill` 挂账记录。 |
| 库存处理 | 释放车组库存、具体车辆和派车关系。 |
| 状态结果 | 订单主状态进入 `no_show`。 |

### 4.3 爽约操作流程
1. 订单进入 `pickup_overdue`。
2. 系统按订单规则快照计算保留时长和爽约费用。
3. 达到爽约条件且自动释放资源开启时，系统自动释放车辆资源，客户自付订单扣取爽约费或生成待补缴，企业月结订单生成爽约费月结挂账，订单进入 `no_show`。
4. 自动释放关闭或自动任务未执行前，门店可点击“标记爽约”手动处理。
5. 手动处理时，系统展示爽约费；客户自付订单展示预授权可扣金额、剩余释放金额和待补缴金额，企业月结订单展示企业挂账金额。
6. 门店填写爽约原因和联系备注后确认，系统释放车辆资源，订单进入 `no_show`。

## 5. 拒绝履约

### 5.1 适用场景
拒绝履约适用于取车办理已开始但客户不满足交车条件：
* 客户命中黑名单。
* 身份证、驾照或驾驶资质不通过。
* 驾照过期或证件照片不完整。
* 风控系统拒绝。
* 客户无法完成预授权或取车资金要求。
* 门店按规则判断客户无法安全履约。

### 5.2 处理规则
| 项目 | 规则 |
| :--- | :--- |
| 触发状态 | `inspecting`。 |
| 状态结果 | `rejected`。 |
| 车辆处理 | 释放已锁车辆和派车关系。 |
| 草稿处理 | 保留取车办理草稿、证件校验结果、风控结果和操作人。 |
| 费用处理 | 客户责任时可生成拒绝履约费用；门店责任时不收取费用。企业月结订单产生的拒绝履约费用统一进入企业月结账单。 |
| 预授权处理 | 客户自付订单有费用时可按费用扣取预授权，无费用时释放预授权；企业月结订单不处理预授权。 |
| 权限要求 | 拒绝履约需店长或具备风控处理权限的人员确认。 |

### 5.3 操作流程
1. 取车向导中点击“拒绝履约”。
2. 选择拒绝原因，填写备注。
3. 系统展示费用处理方式和预授权处理结果。
4. 点击确认后，系统释放车辆，处理费用和预授权，写入操作记录。
5. 订单主状态更新为 `rejected`。

## 6. 关闭订单

### 6.1 适用场景
关闭订单是高权限异常作废操作，适用于客户无责场景：
* 门店无车可交付。
* 车辆严重故障导致无法履约。
* 不可抗力导致订单无法继续。
* 重大客诉需要全额退款安抚。
* 出险、事故或门店责任导致订单无法按原流程结算。

### 6.2 处理规则
| 项目 | 规则 |
| :--- | :--- |
| 责任归属 | 客户无责。 |
| 状态结果 | `closed`。 |
| 资金处理 | 客户自付订单全额退还已实收款并释放未扣款预授权；企业月结订单不处理客户退款和预授权。 |
| 费用处理 | 不向客户或企业收取取消费、爽约费或拒绝履约费用。 |
| 车辆处理 | 释放库存或转入维修、整备、停用状态。 |
| 权限要求 | 店长、区域经理或管理员。 |
| 审计要求 | 必填关闭原因、备注、客户沟通确认和操作人。 |

### 6.3 操作流程
1. 操作人点击“关闭订单”。
2. 客户自付订单展示已收款项、预授权状态和全额退款金额；企业月结订单展示关闭后不生成企业挂账。
3. 操作人选择关闭原因，填写备注。
4. 操作人勾选“已与客户沟通并同意全额退款”。
5. 点击“确认关闭订单”。
6. 客户自付订单全额退款并释放预授权；企业月结订单取消预计挂账。系统释放车辆、写入审计日志。
7. 订单主状态更新为 `closed`。

## 7. 费用中心规则

| 费用类型 | 字段值 | 来源 | 阶段 | 规则 |
| :--- | :--- | :--- | :--- | :--- |
| 取消费 | `cancellation_fee` | 系统 | `rental` | 标准取消产生，客户责任时收取；待支付资源保留超时取消不生成取消费。 |
| 爽约费 | `no_show_fee` | 系统 | `rental` | 逾期未取并标记爽约后产生。 |
| 拒绝履约费用 | `rejection_fee` | 系统/人工 | `rental` | 取车资质不通过且客户责任时产生。 |
| 关闭退款 | `close_refund` | 系统 | `rental` | 关闭订单时按全额退款记录。 |

资金记录规则：
* 预授权冻结释放写入 `pre_auth_release`。
* 预授权扣取消费、爽约费、拒绝履约费用写入 `pre_auth_capture`。
* 实收款退款写入 `refund`。
* 企业月结订单的取消费、爽约费和拒绝履约费用写入 `monthly_billing` 费用项，并同步生成 `monthly_bill` 交易记录。
* 特殊减免写入 `waiver`。
* 所有费用、资金动作和状态变更写入审计日志。

## 8. UI/UX 规则

* 取消订单弹窗展示当前状态、取消类型、取消费、应退款、预授权处理方式和取消原因。
* 待支付资源保留超时由系统自动取消，订单释放车组库存、具体车辆绑定和派车关系，写入系统操作日志，不弹出人工确认窗口。
* 爽约弹窗展示预计取车时间、保留时长、爽约费、费用规则来源、预授权可扣金额、待补缴金额和联系备注。
* 拒绝履约弹窗展示拒绝原因、费用处理、预授权处理和权限提示。
* 关闭订单弹窗展示高风险提示、全额退款明细、预授权释放结果、关闭原因和客户沟通确认。
* 企业月结订单的取消、爽约、拒绝履约和关闭弹窗隐藏客户收款、退款和预授权处理区，展示企业挂账金额或“不生成企业挂账”。
* `cancelled`、`no_show`、`rejected`、`closed` 行在订单列表中降低视觉权重，只保留“详情”。
* 取消、爽约、拒绝履约和关闭订单确认后，页面头部状态立即更新。

## 9. 验收标准

| 场景 | 验收标准 |
| :--- | :--- |
| 待支付取消 | `pending_payment` 订单可取消，未产生资金时直接进入 `cancelled`。 |
| 已预订取消 | `reserved` 订单可取消，系统展示取消费、应退款和预授权处理结果。 |
| 逾期未取取消 | `pickup_overdue` 订单可取消，按有责取消试算费用。 |
| 逾期未取继续取车 | `pickup_overdue` 订单仍可进入取车流程。 |
| 标记爽约 | `pickup_overdue` 订单达到保留时长后，可由系统自动处理或由门店手动标记爽约，处理后进入 `no_show`。 |
| 爽约费用 | 爽约后按订单规则快照生成爽约费，并按规则扣取或释放预授权。 |
| 取车拒绝 | `inspecting` 订单可拒绝履约，确认后进入 `rejected`。 |
| 关闭订单 | 可关闭状态执行关闭后进入 `closed`，全额退款并释放预授权。 |
| 企业月结终止 | 企业月结订单终止时不展示客户支付、退款和预授权入口；客户有责费用生成企业月结挂账，客户无责关闭不生成企业挂账。 |
| 终态展示 | `cancelled`、`no_show`、`rejected`、`closed` 订单列表只展示“详情”。 |
| 操作日志 | 取消、爽约、拒绝履约、关闭订单均写入操作记录和费用审计日志。 |
