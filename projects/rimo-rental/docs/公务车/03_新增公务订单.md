## 1. 页面范围

原型页面：`02_原型和PRD/原型/企业端/official_order_add.html`

页面用途：企业管理员代员工建立公务车预约单。

## 2. 页面口径

1. 页面使用“员工”“用车员工”“公务订单”口径。
2. 页面不使用“会员”“会员方案”等个人租赁口径。
3. 页面不展示订单摘要、支付金额、押金、预授权、退款、个人优惠。
4. 页面内容区左上角不单独展示大标题，以表单区块承载信息。

## 3. 表单结构

### 3.1 员工资料

| 字段 / 操作 | 必填 | 说明 |
| --- | --- | --- |
| 用车员工 | 是 | 点击“选择员工”弹窗选择 |
| 员工基础信息 | 系统带入 | 选择员工后带入姓名、手机号、所属分公司、部门 |

员工资料区应保持紧凑，不展示多张信息卡。

### 3.2 用车资料

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 同行人数 | 否 | 默认 1，可手动调整 |
| 目的地 | 是 | 填写本次公务用车目的地 |
| 用车事由 | 是 | 填写本次用车原因 |
| 取车据点 | 是 | 按员工所属分公司可用据点筛选 |
| 还车据点 | 是 | 默认自动同取车据点，不允许手动修改 |
| 预定车款 | 是 | 按据点可用车辆车款筛选 |
| 车牌号 | 是 | 按据点、车款、车辆状态和时间冲突筛选 |
| 预定日期（起） | 是 | 预约取车时间 |
| 预定日期（止） | 是 | 预约还车时间 |

字段顺序：同行人数、目的地应位于用车事由上方。

## 4. 选择员工弹窗

弹窗用于从企业员工中选择用车员工。

### 4.1 筛选条件

1. 分公司
2. 部门
3. 关键字搜索：员工姓名、手机号、身份证号

### 4.2 列表字段

| 字段 | 说明 |
| --- | --- |
| 员工姓名 | 员工姓名 |
| 手机号 | 员工手机号 |
| 身份证号 | 员工证件号 |
| 分公司 | 员工所属分公司 |
| 部门 | 员工所属部门 |
| 状态 | 员工启用状态 |
| 操作 | 选择 |

### 4.3 选择规则

1. 只能选择已审核通过且未停用的员工。
2. 选择员工后，系统带入员工资料。
3. 更换员工后，系统清空取车据点、预定车款、车牌号、目的地、用车事由和预约时间，避免沿用旧员工的数据范围。

## 5. 车辆筛选规则

系统选择车辆时必须同时满足：

1. 车辆启用且上架。
2. 车辆未被管控。
3. 车辆所在据点属于员工所属分公司可用据点。
4. 车辆在预约时间段内没有排程冲突。
5. 车辆符合用车管理中的员工 / 部门使用范围限制。
6. 车辆如需合约配额，必须存在有效配额。

## 6. 提交规则

1. 必填字段缺失时，不允许提交。
2. 预约还车时间必须晚于预约取车时间。
3. 提交时系统重新校验车辆是否仍可预约。
4. 员工所属分公司未开启预约审批时，订单创建后直接进入已预约。
5. 员工所属分公司开启预约审批时，订单创建后进入待审批。
6. 订单创建后保存员工、车辆、据点、预约时间、目的地、用车事由和审批规则快照。

## 7. 验收标准

1. 管理员可以通过分公司、部门和关键字筛选员工。
2. 选择员工后，可用据点和车辆范围按员工所属分公司重新计算。
3. 页面不出现会员、会员方案、支付、预授权、退款字段。
4. 开启审批的分公司提交后进入待审批。
5. 未开启审批的分公司提交后直接创建有效预约。
