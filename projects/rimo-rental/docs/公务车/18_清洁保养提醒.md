## 1. 功能范围

企业端新增“清洁保养提醒”页面，用于配置公务车清洁、保养提醒规则，并按车辆基线资料持续生成提醒。

页面文件：`02_原型和PRD/原型/企业端/official_vehicle_reminders.html`

本期只做提醒闭环：

- 不生成工单
- 不记录维修或清洁费用
- 不自动下架车辆
- 不自动将车辆切换为被管控中
- 不阻断员工预约
- 不替代营运端平台级运维规则

## 2. 角色权限

### 2.1 企业总管理员

- 查看企业全部公务车提醒
- 新增、编辑、启用、停用企业通用保养母版和清洁母版
- 查看、编辑、启用、停用任一分公司的专用母版
- 控制“是否启用分公司专用”开关
- 维护外部 Email 通知设定
- 查看外部 Email 发送记录
- 为车辆套用或改绑母版
- 登记已保养、登记已清洁
- 忽略提醒

### 2.2 分公司管理员

- 只查看本分公司范围内的公务车提醒
- 只为资产归属为本分公司的车辆套用或改绑母版
- 只登记资产归属为本分公司的车辆已保养、已清洁
- 只忽略本分公司提醒
- “是否启用分公司专用”开启时，可维护本分公司专用母版
- “是否启用分公司专用”关闭时，不显示新增母版按钮，只可查看母版表；车辆套用仍通过右上角“设定”弹窗中的“车辆套用”页签进入
- 不可操作企业通用母版、其他分公司专用母版和外部 Email 通知设定

### 2.3 员工

员工不进入本页面。

## 3. 页面结构

页面包含：

1. 顶部统计卡
2. 筛选区
3. 提醒列表
4. 通知设定按钮
5. 设定按钮
6. 通知设定弹窗：包含“通知人设定”和“发送记录”两个页签
7. 清洁保养提醒设定弹窗

页面不提供“刷新判断”按钮。提醒判断依赖定时任务、事件重算和页面重新载入后的最新数据。

## 4. 顶部统计卡

统计卡展示：

- 待保养车辆数：当前存在待处理保养提醒的去重车辆数
- 待清洁车辆数：当前存在待处理清洁提醒的去重车辆数
- 今日新增待处理提醒数：今天生成且当前仍为待处理状态的提醒数
- 未配置母版 / 资料待补车辆数

已完成、已忽略、已失效的提醒不计入顶部统计卡。

点击规则：

- 点击待保养车辆数：筛选保养提醒 + 待处理
- 点击待清洁车辆数：筛选清洁提醒 + 待处理
- 点击今日新增待处理提醒数：筛选当天新增的待处理提醒
- 点击未配置母版 / 资料待补车辆数：打开右上角“设定”弹窗，进入“车辆套用”页签，只展示保养或清洁任一资料待补的车辆
- 统计卡被选中后，再次点击同一张统计卡，取消选中并清空该卡片带入的筛选条件

## 5. 提醒列表

列表字段：

- 车牌 / 车型
- 分公司 / 据点
- 提醒类型
- 当前套用母版
- 当前值
- 触发时间
- 状态
- 操作

当前值展示规则：

- 保养提醒显示两行：
  - 当前里程 + 已行驶里程，例如：`当前 25,700 km｜已行驶 5,150 km`
  - 上次保养基线，例如：`上次保养：20,550 km / 2026-01-12`
- 清洁提醒显示两行：
  - 已用车次数 + 距上次清洁天数，例如：`已用车 8 次｜距上次清洁 17 天`
  - 上次清洁基线，例如：`上次清洁：2026-05-08`

当前套用母版展示规则：

- 第一行显示母版范围和母版名称，例如：`企业通用｜新车`
- 第二行只显示到期阈值，例如：`5,000 km / 6 个月`、`10 次 / 15 天`
- 列表中不显示提前提醒公里数、提前提醒次数或提前提醒天数

状态展示：

- `pending + dueLevel=upcoming`：即将到期
- `pending + dueLevel=overdue`：已到期
- `ignored`：已忽略
- `completed`：已完成
- `invalidated`：已失效

操作规则：

| 状态 | 操作 |
| --- | --- |
| 即将到期 / 已到期 | 登记已保养或登记已清洁、忽略本次、查看车辆 |
| 已忽略 | 查看车辆 |
| 已完成 | 查看登记内容、查看车辆 |
| 已失效 | 查看车辆 |

忽略只表示本次提醒不再作为待处理提醒触达，不代表车辆已经完成保养或清洁。已忽略记录仅保留历史，不允许登记完成；车辆完成保养或清洁后，需要由新的待处理提醒执行登记完成。

已失效表示提醒对应的车辆母版或资产归属条件已经发生变化，原提醒不再作为当前待处理提醒。已失效记录仅保留历史，不允许登记完成或忽略。

“查看车辆”用于跳转公务车辆管理页面查看对应车辆资料。提醒列表只处理提醒记录本身，不在单条提醒记录中提供车辆提醒设定入口。车辆提醒母版和基线资料通过右上角“设定”弹窗中的“车辆套用”页签维护。

已完成提醒保留登记完成快照。用户点击“查看登记内容”后，以只读弹窗展示登记时保存的内容，不允许在该弹窗中修改。展示字段包括：

- 提醒类型
- 车辆
- 登记完成日期
- 登记完成里程，仅保养提醒展示
- 登记备注
- 登记人
- 登记时间
- 登记后基线说明

## 6. 母版规则

### 6.1 保养母版

字段：

- 母版名称
- 适用范围：企业通用 / 分公司专用
- 归属分公司：分公司专用时记录
- 状态：启用 / 停用
- 保养里程阈值 `thresholdMileageKm`
- 提前提醒公里数 `advanceMileageKm`
- 保养月份阈值 `thresholdMonths`
- 提前提醒天数 `advanceDays`
- 备注

规则：

- 企业通用母版名称在当前企业内唯一
- 分公司专用母版名称在当前分公司内唯一
- 分公司专用母版只归属 1 个分公司
- 新增母版弹窗不展示“适用范围”“归属分公司”字段
- 企业总管理员新增时默认创建企业通用母版
- 分公司管理员新增时默认创建当前分公司的分公司专用母版
- 停用母版后，不允许新绑定车辆；已绑定车辆保留当前绑定并继续计算
- 本期不做企业通用母版与分公司专用母版之间的继承、联动和自动同步
- 提前提醒公里数必须小于保养里程阈值

### 6.2 清洁母版

字段：

- 母版名称
- 适用范围：企业通用 / 分公司专用
- 归属分公司：分公司专用时记录
- 状态：启用 / 停用
- 清洁用车次数阈值 `thresholdUseCount`
- 提前提醒次数 `advanceUseCount`
- 清洁天数阈值 `thresholdDays`
- 提前提醒天数 `advanceDays`
- 备注

规则：

- 企业通用母版名称在当前企业内唯一
- 分公司专用母版名称在当前分公司内唯一
- 分公司专用母版只归属 1 个分公司
- 停用母版后，不允许新绑定车辆；已绑定车辆保留当前绑定并继续计算
- 提前提醒次数必须小于清洁用车次数阈值
- 提前提醒天数必须小于清洁天数阈值

### 6.3 母版删除规则

- 企业总管理员可删除企业通用母版和任一分公司的专用母版
- 分公司管理员只可删除本分公司创建的分公司专用母版
- 分公司管理员不可删除企业通用母版，也不可删除其他分公司的专用母版
- “是否启用分公司专用”关闭时，分公司管理员不显示新增、编辑、删除分公司专用母版入口
- 删除前必须校验已套用车辆数
- 已被车辆套用的母版不允许删除，需要先在车辆套用页签或新增 / 编辑车辆页面改绑或清空对应车辆母版
- 删除母版不影响历史已完成、已忽略、已失效提醒记录；历史记录继续保留当时母版名称快照
- 删除成功后，该母版不再出现在新增 / 编辑车辆和车辆套用可选母版列表中

## 7. 车辆套用规则

每台公务车分别绑定：

- 1 个保养母版
- 1 个清洁母版

车辆可选母版范围：

- 企业通用母版
- 当前车辆资产归属分公司的专用母版

车辆不可绑定其他分公司的专用母版。

车辆绑定入口：

- 新增 / 编辑公务车辆页面的“清洁保养”资料区，可以直接选择保养母版和清洁母版
- 清洁保养提醒页面右上角“设定”弹窗的“车辆套用”页签，可以批量查看车辆并逐车调整母版和基线资料
- 两个入口使用同一组车辆字段，保存后互相同步展示

车辆套用页签操作要求：

- 设定弹窗需要使用大窗口展示，避免车辆字段过多导致横向操作困难
- 车辆套用页签保留分页，默认每页 10 台，支持切换每页 20 台、50 台
- 车辆套用页签支持按车牌 / 车型 / 据点关键字搜索
- 车辆套用页签支持按资产归属分公司筛选
- 车辆套用页签支持按资料状态筛选：资料待补、保养待补、清洁待补、已可计算
- 顶部统计卡“未配置母版 / 资料待补”进入车辆套用页签时，默认带入资料待补筛选

批量套用母版：

- 车辆套用页签支持批量套用母版；批量只更新保养母版和清洁母版，不批量更新上次保养日期、上次保养里程、上次清洁日期等车辆基线资料
- 企业总管理员批量套用时只能选择企业通用母版；分公司管理员批量套用时只能操作本分公司车辆，并可选择企业通用母版或本分公司专用母版
- 批量保存时后端逐车校验车辆操作权限和母版可选范围，任一车辆不符合规则时整批不保存并提示原因
- 批量保存成功后，系统重新计算所选车辆的提醒

车辆套用弹窗中的基线字段与公务车辆资料共用同一组车辆字段，不另存副本：

| 车辆套用弹窗字段 | 对应新增 / 编辑车辆字段 | 规则 |
| --- | --- | --- |
| 保养母版 | 保养母版 | 双向同步。新增或编辑车辆时选择后，车辆套用弹窗展示同一绑定；在车辆套用弹窗调整后，回写车辆资料。 |
| 清洁母版 | 清洁母版 | 双向同步。新增或编辑车辆时选择后，车辆套用弹窗展示同一绑定；在车辆套用弹窗调整后，回写车辆资料。 |
| 上次保养日期 | 上次保养时间 | 双向同步。新增或编辑车辆时填写后，车辆套用弹窗展示同一值；在车辆套用弹窗调整后，回写车辆资料。 |
| 上次保养里程 | 上次保养里程 | 双向同步。新增或编辑车辆时填写后，车辆套用弹窗展示同一值；在车辆套用弹窗调整后，回写车辆资料。 |
| 上次清洁日期 | 上次清洁时间 | 双向同步。新增或编辑车辆时填写后，车辆套用弹窗展示同一值；在车辆套用弹窗调整后，回写车辆资料。 |

新增 / 编辑车辆页面选择母版时，也必须按车辆资产归属分公司过滤可选范围，只允许选择企业通用母版或当前车辆资产归属分公司的专用母版。车辆资产归属分公司变更后，如已选母版不再符合可选范围，系统清空对应母版并要求用户重新选择。

车辆套用弹窗不提供“清洁后用车次数”输入列。清洁后用车次数由系统根据上次清洁日期后的已完成公务车预约自动累计，用户不能手动填写或修改。

清洁后用车次数初始化和重置规则：

- 新增车辆时填写上次清洁时间后，清洁后用车次数从 0 次开始计算
- 车辆套用弹窗首次补填或修改上次清洁日期后，清洁后用车次数重置为 0 次，并从新的上次清洁日期开始重新累计
- 登记已清洁后，清洁后用车次数重置为 0 次，并从登记清洁日期开始重新累计
- 公务车预约完成后，如订单完成时间晚于车辆当前上次清洁日期，系统将该车辆清洁后用车次数 +1

车辆资产归属分公司变更时：

- 企业通用母版保留
- 旧分公司专用保养母版自动清空
- 旧分公司专用清洁母版自动清空
- 被清空绑定的车辆进入“资料待补”
- 原资产归属分公司下，基于旧分公司专用母版生成且尚未完成的提醒记录保留历史，但状态改为已失效
- 已失效提醒不计入待处理统计，不进入右上角待处理提醒，不触发新的外部 Email

资料待补按提醒类型独立判断：

| 类型 | 待补范围 | 影响 |
| --- | --- | --- |
| 保养待补 | 未绑定保养母版；保养母版缺少必要阈值；上次保养日期缺失，且当前里程或上次保养里程缺失，导致保养时间维度和保养里程维度都无法计算 | 不生成保养提醒，不影响清洁提醒 |
| 清洁待补 | 未绑定清洁母版；清洁母版缺少必要阈值；缺少上次清洁日期 | 不生成清洁提醒，不影响保养提醒 |

未配置母版 / 资料待补车辆数统计保养或清洁任一类型待补的车辆。车辆套用列表需要分别展示保养可计算 / 保养待补、清洁可计算 / 清洁待补。

保养和清洁分开判断。只补齐保养母版和保养基线时，系统可正常计算保养提醒；清洁资料未补齐只影响清洁提醒。只补齐清洁母版和上次清洁日期时，系统可正常计算清洁提醒；保养资料未补齐只影响保养提醒。

## 8. 提醒计算

### 8.1 保养提醒

计算字段：

- 当前里程 `currentMileage`
- 上次保养里程 `lastMaintenanceMileage`
- 上次保养日期 `lastMaintenanceAt`
- 保养里程阈值 `thresholdMileageKm`
- 提前提醒公里数 `advanceMileageKm`
- 保养月份阈值 `thresholdMonths`
- 提前提醒天数 `advanceDays`

计算方式：

```text
mileageDelta = currentMileage - lastMaintenanceMileage
dateDue = lastMaintenanceAt + thresholdMonths
mileageReminderPoint = thresholdMileageKm - advanceMileageKm
dateReminderPoint = dateDue - advanceDays
```

触发规则：

- `mileageDelta >= mileageReminderPoint` 且未达到最终阈值：生成或更新为“即将到期”
- `currentDate >= dateReminderPoint` 且未达到最终阈值：生成或更新为“即将到期”
- `mileageDelta >= thresholdMileageKm`：生成或更新为“已到期”
- `currentDate >= dateDue`：生成或更新为“已到期”

保养里程维度和保养时间维度按可用资料分别计算：

- 当前里程、上次保养里程、保养里程阈值、提前提醒公里数齐全时，计算里程维度
- 上次保养日期、保养月份阈值、提前提醒天数齐全时，计算时间维度
- 两个维度满足任一条件即可生成或更新保养提醒
- 某一维度资料缺失时只跳过该维度，不阻断另一维度计算
- 两个维度都无法计算时，该车辆标记为保养待补，不生成保养提醒

### 8.2 清洁提醒

计算字段：

- 上次清洁后已完成用车次数 `completedUseCountSinceCleaning`，由系统累计，不在车辆套用弹窗手动输入
- 上次清洁日期 `lastCleaningAt`
- 清洁次数阈值 `thresholdUseCount`
- 提前提醒次数 `advanceUseCount`
- 清洁天数阈值 `thresholdDays`
- 提前提醒天数 `advanceDays`

计算方式：

```text
cleaningDateDue = lastCleaningAt + thresholdDays
useCountReminderPoint = thresholdUseCount - advanceUseCount
cleaningDateReminderPoint = cleaningDateDue - advanceDays
```

触发规则：

- `completedUseCountSinceCleaning >= useCountReminderPoint` 且未达到最终阈值：生成或更新为“即将到期”
- `currentDate >= cleaningDateReminderPoint` 且未达到最终阈值：生成或更新为“即将到期”
- `completedUseCountSinceCleaning >= thresholdUseCount`：生成或更新为“已到期”
- `currentDate >= cleaningDateDue`：生成或更新为“已到期”

清洁提醒缺少清洁母版或上次清洁日期时，标记为清洁待补，不生成清洁提醒，但不影响保养提醒。

## 9. 提醒周期与去重

提醒周期以车辆基线是否更新为准：

- 保养基线：上次保养日期 + 上次保养里程
- 清洁基线：上次清洁日期 + 清洁后累计用车次数

基线未更新前：

- 同一车辆、同一提醒类型只保留 1 条待处理提醒
- 已有待处理提醒时，只更新原提醒当前值，不新增第二条
- 即使车辆继续行驶、继续用车或继续经过时间，并再次跨过同一母版阈值倍数，也不新增同类型提醒

示例：上次保养里程为 1,000 km，母版为每 5,000 km 保养一次。车辆到 6,000 km 时生成保养提醒，用户未登记已保养；车辆继续到 11,000 km 时，不生成第二条保养提醒，只更新原提醒当前值。

登记已保养或登记已清洁后，系统回写新基线，才进入下一轮提醒周期。

## 10. 登记与忽略

### 10.1 登记已保养

适用记录：

- 待处理保养提醒

操作字段：

- 完成日期
- 完成里程
- 备注

字段规则：

- 完成日期必填，默认带入当前日期，允许用户修改
- 完成里程必填，默认带入车辆当前里程，允许用户修改
- 完成里程不得小于 0
- 如车辆已有上次保养里程，完成里程不得小于原上次保养里程
- 如车辆当前里程存在，完成里程原则上不应大于车辆当前里程；若业务允许补录高于当前里程的保养记录，提交时需要提示确认
- 备注选填，用于记录保养厂商、保养项目或内部说明

处理结果：

- 提醒状态改为已完成
- 回写车辆资料中的上次保养日期，对应新增 / 编辑车辆页面的“上次保养时间”
- 回写车辆资料中的上次保养里程，对应新增 / 编辑车辆页面的“上次保养里程”
- 保存登记完成快照，用于已完成记录查看登记内容
- 当前保养提醒从待处理提醒中移除，不再出现在待保养车辆统计中
- 右上角站内提醒同步改为已处理
- 写入操作记录，记录提醒编号、车辆、原基线、新基线、操作人、操作时间和备注
- 系统立即重算该车辆保养提醒；若新基线仍满足提醒条件，则允许生成下一轮提醒
- 进入下一轮保养提醒周期

### 10.2 登记已清洁

适用记录：

- 待处理清洁提醒

操作字段：

- 完成日期
- 备注

字段规则：

- 完成日期必填，默认带入当前日期，允许用户修改
- 备注选填，用于记录清洁地点、清洁项目或内部说明

处理结果：

- 提醒状态改为已完成
- 回写车辆资料中的上次清洁日期，对应新增 / 编辑车辆页面的“上次清洁时间”
- 清洁后累计用车次数重置为 0
- 保存登记完成快照，用于已完成记录查看登记内容
- 当前清洁提醒从待处理提醒中移除，不再出现在待清洁车辆统计中
- 右上角站内提醒同步改为已处理
- 写入操作记录，记录提醒编号、车辆、原上次清洁日期、新上次清洁日期、操作人、操作时间和备注
- 系统立即重算该车辆清洁提醒；若新基线仍满足提醒条件，则允许生成下一轮提醒
- 进入下一轮清洁提醒周期

### 10.3 忽略本次

适用记录：

- 待处理保养提醒
- 待处理清洁提醒

操作字段：

- 备注，选填

处理结果：

- 提醒状态改为已忽略
- 记录操作人、操作时间和备注
- 不改写保养或清洁基线
- 不回写车辆资料中的上次保养时间、上次保养里程或上次清洁时间
- 不重置清洁后累计用车次数
- 不代表车辆已经完成保养或清洁
- 已忽略记录仅保留历史，只允许查看车辆，不允许登记完成
- 当前提醒从待处理提醒中移除，不再计入待保养车辆数、待清洁车辆数和今日新增待处理提醒数
- 右上角站内提醒同步改为已处理
- 写入操作记录，记录提醒编号、车辆、提醒类型、提醒程度、操作人、操作时间和备注

忽略后的再提醒规则：

- 忽略“即将到期”后，同一周期达到最终阈值时，重新生成 1 条“已到期”提醒
- 忽略“已到期”后，基线未变化前不重复生成同一周期到期提醒
- 已忽略提醒不阻断后续车辆预约、不改变车辆状态、不生成工单

## 11. 通知设定

### 11.1 站内提醒

系统生成待处理提醒后，同步进入企业端右上角提醒。

规则：

- 企业总管理员收到企业全部提醒
- 分公司管理员只收到资产归属为本分公司的车辆提醒
- 已读只表示账号查看过提醒，不代表业务已处理
- 登记完成或忽略后，右上角提醒同步改为已处理

站内提醒内容：

- 保养标题：`【保养提醒】{车牌号} {即将到期/已到期}`
- 清洁标题：`【清洁提醒】{车牌号} {即将到期/已到期}`
- 内容展示车辆、据点、母版名称、当前值和规则阈值

### 11.2 外部 Email 通知

“通知设定”按钮仅企业总管理员可见。

通知设定弹窗包含两个页签：

- 通知人设定：维护外部通知接收人、通知范围和车辆范围
- 发送记录：查看外部 Email 通知的发送日志

字段：

- 通知人
- Email
- 通知范围：保养提醒 / 清洁提醒，可多选
- 车辆范围：全部车辆 / 按条件匹配，默认全部车辆
- 状态：启用 / 停用

规则：

- 通知人不要求是系统用户
- 通知人姓名、Email 手动填写
- Email 必须校验格式
- 通知范围至少选择 1 项
- 车辆范围选择“全部车辆”时，匹配当前企业全部公务车提醒
- 车辆范围选择“按条件匹配”时，必须先选择 1 个匹配条件类型，并至少选择 1 个条件值
- 匹配条件类型包括车辆来源、资产归属分公司、所属据点、绑定合约
- 每条通知设定只能选择 1 个匹配条件类型；该条件类型下的具体值可多选，多选时按“任一命中”判断
- 后续新增车辆只要满足该条件，自动纳入该通知人的通知范围，不需要重新设置通知人
- 外部 Email 只做提醒触达，不授予系统权限
- 停用后不再发送新邮件

发送规则：

- 首次生成待处理提醒时，按提醒类型、车辆范围匹配启用通知设定并发送
- 同一提醒从“即将到期”变为“已到期”时，可再次发送 1 次
- 同一提醒、同一提醒程度、同一 Email 不重复发送
- 邮件发送失败不影响提醒生成和页面处理状态，需要记录失败原因

### 11.3 Email 发送记录

Email 发送记录用于让企业总管理员查看外部通知是否成功发出，不作为提醒处理状态，不允许手动新增、编辑或删除。

发送记录主表按“车辆 + 通知类型 + 提醒阶段 + 触发发送时间”聚合展示。也就是同一台车的一次保养或清洁提醒，即使发送给多名接收人，主表也只展示 1 条记录；具体发给哪些人，在该记录的发送明细中查看。

发送记录主表字段：

| 字段 | 说明 |
| --- | --- |
| 发送时间 | 系统实际发起 Email 发送的时间 |
| 通知类型 | 保养提醒 / 清洁提醒 |
| 提醒阶段 | 即将到期 / 已到期 |
| 车辆 | 车牌号 + 车型 |
| 收件人数 | 本次通知命中的接收人数 |
| 成功数 | 本次通知发送成功的接收人数 |
| 失败数 | 本次通知发送失败的接收人数 |
| 发送状态 | 成功 / 部分失败 / 失败 |
| 关联提醒状态 | 当前关联提醒的状态：待处理 / 已忽略 / 已完成 / 已失效 |

发送明细字段：

| 字段 | 说明 |
| --- | --- |
| 接收人 | 通知设定中的通知人姓名 |
| Email | 通知设定中的接收 Email |
| 发送状态 | 成功 / 失败 |
| 失败原因 | 发送失败时记录失败原因；发送成功时为空 |

筛选条件：

- 通知类型
- 发送状态
- 接收人 / Email / 车牌号关键字
- 发送时间区间

记录规则：

- 每次系统针对一台车的一次提醒触发外部 Email 通知时，生成 1 条发送记录主表。
- 本次通知命中的每个接收人生成 1 条发送明细。
- 所有接收人发送成功时，主表发送状态为“成功”。
- 所有接收人发送失败时，主表发送状态为“失败”。
- 部分接收人发送成功、部分接收人发送失败时，主表发送状态为“部分失败”。
- 发送记录不影响提醒状态；提醒仍按待处理、已忽略、已完成、已失效等状态流转。
- 后续提醒状态发生变化时，发送记录中的关联提醒状态展示当前最新状态。

## 12. 重算时机

系统每日执行一次提醒刷新任务。

以下事件发生后，需要重算对应车辆提醒：

- 车辆当前里程更新
- 公务车预约完成
- 车辆保存或改绑提醒母版
- 新增 / 编辑车辆时保存保养母版、清洁母版或清洁保养基线资料
- 车辆资产归属分公司变更
- 登记已保养
- 登记已清洁

重算只生成新提醒或更新现有提醒，不生成工单，不改变车辆状态。

## 13. 验收标准

1. 企业总管理员可维护企业通用母版、分公司专用母版、通知设定和车辆套用。
2. 分公司管理员只可维护本分公司专用母版；分公司专用关闭时不显示新增母版按钮。车辆套用通过右上角“设定”弹窗中的“车辆套用”页签进入。
3. 每台车可分别绑定 1 个保养母版和 1 个清洁母版。
4. 车辆只能绑定企业通用母版或当前资产归属分公司的专用母版。
5. 车辆资产归属分公司变更后，旧分公司专用母版自动清空并标记资料待补。
6. 已被车辆套用的保养母版或清洁母版不允许删除；未被车辆套用的母版可按权限删除，删除后不再出现在可选母版列表。
7. 保养提醒按里程或时间任一维度达到提前提醒点生成“即将到期”，达到最终阈值生成或更新为“已到期”。
8. 清洁提醒按用车次数或时间任一维度达到提前提醒点生成“即将到期”，达到最终阈值生成或更新为“已到期”。
9. 基线未更新前，同一车辆、同一提醒类型不重复生成多条待处理提醒，只更新原待处理提醒当前值；即将到期进入已到期时更新原记录。
10. 待处理提醒可登记完成、忽略或查看车辆。
11. 已忽略提醒仅保留历史，只可查看车辆，不允许登记完成。
12. 已完成提醒不再展示登记和忽略操作，但可查看登记完成时保存的登记内容。
13. 忽略即将到期提醒后，原记录保留为已忽略；达到最终阈值时仍生成 1 条新的已到期待处理提醒。
14. 忽略已到期提醒后，基线未变化前不重复生成同一周期提醒。
15. 资料待补只阻断对应类型提醒。保养资料待补不影响清洁提醒计算，清洁资料待补不影响保养提醒计算。
16. 顶部统计卡可点击并按对应口径筛选；已完成、已忽略、已失效的提醒不计入顶部统计卡。
17. 页面不显示“刷新判断”按钮。
18. 外部 Email 通知只对企业总管理员开放配置，发送失败不影响页面提醒状态。
19. 新增 / 编辑车辆页面可直接选择保养母版和清洁母版，可选范围与车辆套用弹窗一致。
20. 车辆套用弹窗中的保养母版、清洁母版、上次保养日期、上次保养里程、上次清洁日期与新增 / 编辑车辆页面的同名业务字段双向同步。
21. 车辆套用页签支持关键字、分公司和资料状态筛选，并保留分页。
22. 车辆套用页签支持批量套用母版；批量只更新保养母版和清洁母版，不批量更新车辆基线资料。
23. 企业总管理员批量套用时只能选择企业通用母版；分公司管理员批量套用时只能操作本分公司车辆，并可选择企业通用母版或本分公司专用母版。
24. 车辆套用弹窗不展示“清洁后用车次数”输入列；清洁后用车次数由系统从 0 开始自动累计。
25. 只配置保养母版和可用保养基线时，系统允许生成保养提醒；只配置清洁母版和上次清洁日期时，系统允许生成清洁提醒。
26. 修改上次清洁日期或登记已清洁后，清洁后用车次数重置为 0，并从新的上次清洁日期开始重新累计。
27. 通知设定弹窗包含发送记录页签，企业总管理员可按通知类型、发送状态、关键字和发送时间区间查询 Email 发送记录。
28. 发送记录主表按“车辆 + 通知类型 + 提醒阶段 + 触发发送时间”聚合展示；同一次提醒通知发给多名接收人时，主表只显示 1 条记录，接收人发送结果在发送明细中查看。
29. 发送状态按明细汇总：全部成功为“成功”，全部失败为“失败”，部分成功部分失败为“部分失败”；发送失败记录失败原因，但不改变提醒处理状态。
30. 车辆资产归属分公司变更导致原分公司专用母版清空时，原分公司未完成提醒记录保留为已失效，已失效记录只可查看车辆，不允许登记完成或忽略。
