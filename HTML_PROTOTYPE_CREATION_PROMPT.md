## 提示词

```
请根据 PRD 创建或修改一个可独立预览、后续可直接迁移到 Vue 3 工程的 HTML 原型。

1. 先读取 PRD 和指定的旧 HTML。旧 HTML 只用于对齐布局、间距、字体、颜色、按钮、卡片、表格、表单和弹窗效果，不得直接复制其旧外壳、全局样式或不规范代码。
2. 完整实现页面字段、模拟数据、筛选、分页、状态、校验、按钮、弹窗和抽屉，不得用占位内容代替，也不得为了省事删除交互。
3. 页面结构统一为：
   - 独立预览用侧栏和顶栏标记 `data-prototype-shell`；
   - 业务内容只放在一个 `data-page-content` 节点内；
   - 弹窗和抽屉放在内容区之后并标记 `data-page-overlay`。
4. 使用 Vue 3 Composition API 风格组织状态和方法。禁止内联 onclick、直接操作 DOM、`window.location` 和跳转 `.html`；Element Plus 标签必须显式闭合。
5. 页面样式必须放在唯一页面根类名下，不污染全局；沿用工程主题变量和现有字体，不新增 CDN、外部字体、外部图片或项目专用技术命名。
6. 原型独立运行所需的依赖引入和 `createApp/mount` 代码集中放在 `PROTOTYPE_BOOTSTRAP_START/END` 标记内，业务逻辑不得写在启动代码中。
7. 页面完成后对照参考 HTML 检查视觉，并确认按钮、弹窗、抽屉、分页和校验都能实际操作，浏览器控制台无错误。
8. 交付时只说明：修改文件、参考页面、已实现交互和仍待确认事项。

不要重新设计业务，不要修改已确认口径。视觉保持与参考页面一致，代码按以上规范重新实现。
```

## 使用流程

需求阶段只修改 HTML；页面定案后停止修改 HTML，并单向迁移到 Vue；迁移验收后只维护 Vue，HTML 作为历史参考。
