## 1. 文档定位

本文用于从全局角度说明门市租赁业务的范围、边界、核心对象、主要功能模块和对应 PRD 文件入口。

阅读完本文后，业务人员和技术人员应能快速理解：

1. 门市租赁包含哪些业务模式。
2. 订单、车辆、门市/据点、价格、优惠、规则、企业资格之间如何关联。
3. 目前本期已经覆盖到哪些功能。
4. 每块能力对应应该阅读哪份 PRD。

### 1.1 文档维护规则

1. 门市租赁下所有 PRD 文档必须保留文档级“版本记录”区块。
2. 后续每次修改 PRD 时，必须在对应文档版本记录中新增一行，记录版本号、日期和本次调整概括。
3. 文档级版本记录用于说明 PRD 内容变更；标准价、行销方案、业务规则等业务对象自身的版本记录仍按各专项 PRD 规则执行，不与文档版更混用。

本文不展开细节规则，细节以对应 PRD 正文为准。

## 2. 业务范围

门市租赁当前包含两种模式：

1. **门市租车**
   - 有店员现场服务。
   - 客户预约和履约时面向门市。
   - 具体据点用于内部备车、停放、整备和归还登记。

2. **24 小时自助租车**
   - 客户面向具体自助据点。
   - 需要依赖据点能力、车机能力和自助取还车指引。
   - 与门市租车共用车组、标准价、动态调价、行销方案、增值服务和业务规则。

门市租赁整体属于公开租赁业务的一部分，与分时租赁、公务车并列，但在订单模型、履约方式、站点口径和调度方式上独立。

## 3. 核心业务边界

### 3.1 客户预订对象

门市租赁订单默认按**车组**预订，不在下单时锁定具体车牌。

订单创建成功后，系统先锁定车组库存；具体车辆由调度系统在取车前完成预分配和硬锁定。

### 3.2 门市与据点口径

1. 门市租车面向门市。
2. 24 小时自助租车面向具体据点。
3. 车辆始终挂在具体据点下。
4. 门市租车的异地还车费和调度费按门市判断，不按据点判断。

### 3.3 调度边界

1. 自动派车、人工改派、改还车门市/据点，本期只支持同车行范围。
2. 普通调度、归位、备车只改变车辆当前位置，不改变车辆资产所属车行。
3. 取车前必须存在已硬锁车辆。
4. AI 调度分析只输出风险和推荐，不得突破同车行、车组、车辆状态、时间窗、硬锁和人工保护规则。

### 3.4 资金与结算边界

1. 个人自付订单和企业非月结订单走客户支付、预授权、退款规则。
2. 企业月结订单不发起客户支付或预授权，最终费用进入企业账单。
3. 取车、还车、续租、取消、爽约和完成后追缴都要按对应资金口径处理。

### 3.5 快照与历史追溯边界

1. 标准价、动态调价、行销方案、增值服务、业务规则、调度规则都需要版本记录。
2. 订单保存订单级基线快照。
3. 续租、改租期、改车组、取车现场改行销方案、逾时重计费等商业变化，按事件级增量快照处理。
4. 不允许只通过主数据 ID 回查当前配置解释历史订单。

## 4. 核心对象

门市租赁涉及的核心对象如下：

1. **订单**
   - 承载客户约定的取还车门市/据点、车组、租期、价格、状态和履约过程。

2. **车组**
   - 作为门市租赁的基础销售与计费对象。

3. **车辆**
   - 承载实际履约能力、资产归属、当前位置、启用状态和营运状态。

4. **门市/据点**
   - 承载客户履约地点、车辆停放地点和服务能力配置。

5. **价格与优惠**
   - 包含车组标准价、动态调价、行销方案、优惠券、积分、增值服务。

6. **业务规则**
   - 包含支付、预授权、取还车、取消、爽约、超时、续租、调度锁定等规则。

7. **企业资格与结算方式**
   - 包含个人自付、企业非月结、企业月结三种资金口径。

8. **版本与快照**
   - 用于还原订单当时的合同承诺、计费依据和履约依据。

## 6. 当前本期已经明确的关键逻辑

以下关键口径已经明确，可作为开发实施边界：

1. 门市租赁订单按车组销售，不在下单时锁具体车辆。
2. 门市租车和 24 小时自助共用价格、行销、增值服务和业务规则。
3. 门市租车客户面向门市，24 小时自助客户面向具体据点。
4. 异地还车费和调度费按门市判断，不按据点判断。
5. 自动派车、人工改派、改还车门市/据点，本期只支持同车行范围。
6. 订单、派车、车辆启用状态、车辆营运状态是四条独立状态轴线。
7. 企业月结订单不走客户支付、客户预授权和客户退款。
8. 价格与规则快照统一采用“订单级基线快照 + 事件级增量快照”。
9. 门市或据点关闭/暂停后，本期不自动改站、不自动转单、不自动取消，统一提示联系客服人工处理。
10. 24 小时自助订单支持改还车据点；跨门市时仍按门市口径处理冲突校验和费用差额。
11. 车组库存按“时间窗 + 服务范围 + 车组”计算，每笔有效订单占用 1 个车组库存名额；预分配、硬锁、人工改派和免费升等只绑定具体车辆，不重复扣减车组库存。
12. 24 小时自助取车前，个人自付和企业非月结订单必须结清整笔订单应收金额后才允许开锁；24 小时自助还车时，必须先完成补缴、退款受理、账单平衡或企业月结挂账后才允许落锁。
13. 24 小时自助取还车必须闭合资金、验车记录、车机开锁/落锁和订单状态；车机失败时不重复收款、扣款、退款或月结挂账，通过自助履约状态进入重试或人工处理。
14. 智能调度同车组订单按调度占用时长优先排序；基准日均里程只用于分配后更新车辆模拟里程，不用于订单排序权重。
15. AI 调度分析采用租户自有模型 Key 接入，平台负责编排系统提示词、调度数据包、模型调用、结果解析、系统校验和审计日志。

## 7. 当前本期明确不做或不深化的范围

以下内容属于当前边界外，不作为本期闭环要求：

1. 跨车行自动派车
2. 跨车行人工改派

4. 跨车行转单
5. 跨车行还车
6. 门市或据点关闭后的自动改站、自动转单、自动取消
7. 增值服务跨门市归还导致的库存自动流转
8. 统一排程中心的全业务线占用联动
9. 审批流
10. 复杂自动化客服兜底

## 9. 当前复审结论

截至当前版本，门市租赁主链路已经形成完整闭环：

1. 下单
2. 车组库存锁定
3. 待支付 / 已预约
4. 预分配 / 硬锁定
5. 取车
6. 用车中
7. 还车
8. 待结算 / 待补缴 / 待退款 / 完成
9. 取消 / 爽约 / 关闭 / 事故 / 追缴
10. 价格、方案、规则、服务版本追溯

当前未发现新的阻断性逻辑漏洞。剩余未展开部分主要属于本期明确不做或后续规划范围，不属于当前主流程缺口。
