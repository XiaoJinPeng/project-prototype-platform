# 门市租赁客户与业务沟通提纲

## 1. 使用方式

本文用于对客户、业务人员、实施人员介绍门市租赁整体业务规划和设计。

讲解时以本文为主线，配合以下材料使用：

1. 主讲材料：`门市租赁业务方案概览.md`
2. 原型演示：营运端中的订单、站点、车辆、价格、行销、规则页面
3. 追问展开：对应详细 PRD

本文不用于研发交底。研发交底时，改用 `门市租赁业务总索引.md` 和详细 PRD。

---

## 2. 开场说明

### 2.1 开场目标

本次说明的目标，是让对方快速理解：

1. 门市租赁要解决什么业务场景
2. 这条业务线包含哪些模式
3. 用户和门店的完整流程是什么
4. 后台需要具备哪些支撑能力
5. 本期已经覆盖到什么范围
6. 本期暂不做什么

### 2.2 开场话术

门市租赁这条业务线，面向的是按天或按多天租车、按车组预约、在线下完成交付和验车的场景。  
当前整体规划包含两种模式：门市租车和 24 小时自助租车。  
两种模式共用价格、优惠、规则和调度底座，但 24 小时自助租车不是把门店流程简单搬到线上，而是一套面向具体据点和自助交付能力的履约模式。  
本次主要从业务定位、主流程、后台能力、关键规则和本期边界五个部分说明。

---

## 3. 第一部分：业务定位

### 3.1 先讲业务范围

讲解顺序：

1. 门市租赁是公开租赁业务的一部分
2. 当前包含两种模式
3. 两种模式共用同一套业务底座
4. 重点不是“系统页面”，而是“完整履约闭环”

### 3.2 对外说明口径

门市租赁当前包含：

1. **门市租车**
   - 客户在线预约，到门市由店员办理取车和还车
   - 适用于需要人工验车、资格核验、现场交付的场景

2. **24 小时自助租车**
   - 客户在线预约，到具体自助据点自行取还车
   - 适用于具备车机、自助交付和自助还车能力的场景

两种模式共用：

1. 车组
2. 标准价
3. 动态调价
4. 行销方案
5. 增值服务
6. 租赁业务规则
7. 订单计费逻辑
8. 调度与派车能力

### 3.3 这一段一定要单独讲清楚的区别

| 维度 | 门市租车 | 24 小时自助租车 |
| --- | --- | --- |
| 客户面向地点 | 门市 | 具体自助据点 |
| 取还车方式 | 店员现场交付 | 客户自助取还 |
| 资格核验 | 可现场人工核验 | 不依赖现场人工核验 |
| 优惠方案限制 | 可支持到店核验类方案 | 到店人工核验类方案本期不适用 |
| 车机要求 | 可无车机运行 | 必须具备车机能力 |
| 据点作用 | 主要承担备车、停放、整备、归还登记 | 直接作为客户履约地点 |

这一段要让对方明确感受到：24 小时自助租车不是“没有店员的门市租车”，而是站点口径和履约方式都不同的一种模式。

### 3.4 这一段的结束话术

所以门市租赁不是单一页面功能，而是一整套从预约、派车、取车、用车、还车到结算的业务链路。

## 4. 第二部分：角色与使用场景

### 4.1 先讲业务参与角色

讲解时按角色来讲，比较容易理解：

1. 客户
2. 门店店员
3. 营运后台人员
4. 企业客户

### 4.2 对外说明口径

不同角色在门市租赁中的职责如下：

1. **客户**
   - 负责预约、选择车组、选择取还车地点、选择优惠方案、支付或预授权

2. **门店店员**
   - 负责取车现场的身份核验、证件检查、资格核验、验车、交车
   - 负责还车现场的验车、费用确认和归还登记

3. **营运后台人员**
   - 负责订单管理、价格配置、行销方案、增值服务、业务规则、车辆调度、异常处理

4. **企业客户**
   - 负责企业资格和月结场景下的订单归属与结算口径

### 4.3 这一段的结束话术

从角色上看，门市租赁比纯线上租赁更重履约现场能力，因此后台设计也会比普通线上下单更完整。

---

## 5. 第三部分：主业务流程

### 5.1 先讲主链路

这一段固定按以下顺序讲：

`预约下单 -> 支付 / 预授权 -> 派车 -> 取车 -> 用车中 -> 还车 -> 结算 -> 完成`

### 5.2 对外说明口径

#### 1. 预约下单

客户选择：

1. 取车方式
2. 取还车门市或据点
3. 租期
4. 车组
5. 行销方案
6. 增值服务
7. 使用身份

系统根据规则计算预估费用，并按对应口径处理支付或预授权。

#### 2. 派车

门市租赁按车组销售，下单时不直接锁定具体车辆。  
订单建立后，系统先锁车组库存，后续由系统或人工完成具体车辆分派。

#### 3. 取车

门市租车在取车现场完成：

1. 身份确认
2. 证件检查
3. 行销方案资格核验
4. 验车
5. 收款或尾款处理
6. 发车

24 小时自助租车这一段要补一句：

- 客户不经过门店柜台，直接面向具体自助据点完成取车
- 依赖车机和自助交付能力
- 不走门店现场人工核验流程

#### 4. 用车中

客户开始使用车辆。期间可能发生：

1. 延长用车
2. 改还车门市或据点
3. 事故处理
4. 人工改派或换车

#### 5. 还车

还车时完成：

1. 还车验车
2. 费用重算
3. 补缴、退款或挂账
4. 车辆状态回写
5. 订单完成

这里也要单独补一句：

- 门市租车由门店办理还车和验车
- 24 小时自助租车由客户在具体自助据点完成还车

### 5.3 这一段的结束话术

所以门市租赁的核心，不只是把订单建出来，而是把后续的交车、用车、还车和结算动作完整闭环起来。

## 6. 第四部分：后台能力模块

### 6.1 讲解顺序

主流程讲完后，再讲后台能力支撑。

按以下顺序讲最清楚：

1. 订单管理
2. 服务站点管理
3. 车辆与调度
4. 价格体系
5. 行销与增值服务
6. 业务规则
7. 企业资格与月结

### 6.2 对外说明口径

#### 1. 订单管理

后台能够处理：

1. 订单列表
2. 订单详情
3. 取车
4. 还车
5. 续租
6. 改派
7. 费用管理
8. 操作记录、改派记录、事故记录

#### 2. 服务站点管理

后台统一管理：

1. 车行
2. 门市
3. 据点
4. 各据点支持哪些服务模式
5. 当前服务状态
6. 停车容量与空位情况

#### 3. 车辆与调度

后台统一管理：

1. 车辆档案
2. 资产归属
3. 当前所在门市和据点
4. 启用状态
5. 营运状态
6. 自动派车、人工改派和异常调度

#### 4. 价格体系

后台统一管理：

1. 车组标准价
2. 动态调价
3. 里程规则
4. 异地还车费和调度费口径

#### 5. 行销与增值服务

后台统一管理：

1. 行销方案
2. 优惠资格
3. 到店人工核验规则
4. 增值服务

#### 6. 业务规则

后台统一管理：

1. 支付与预授权
2. 取消与爽约
3. 取还车超时
4. 延长用车
5. 调度锁定
6. 费用处理口径

#### 7. 企业资格与月结

后台需要区分：

1. 个人自付
2. 企业非月结
3. 企业月结

不同身份对应不同的支付、预授权、账单和结算口径。

### 6.3 这一段的结束话术

因此，门市租赁的后台能力不是单点功能，而是订单、站点、车辆、价格、优惠、规则、企业口径几块能力一起配合。

## 7. 第五部分：关键业务规则

### 7.1 这一段只讲关键规则，不展开所有细节

重点只讲影响对方理解整体设计的规则。

### 7.2 对外说明口径

当前有几条关键规则需要先统一理解：

1. **按车组预约，不按具体车牌预约**
   - 客户买的是车组，不是某一台具体车辆

2. **具体车辆在取车前完成分派**
   - 下单先锁车组库存，后续再锁具体车

3. **门市租车和 24 小时自助共用同一套价格和规则底座**
   - 差别主要在履约方式，不在价格底座

4. **企业月结与个人自付是两套资金口径**
   - 企业月结不走客户支付、客户预授权和客户退款

5. **取车现场允许做人工资格核验**
   - 某些优惠方案需要到店核验证件
   - 资格不符时，门店可以现场更换方案并重算费用

6. **还车按实际结果结算**
   - 还车后可能进入补缴、退款、挂账或直接完成

7. **本期调度边界控制在同车行范围**
   - 自动派车、人工改派、改还车门市/据点，本期先按同车行闭环

### 7.3 这一段的结束话术

这几条规则决定了门市租赁的业务骨架，后续页面和流程都是围绕这些规则展开。

## 8. 第六部分：本期范围与边界

### 8.1 一定要单独讲本期边界

这一段用于防止客户或业务默认本期包含所有复杂场景。

### 8.2 对外说明口径

本期已经覆盖的范围包括：

1. 门市租车与 24 小时自助租车两种模式
2. 预约、派车、取车、用车、还车、结算完整链路
3. 个人自付、企业非月结、企业月结三种资金口径
4. 车组价格、动态调价、行销方案、增值服务、业务规则
5. 同车行范围内的调度、改派、改还车处理
6. 历史版本和订单快照追溯

本期暂不做或不深化的内容包括：

1. 跨车行自动派车
2. 跨车行人工改派
3. 跨车行改还车门市或据点
4. 站点关闭后的自动改站、自动转单、自动取消
5. 统一排程中心正式落地
6. 审批流
7. 复杂自动客服兜底

### 8.3 这一段的结束话术

本期先把核心主链路和同车行范围内的履约闭环做完整，复杂跨组织场景留到后续阶段单独规划。

## 11. 结尾总结

### 11.1 结尾话术

门市租赁当前的规划目标，不是单独做一个下单页面，而是完整建立一条可运营、可履约、可结算、可追溯的业务链路。  
当前整体设计已经把客户预约、门店交付、营运调度、价格优惠、规则配置和企业结算几块能力串成了一套完整框架。  
本期先把核心主链路和同车行范围内的履约闭环做完整，后续再逐步扩展更复杂的跨组织和自动化场景。
