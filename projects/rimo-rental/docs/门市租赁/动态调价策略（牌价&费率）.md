## 1. 文档概述

### 1.1 背景
动态调价策略模块是租车业务的核心定价引擎，主要用于管理 "门市租车" 业务下各车型组的**基准日租金**及**动态调价策略**。
*范围说明*：本文件仅聚焦于 **"门市租车"** 业务场景。

### 1.2 目标
构建一套安全、灵活的价格管理体系，实现：
1.  **基准定价**：设定各车型组的标准门店挂牌价。
2.  **安全管控**：通过 "锁定/解锁" 机制防止误操作。
3.  **定时调价**：支持预设未来的统一价格调整计划（如旺季调价、淡季降价），系统自动执行，无需人工守夜。
4.  **价格执行逻辑**：明确新老订单的价格适用规则。

## 2. 核心概念

| 概念 | 说明 | 示例 |
| :--- | :--- | :--- |
| **标准日租金** | 车型组的基础日租金，也是前台展示的默认划线价。 | 经济型组：¥150/天 |
| **定时调价计划** | 预设在未来某个特定时间点自动生效的新标准价表。 | "五一调价"：5月1日 00:00 生效，全线统一涨价20%。 |
| **价格锁定** | 只有在"解锁"状态下才允许修改价格，平时处于只读模式。 | UI显示锁图标，输入框禁用。 |
| **生效逻辑** | 调价仅对**新创建**的订单生效，不影响已预订或进行中的订单。 | 用户上周订的五一用车，仍按上周的低价结算。 |

## 3. 功能需求详情

### 3.1 门市标准价管理
*   **展示形式**：列表展示 (Table)。
*   **数据项**：
    *   **车组名称/车型明细**：展示该组包含的具体车型。
    *   **标准日租金**: 核心定价字段，支持精确到元。
    *   **上次更新时间**: 记录最后一次改价的时间。
*   **交互逻辑 - 安全锁定**：
    *   **默认状态**：锁定 (只读)。
    *   **解锁流程**：点击 "解锁编辑" -> 界面变为可编辑状态。
    *   **保存流程**：修改价格 -> 点击 "保存" -> 数据提交并重新锁定。
    *   **取消**：放弃本次修改，恢复原值并锁定。

### 3.2 定时调价计划
*   **场景**：应对旺季、淡季或统一价格策略调整，运营人员提前录入标准价表，系统到点自动切换。
*   **计划列表**：
    *   展示所有 "待执行" 和 "已执行" 的历史计划。
    *   状态流转：`待执行` -> (时间到达) -> `已执行`。
*   **新增计划 (配置弹窗)**：
    *   **生效时间 (必填)**: 精确到分钟 (e.g., 2024-05-01 00:00)。
    *   **计划摘要**: 用于备注 (e.g., "国庆旺季第一波涨价")。
    *   **价格配置表**:
        *   列出所有车型组及"当前价格"。
        *   提供 **"新价格"** 输入列。
        *   **快捷工具**: 支持 "批量上调/下调百分比" (e.g., 全选 -> 批量涨价 10%)，便于快速操作。
*   **取消计划**:
    *   仅 "待执行" 状态的计划可被取消。

## 4. 业务逻辑流程

### 4.1 价格执行与生效机制
1.  **基础定价**: 运营人员设置 "经济型组" 日租金为 ¥100。
2.  **创建计划**: 运营人员创建 "旺季价格计划"，设定本周六 00:00 生效，价格调整为 ¥120。
3.  **时间轴推演**:
    *   周五 23:59: 用户下单，系统取当前价 **¥100**。
    *   **周六 00:00 (触发点)**: 系统后台作业自动运行，将 "经济型组" 的基础日租金更新为 **¥120**。计划状态变为 "已执行"。
    *   周六 00:01: 新用户下单，系统取最新价 **¥120**。
4.  **老订单基线保护**: 周五下单的用户，若订单后续没有发生续租、改租期、改车组、到店改方案或逾时重计费等商业条件变化，则原订单基线部分依然按 **¥100** 结算；如后续发生商业条件变化，则仅对变化部分按事件发生时命中的有效版本生成事件级增量快照并结算差额。

### 4.2 订单计费快照边界
*   动态调价只更新车组标准价主数据。新版本生效后，影响该时间之后创建的新订单，以及既有订单后续发生的商业条件变化事件。
*   订单创建成功时，系统将当时的车组标准价写入订单级基线快照；后续调价计划执行后，不反向改写已创建订单的基线快照。
*   纯履约动作，如派车、改派、硬锁、据点调整、交还车验车，不生成新的价格事件快照，继续读取订单基线快照。
*   既有订单如发生续租、改租期、改车组、取车现场改行销方案、逾时重计费等需要重定价的商业事件，系统按事件发生时命中的标准价版本、行销方案版本和业务规则版本生成事件级增量快照，仅重算该事件影响的增量差额，不覆盖原订单基线快照。
*   车组标准价本身不按平日/假日拆分。订单租期内跨平日/假日时，平日/假日差异由行销优惠方案的折扣配置产生，不由标准价分段产生。
*   行销优惠方案、优惠券、积分和增值服务价格先按订单或事件快照执行。结算时读取“订单基线快照 + 事件快照链”，不得回查当前主数据重新解释历史订单。

### 4.3 标准价版本记录
门市标准价直接编辑保存或定时调价计划执行成功时，系统生成新的标准价版本，并将该版本设置为后续新建订单的可用价格来源。

| 场景 | 版本处理 |
| :--- | :--- |
| 手动修改标准价 | 保存成功后生成新的 `priceVersionId`，记录修改前价格、修改后价格、保存人、保存时间和变更说明。 |
| 定时调价计划执行 | 到达生效时间并执行成功后生成新的 `priceVersionId`，关联调价计划 ID、计划名称和执行时间。 |
| 计划取消 | 不生成新的价格版本，仅保留取消记录。 |
| 历史查看 | 历史版本只读展示，可用于订单详情页查看订单下单当时的标准价来源。 |

订单创建成功时，系统记录当时命中的标准价版本号和标准价明细；如后续商业事件触发重定价，再记录对应事件时命中的标准价版本号和标准价明细。后台查看订单时，可从订单详情的计费快照快速跳转或查看订单基线版本及各事件版本。

动态调价策略页需要提供“历史版本”入口，支持按版本号、车组、保存时间、来源操作筛选。点击版本号打开版本详情，版本详情直接沿用门市标准价表格的展示结构，以历史版本当时的标准日租金、车组、车型明细、版本号和更新时间展示。调价来源、执行计划、执行时间、操作人和变更说明在表格上方展示。订单详情页点击 `priceVersionId` 时，可弹窗查看同一份版本详情，或跳转至动态调价策略页的对应历史版本。

## 5. 权限与验证

| 角色 | 权限 |
| :--- | :--- |
| **超级管理员** | 拥有所有改价权限。 |
| **运营总监** | 拥有所有改价权限。 |
| **门店店长** | **无权限** (通常定价权收归总部，门店仅执行)。或仅有"查看权"。 |

## 6. 数据校验
*   **价格非负**: 租金必须 >= 0。
*   **时间有效性**: 定时计划的生效时间必须晚于当前时间。
*   **冲突检测**: 同一时间点不能存在两个待执行的计划（避免执行顺序冲突）。
