## 1. 文档概述

### 1.1 背景
在基础产品定价模块中，"产品"特指**车辆租赁产品组**。它是一组具有相似属性、统一标准价和里程限制规则的车型集合。
*区别说明*：本模块不同于"增值服务"，本模块核心管理的是**车+车组标准价+里程规则**的打包定义。

### 1.2 目标
为"门市租车"业务提供灵活的产品组配置能力，实现：
1.  **产品组定义**：将不同车型归类为标准化的产品组（如：经济型、舒适型、豪华型）。
2.  **定价策略**：设定车组统一标准价（牌价）。
3.  **里程规则引擎**：灵活配置"限制里程"、"超里程费"及"封顶逻辑"，支持统一配置或按车型单独配置。
4.  **车型关联**：建立产品组与具体车型款型的绑定关系。

## 2. 核心概念

| 概念 | 说明 | 示例 |
| :--- | :--- | :--- |
| **产品组** | 销售端呈现给用户的最小SKU单元。 | "经济型轿车组 - 每天¥150" |
| **车组标准价（牌价）** | 该组车辆的门市租车统一日租标准价，不含保险、增值服务和行销优惠。标准价本身不区分平日/假日。 | ¥200/天 |
| **里程规则** | 定义包含的免费公里数及超额收费标准。 | 每天限200km，超每公里¥2.0 |
| **统一/独立配置** | 决定规则是应用于组内所有车，还是特定车型特殊处理。 | 统一：全组限200km。独立：组内"坦克300"限300km。 |

## 3. 功能需求详情

### 3.1 门市租车产品组列表
*   **展示形式**：卡片式网格布局。
*   **卡片内容**：
    *   **头部**：分组名称、ID、业务标识（门市）。
    *   **定价区**：车组标准价（牌价，¥/天）。
    *   **规则区**：
        *   若为统一配置：显示具体的免费里程、超价、封顶。
        *   若为独立配置：显示"个性化里程费"标签及包含车型数量。
    *   **车型区**：列出前3个关联车型的名称 (e.g., 丰田卡罗拉, 大众朗逸...)。
*   **操作**：
    *   **新增产品组**：弹出配置对话框。
    *   **配置详情**：编辑现有分组。
    *   **删除**：物理删除分组（需校验：若该组下已有生效订单或车辆，不允许删除并提示）。

### 3.2 产品组配置
*用户点击"新增"或"配置详情"时触发的抽屉/弹窗。*

#### 3.2.1 基础信息
1.  **产品组名称**: 必填 (e.g., "商务行政组")。
2.  **车组标准价（牌价）**: 必填，单位：元/天。
    *   *说明*：这是门市租车基础计费价，实际订单会在标准价基础上匹配行销优惠方案、优惠券和积分抵扣。标准价保持统一，不按平日/假日拆分；平假日差异由行销方案配置不同折扣价形成。

#### 3.2.2 里程费规则 - 核心亮点
提供开关切换：**统一配置 / 单独配置**。

*   **模式 A：统一配置**
    *   适用于组内车型成本差异不大的情况。
    *   **字段定义**:
        *   **每日免费里程**: 整数 (e.g., 200)。0表示不限里程。
        *   **超程单价**: 浮点数 (e.g., 3.5)。
        *   **每日封顶金额**: 整数 (e.g., 200)。0表示上不封顶。
            *   *示例逻辑*: 超出100km，每公里2元，即200元。若封顶设为150，则只收150。

*   **模式 B：单独配置**
    *   适用于组内混有特殊高油耗/高损耗车型的情况。
    *   在下方的"关联车型列表"中，为每一行车型单独开启输入框，分别输入上述三个参数。

#### 3.2.3 特殊车辆管控
*   **标记为特殊产品组**: 开关。
*   **限制逻辑**:
    *   若开启，需选择 **最低会员等级要求** (e.g., 仅限"白银会员"及以上可预订)。
    *   *场景*: 跑车组或高性能车组，不对新用户开放。

#### 3.2.4 关联车型
*   **添加车型**: 弹窗选择器。
    *   从系统"车辆库"中选择车型（品牌+车系+年款）。
    *   *过滤逻辑*: 仅展示未被绑定到其他产品组的车型（通常一辆车只能属于一个分时或门市组）。
*   **列表展示**:
    *   显示车型图片、品牌、型号。
    *   若处于"单独配置"模式，此处直接显示可编辑的里程参数输入框。
*   **移除**: 解除该车型与本组的绑定。

### 3.3 门市租金计费口径
门市租车订单以车组标准价作为基础计费价。订单创建时记录车组标准价基线快照，并写入订单计费快照；后续车组改价或动态调价不反向改写已创建订单的基线快照。既有订单如发生续租、改租期、改车组、取车现场改行销方案或逾时重计费等商业条件变化，则按事件发生时命中的有效版本生成事件级增量快照，仅重算该事件影响的增量差额。

1. **起租规则**：起租 24 小时，租期不足 24 小时按 24 小时计费。
2. **超时折算**：超过完整 24 小时后，剩余时长不足 1 小时按 1 小时计；每个计费小时按车组标准价 10% 计费。
3. **封顶规则**：剩余计费小时达到第 5 小时起，按 1 天计费。
4. **跨平假日规则**：车组标准价本身不区分平日/假日；行销优惠方案区分平日/假日折扣时，以每一轮 24 小时的起始时间判断该轮适用平日折扣或假日折扣。超过完整 24 小时后的剩余时长视为新一轮 24 小时的开始，按该开始时间判断平日/假日。

### 3.4 标准价与里程规则版本记录
车组标准价、里程规则和产品组适用范围发生保存时，系统生成新的标准价版本，已生成版本只读保留，不允许被覆盖修改。

| 版本字段 | 规则 |
| :--- | :--- |
| 标准价版本号 | 系统生成唯一 `priceVersionId`，用于订单计费快照追溯。 |
| 产品组信息 | 记录产品组 ID、产品组名称、业务线、关联车型范围。 |
| 标准价 | 记录保存时的日租标准价。标准价本身不区分平日/假日。 |
| 里程规则 | 记录统一里程规则或按车型独立里程规则，包含免费里程、超程单价和封顶金额。 |
| 生效状态 | 记录当前生效、历史版本或未来待生效。 |
| 操作信息 | 记录保存人、保存时间、变更说明和来源操作。 |

订单创建成功时，系统将命中的 `priceVersionId` 以及该版本中的标准价、里程规则复制到订单级基线快照。后续产品组改价、里程规则调整、车型绑定变化不反向改写已创建订单的基线快照；若订单后续发生续租、改租期、改车组、逾时重计费等商业事件，则按事件发生时命中的 `priceVersionId` 和规则生成事件级增量快照，仅结算事件差额。

产品组列表和配置详情页需要提供历史版本入口。点击版本号或“历史版本”时，打开版本详情弹窗，版本详情沿用当前产品组配置页的字段命名和展示结构，以只读方式展示该版本内的完整设定值，包括产品组基础信息、标准价、里程规则、车型绑定、特殊车辆管控、保存人、保存时间和变更说明。历史版本为只读，不允许在历史版本弹窗内直接编辑；如需恢复历史配置，需要在当前配置页重新保存并生成新的版本。

## 4. 业务逻辑流程

1.  **创建组**: 运营人员创建一个 "新手练手组"，定价 ¥80/天。
2.  **设定规则**: 配置统一里程限制：每天限100km（防止过度损耗），超程 ¥5.0/km（高惩罚）。
3.  **绑定车型**: 选择系统中已有的 "五菱宏光MINI", "奇瑞QQ"。
4.  **前端下单**:
    *   用户在APP选择 "门市租车"。
    *   系统展示 "新手练手组"，包含五菱和奇瑞。
    *   用户下单租3天。
    *   **预估里程费**: 此时为0。
5.  **还车结算**:
    *   还车录入：行驶了 400km。
    *   额度：100km * 3天 = 300km。
    *   超额：400 - 300 = 100km。
    *   费用：100km * ¥5.0 = ¥500。
    *   系统自动将其计入订单的 "里程费" 科目。

## 5. 权限与验证

| 角色 | 权限 |
| :--- | :--- |
| **超级管理员** | 所有权限。 |
| **门店经理** | 可查看，不可修改（产品组通常由总部统一标准化，避免门店乱定价）。 |
| **运营专员** | 可查看。 |

## 6. 数据验证规则
*   **必填校验**: 名称、价格、免费里程不能为空。
*   **逻辑校验**: 封顶金额若大于0，必须大于 (超程单价 * 1)。
*   **唯一性**: 一个车型 ID 在同一时间只能属于一个有效的产品组。
