## 1. 文档概述

### 1.1 背景
增值服务管理模块用于管理租车业务中的**增值服务**。与核心车辆产品不同，增值服务通常作为附加选项存在，旨在提升用户体验和增加客单价。
*范围说明*：本文件仅聚焦于 **"门市租车"** 业务场景。

### 1.2 目标
构建一套通用的增值服务管理体系，支持：
1.  **多品类管理**：涵盖保险、实物设备、人工服务等多种类型。
2.  **灵活定价**：支持按天、按次计费。
3.  **库存控制**：针对实物设备（如儿童座椅）实现分门店的库存管理。
4.  **准入限制**：基于会员等级设定服务的购买资格。

---

## 2. 核心概念

| 概念       | 说明                  | 示例                  |
| :------- | :------------------ | :------------------ |
| **保险类**  | 虚拟权益，无实体库存，通常按天收费。  | 尊享全险、人身意外险。         |
| **设备类**  | 实物资产，**有库存限制**，需归还。 | 儿童座椅、GPS导航、雪地防滑链。   |
| **服务类**  | 人工履约，无实体库存，按次收费。    | 加油服务费、送车上门、异地还车费。   |
| **门店库存** | 实物设备在各门店的物理库存数量。    | 北京店有10个安全座椅，上海店有5个。 |

## 3. 功能需求详情

### 3.1 增值服务列表
*   **展示形式**：数据表格。
*   **列表字段**：
    *   **基础信息**：服务名称、编码、图标。
    *   **类型标签**：保险 (红)、设备 (绿)、服务 (灰)。
    *   **适用会员**：展示允许购买的会员等级标签 (e.g., "黄金会员", "所有会员")。
    *   **价格**：金额及单位 (e.g., ¥50/天)。
    *   **库存/限购**：
        *   若开启库存管理：显示 "按门店管理" 标签。
        *   若无限库存：显示 "无限库存"。
        *   单单限购：显示最大可购数量 (e.g., 限2件)。
    *   **状态**：启用/禁用开关。
*   **操作**：
    *   **新增门市服务**：打开配置弹窗。
    *   **编辑**：修改现有服务。
    *   **库存管理**：仅针对开启库存追踪的服务，打开门店库存弹窗。
    *   **删除**：物理删除（需二次确认）。

### 3.2 服务配置 (新增/编辑)
*通过侧滑抽屉或弹窗进行配置。*

#### 3.2.1 基础定义
1.  **服务名称**: 必填。
2.  **服务编码**: 唯一标识，用于系统对接。
3.  **服务类型**: 下拉选择 (保险/设备/服务)。
4.  **图标**: 选择内置图标。
5.  **会员限制**: 多选框，选择允许购买此服务的会员等级。不选默认为"全部会员"。
    *   *场景*：某高端接送服务仅限黑钻会员可见。

#### 3.2.2 门市租车定价
1.  **价格**: 必填，支持2位小数。
2.  **计价单位**:
    *   **按天 : 随租期累加 (如保险、Wifi)。
    *   **按次**: 一次性收费 (如清洁费、加油费)。
3.  **最大加购数量**: 限制单个订单最多购买多少份（通常保险限1份，座椅可能限2份）。

#### 3.2.3 库存与状态
1.  **库存管理开关**:
    *   **开启 (按门店管理)**: 适用于"设备类"。开启后，需在列表页点击"库存"按钮单独配置各店数量。系统下单时会校验对应门店是否有剩余库存。
    *   **关闭 (无限供应)**: 适用于"保险类"或"服务类"。
2.  **状态**: 启用/禁用。

---

### 3.3 门店库存管理
*仅对开启"库存管理"的服务可用。*

*   **交互**：弹窗展示所有门店列表。
*   **数据项**：
    *   **门店名称/ID**。
    *   **当前库存数**: 输入框，允许门店经理或管理员调整。
*   **逻辑**：
    *   总库存 = 所有门店库存之和。
    *   下单逻辑：用户选择 A 门店取车 -> 系统检查 A 门店该设备的 `可用库存` (总库存 - 已预订未归还)。
    *   *高级扩展 (暂不实现)*: 异地还车导致的库存流转（A借B还，A库存-1，B库存+1）。当前阶段简化为固定库存池扣减。

### 3.4 增值服务版本记录
增值服务新增、编辑、价格调整、会员限制调整、库存管理开关调整、状态启停和门店库存规则调整时，系统需要生成新的服务版本。

历史版本只读保留，不允许覆盖修改已被订单引用的旧版本。

版本详情至少展示以下信息：

1. 服务基础信息：服务名称、服务编码、服务类型、适用场景、适用会员、状态、服务说明。
2. 价格信息：价格、计价单位、最大加购数量。
3. 库存信息：是否按门店库存管理、当前库存口径。
4. 保存信息：版本号、保存人、保存时间、版本状态。

页面需要提供“历史版本”入口。点击版本号或“历史版本”后，打开只读版本详情弹窗，版本详情沿用订单详情页中的版本详情样式，使用表格展示基础信息和价格/库存信息。

### 3.5 订单快照口径
订单创建成功时，系统需要把本单命中的增值服务版本号和关键配置写入订单级基线快照，至少包含：

1. `serviceVersionId`
2. 服务名称
3. 服务编码
4. 服务类型
5. 价格
6. 计价单位
7. 下单时选择数量
8. 会员限制结果
9. 是否启用库存管理
10. 订单创建时的库存扣减口径

后续增值服务主数据被修改、停用、删除、改价或调整库存规则时，不得反向改写已创建订单中的服务快照。

订单详情页点击 `serviceVersionId` 时，需要能查看该订单下单时命中的服务版本详情，而不是读取当前最新主数据。

本期增值服务以订单级基线快照为主。若后续新增“履约过程中增减服务并重算费用”的能力，再按订单快照机制补充事件级增量快照。

## 4. 业务逻辑流程

### 4.1 保险购买流程 (无限库存)
1.  管理员创建 "尊享全险"，定价 ¥80/天，无限库存。
2.  用户下单租 3 天。
3.  费用 = 80 * 3 = ¥240。
4.  不受门店限制，所有门店均可提供。

### 4.2 儿童座椅租赁流程 (有库存)
1.  管理员创建 "儿童座椅"，定价 ¥20/天，开启库存管理。
2.  设定库存：北京店 2 个，上海店 0 个。
3.  **场景 A (北京下单)**:
    *   用户选北京店取车。系统查询剩余库存 > 0。
    *   允许用户勾选。下单成功，北京店可用库存 -1。
4.  **场景 B (上海下单)**:
    *   用户选上海店取车。系统查询库存 = 0。
    *   前端置灰该选项，提示 "当前门店缺货"。

## 5. 权限与验证

| 角色 | 权限 |
| :--- | :--- |
| **超级管理员** | 拥有所有权限 (定义服务、定价、调拨库存)。 |
| **门店经理** | 无法新增/删除服务，无法修改价格。**仅允许调整自己门店的库存数量**。 |
| **运营专员** | 仅查看。 |

## 6. 数据校验
*   **编码唯一性**: 服务编码不可重复。
*   **价格合法性**: 必须 >= 0。
*   **最大数量**: 必须 >= 1。
