## 1. 页面范围

原型页面：`02_原型和PRD/原型/企业端/dashboard.html`

页面用途：展示企业公务车运营概览，包括员工、车辆、订单、用车时长、使用率、分公司使用状况、部门用车统计和排名数据。

## 2. 查看范围

1. 企业总管理员查看企业全量数据。
2. 企业总管理员选择总公司范围时，分公司使用状况展示各分公司数据。
3. 企业总管理员选择某一分公司时，分公司使用状况展示该分公司内车辆数据。
4. 分公司管理员默认查看本分公司数据。

## 3. 顶部指标卡

| 指标 | 口径 | 展示规则 |
| --- | --- | --- |
| 员工数 | 当前查看范围内所有在职员工数量，不受用车资格状态影响 | 显示在职员工总数 |
| 可用车员工数 | 当前查看范围内在职且用车资格为“启用”的员工数量 | 显示具备用车资格的在职员工数量 |
| 公务车数 | 当前查看范围内所有公务车数量，不受合约与上架状态影响 | 显示公务车总数 |

## 4. 累计指标与较上月

顶部卡片统一使用“较上月”，不使用“同比上月”。“同比”通常表示与上年同期比较，不适用于本页面的上月比较。

### 4.1 总计公务用车次数

“总计公务用车次数”为历史累计指标，不计算与上月累计值之间的增长百分比，也不展示本月新增次数，避免与“本月用车次数”卡片重复。

卡片副文案固定显示“累计至今日”。右侧保留现有迷你折线，用于视觉表达近期上升或下降趋势，不展示较上月百分比。

统计周期：自企业启用公务车业务起，累计至当前时间。当前迷你折线仅作趋势视觉提示，不接入独立趋势数据，也不参与总计公务用车次数的计算。

### 4.2 本月平均使用率较上月

本月平均使用率本身按以下公式计算：

`本月平均使用率 = 本月实际用车分钟数 / 本月车辆理论可用分钟数 * 100%`

与上月比较时采用变化比例：

`较上月 = (本月平均使用率 - 上月同期平均使用率) / 上月同期平均使用率 * 100%`

结果以百分比展示并保留 1 位小数。例如本月 76.4%、上月同期 71.8%，计算结果为 `(76.4% - 71.8%) / 71.8% = 6.4%`，展示“较上月 +6.4%”。

### 4.3 本月用车次数较上月

`较上月 = (本月用车次数 - 上月同期用车次数) / 上月同期用车次数 * 100%`

结果保留 1 位小数，并根据正负值展示上升、下降或持平状态。

### 4.4 比较周期与零值规则

1. 当前月份尚未结束时，上月数据截取至相同日期进度进行比较；当前月份结束后，比较两个完整自然月。
2. 本月用车次数比较中，上月同期次数大于 0 时按公式计算。
3. 上月同期次数为 0、本月次数大于 0 时，展示“上月无数据”，不展示百分比。
4. 上月同期次数与本月次数均为 0 时，展示“持平”。
5. 平均使用率比较中，上月同期使用率为 0、本月使用率大于 0 时，展示“上月无数据”；两者均为 0 时展示“持平”。
## 5. 各分公司使用状况总览

### 5.1 总公司视角

展示各分公司的使用状况。

| 字段 | 说明 |
| --- | --- |
| 分公司 | 分公司名称 |
| 可用车辆 / 车辆总数 | 分公司当前可预约车辆数 / 分公司车辆总数 |
| 本月已完成次数 | 本月已完成公务车订单数量 |
| 本月用车时数 | 本月实际用车总时长，展示到分钟 |
| 使用率 | 本月实际用车分钟数 / 本月车辆理论可用分钟数 * 100% |
| 较上月 | 本月使用率相对上月同期使用率的变化比例，单位为 % |
| 预约待交次数 | 当前仍未完成的订单数量，包含已预约未完成和用车中订单 |
| 平均每次用车时数 | 本月实际用车总时长 / 本月已完成次数 |

### 5.2 分公司视角

展示该分公司内各车辆的使用状况。字段与总公司视角一致，分公司名称替换为车辆。

## 6. 部门用车统计

1. 统计维度支持用车时长和用车次数。
2. 总公司视角下，部门名称前显示分公司名称。
3. 分公司视角下，直接显示部门名称。
4. 图表条形长度只表示当前列表内的相对占比，计算公式为：`当前部门值 / 当前列表最大值 * 100%`。
5. 条形占比不是业务指标，不单独展示百分比；页面展示真实用车时长或用车次数。
6. 图表区域支持上下滚动，能够查看所有部门数据。

## 7. 总体用车率

总体用车率用于观察本月与上月趋势。

计算公式：

`每日使用率 = 当日实际用车分钟数 / 当日车辆理论可用分钟数 * 100%`

规则：

1. 车辆理论可用分钟数按查看范围内可统计车辆数量计算。
2. 已停用、已下架车辆不计入当日可用分钟数。
3. 用车中订单按实际已发生时长计算。
4. 已完成订单按实际取还车时间计算。
