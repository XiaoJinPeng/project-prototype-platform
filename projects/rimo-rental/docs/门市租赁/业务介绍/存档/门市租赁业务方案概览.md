# 门市租赁业务方案概览

## 1. 文档定位

本文用于面向客户、业务人员、实施人员快速介绍门市租赁业务的业务范围、核心流程、主要能力和本期边界。

本文只说明整体方案，不展开底层实现细节。涉及价格、规则、订单状态、调度、快照等详细口径时，以对应 PRD 正文为准。

---

## 2. 业务范围

门市租赁当前包含两种服务模式：

1. **门市租车**
   - 客户在线预约，到门市由店员办理取车和还车。
   - 适用于需要门店人工核验身份、证件、优惠资格、车辆交付和验车的场景。

2. **24 小时自助租车**
   - 客户在线预约，到指定自助据点自行取车和还车。
   - 适用于具备车机能力、自助交付能力的场景。

两种模式共用一套门市租赁业务底座，包括：

1. 车组
2. 标准价
3. 动态调价
4. 行销方案
5. 增值服务
6. 租赁业务规则
7. 订单计费逻辑
8. 调度与派车能力

两者的核心区别，主要在于**取还车方式、客户面向的履约地点和履约操作流程**不同。

### 2.1 两种模式的关键差异

| 维度 | 门市租车 | 24 小时自助租车 |
| --- | --- | --- |
| 客户面向地点 | 门市 | 具体自助据点 |
| 取车方式 | 店员现场办理 | 客户自助取车 |
| 还车方式 | 店员现场办理 | 客户自助还车 |
| 身份与优惠资格核验 | 可由门店现场核验证件、资格和优惠条件 | 不依赖现场人工核验 |
| 行销方案限制 | 可使用到店人工核验类方案 | 开启到店人工核验的方案本期不适用 |
| 车机依赖 | 可无车机运行 | 必须具备车机能力 |
| 据点角色 | 用于内部备车、停放、整备和归还登记 | 直接作为客户取还车地点 |

因此，门市租车的核心是门店服务履约，24 小时自助租车的核心是站点能力和自助交付能力。

## 3. 业务目标

门市租赁解决的是“按天或按多天租车、按车组售卖、在取还车现场完成交付与验车”的业务场景，覆盖以下目标：

1. 支持客户提前预约门市租车或自助租车。
2. 支持门店或系统在履约前完成车辆分派。
3. 支持门店在取车现场完成身份核验、资格核验、验车、补收尾款和交车。
4. 支持按实际还车结果完成验车、结算、补缴、退款或挂账。
5. 支持企业客户与个人客户共用同一套门市租赁业务，但按不同结算方式处理。
6. 支持营运方统一管理门市、据点、车辆、价格、优惠、服务和规则。

## 4. 核心业务对象

### 4.1 车组

门市租赁按**车组**销售，而不是在下单时直接销售具体车牌。

客户预约时选择的是车组、租期、取还车地点、行销方案和增值服务。具体履约车辆由系统或人工在取车前完成分派。

详细口径见：

- `基础产品定价（车组）.md`
- `动态调价策略（牌价&费率）.md`
- `门市租赁调度边界与履约规则.md`

### 4.2 车辆

车辆是实际履约资源，承担交车、用车、还车和后续调度。

车辆需要区分：

1. 资产所属车行
2. 当前所在门市
3. 当前所在据点
4. 启用状态
5. 营运状态
6. 当前支持的服务类型

门市租赁订单履约时，最终需要落到具体车辆。

详细口径见：

- `门市租赁调度边界与履约规则.md`
- `智能调度派车规则.md`
- `服务站点管理.md`

### 4.3 门市与据点

门市租赁的站点体系分为：**车行 -> 门市 -> 据点**

口径如下：

1. 门市租车客户面向门市。
2. 24 小时自助租车客户面向具体据点。
3. 车辆始终挂在具体据点下。
4. 据点承担停车、整备、备车、归还停放等职能。

详细口径见：

- `服务站点管理.md`
- `01_业务知识库与决策台账/业务知识库/门市租赁业务说明.md`

## 5. 客户与结算口径

门市租赁当前涉及三种主要客户口径：

### 5.1 个人自付订单

客户自行完成支付、预授权、补缴和退款。

### 5.2 企业非月结订单

客户以企业资格下单，但费用仍由客户自己支付，后续由客户自行向企业报销。

### 5.3 企业月结订单

订单费用不向客户收取，不发起客户支付和客户预授权，最终统一挂入企业月结账单。

因此，企业月结订单在订单流转、费用展示和结算去向上，与个人订单和企业非月结订单不同。

- `租赁业务规则.md`

## 6. 核心业务流程

### 6.1 预约下单

客户在线选择：

1. 取车方式
2. 取还车门市或据点
3. 租期
4. 车组
5. 行销方案
6. 增值服务
7. 身份类型

系统根据规则计算预估费用，并按租赁业务规则处理支付或预授权。

详细口径见：

- `产品需求文档 - 订单/新建订单.md`
- `租赁业务规则.md`
- `行销方案管理.md`
- `增值服务管理（保险&配件）.md`

### 6.2 派车与锁车

订单建立后，系统先锁定车组库存，不在下单时直接锁定具体车辆。

后续由系统或人工完成：

1. 预分配
2. 硬锁定
3. 异常改派

取车前必须存在已硬锁车辆，才能进入实际发车。

详细口径见：

- `智能调度派车规则.md`
- `智能调度派车算法.md`
- `产品需求文档 - 订单/订单改派.md`
- `门市租赁调度边界与履约规则.md`

### 6.3 取车

门市租车取车时，门店完成：

1. 客户身份确认
2. 证件检查
3. 行销方案资格核验
4. 车辆验车
5. 尾款或履约收款处理
6. 发车

24 小时自助租车不经过门店柜台办理，客户直接面向具体自助据点，通过自助流程完成取车，因此不依赖门店现场人工核验和人工交车。

如果现场资格不符，例如原优惠方案要求学生、教师或其他资格证明，但客户无法出示证明，门店需要在取车现场调整方案并按新方案重算费用。

详细口径见：

- `产品需求文档 - 订单/订单取车.md`
- `行销方案管理.md`
- `产品需求文档 - 订单/费用管理.md`

### 6.4 用车中

订单进入用车中后，客户按租期使用车辆。期间可能发生：

1. 延长用车
2. 改还车门市或据点
3. 事故处理
4. 人工改派或换车

其中，仅改变履约安排但不改变商业承诺的动作，不改变原订单商业基线；若改变商业条件，则需要按对应事件重新计算差额。

详细口径见：

- `产品需求文档 - 订单/延长用车.md`
- `产品需求文档 - 订单/订单改派.md`
- `产品需求文档 - 订单/事故出险处理.md`
- `产品需求文档 - 订单/订单详情.md`

### 6.5 还车与结算

还车时，系统需要根据实际还车结果完成：

1. 还车验车
2. 费用重算
3. 补缴、退款或挂账
4. 车辆状态回写
5. 订单完成

门市租车按门市办理还车；24 小时自助租车按自助据点办理还车。

详细口径见：

- `产品需求文档 - 订单/订单还车.md`
- `产品需求文档 - 订单/费用管理.md`

## 8. 价格与费用逻辑概览

门市租赁的费用结构主要由以下几部分组成：

1. 车组标准价
2. 动态调价
3. 行销方案优惠
4. 增值服务费用
5. 异地还车费或调度费
6. 逾时费用
7. 其他补收或退款

本期计费口径已经明确：

1. 标准价按车组统一设置。
2. 行销方案在标准价基础上生效。
3. 计费结果需要保留历史版本和订单快照。
4. 后续若发生商业条件变化，需要按事件重新计算差额。

## 9. 门店现场办理的重点

门市租赁相较于分时租赁，核心差异在于现场履约能力更重。

门店现场需要处理的重点包括：

1. 身份与证件核验
2. 行销方案资格核验
3. 现场调整优惠方案
4. 车辆交付前验车
5. 还车验车
6. 尾款、补缴或退款处理
7. 异常情况人工兜底

因此，门市租赁的订单详情、取车、还车、改派、费用管理能力，需要比纯线上租赁更完整。

## 10. 本期边界

### 10.1 本期已明确闭环的范围

1. 门市租车与 24 小时自助租车两种模式。
2. 车组预约、车辆派发、取还车、续租、改派、结算全链路。
3. 个人自付、企业非月结、企业月结三种结算口径。
4. 车组价格、动态调价、行销方案、增值服务、业务规则配置。
5. 订单历史版本、价格规则快照和事件级差额追溯。
6. 同车行范围内的改派、改还车门市/据点和履约处理。

### 10.2 本期明确不做或不深化的内容

1. 跨车行自动派车
2. 跨车行人工改派
3. 跨车行改还车门市或改还车据点
4. 门市或据点关闭后的自动改站、自动转单、自动取消
5. 全业务线统一排程中心正式落地
6. 审批流
7. 复杂自动客服兜底

详细边界见：
