## 1. 文档概述

### 1.1 背景
费用管理是门市租车订单的统一资金台账，负责记录订单从下单、取车、用车、还车到完成后追缴的所有费用项、资金动作、发票记录和审计日志。费用管理不单独创造订单流程，而是承接新建订单、取车、还车、续租、改派、事故、取消关闭和追缴等业务节点产生的金额变化。

### 1.2 目标
* **统一账本**：订单内所有费用项和资金记录进入同一套费用中心，避免新建订单、取车、还车、追缴各自形成独立账本。
* **阶段清晰**：区分下单预授权、取车尾款、还车结算、完成后追缴四个阶段。
* **资金可追溯**：预授权冻结、预授权释放、预授权扣款、收款、退款、减免、企业月结均形成交易记录。
* **状态联动**：费用处理结果驱动 `pending_payment`、`payment_due`、`refund_pending`、`completed` 和追缴状态展示。
* **权限可控**：新增费用、减免、退款、作废、追缴通知、金额调整均记录操作人、时间、原因和审计日志。

### 1.3 页面入口
* 订单详情页顶部操作栏：“费用”按钮打开费用管理弹窗。
* 订单详情页右侧财务区：展示计费快照、收费明细、交易记录、发票记录和费用审计。
* 取车向导 Step 2：展示预授权押金状态、订单总额、已收款和应收尾款。
* 还车向导 Step 2：展示最终结算费用、收款、退款和减免入口。
* 订单列表：根据主状态和追缴状态展示“支付”“收款”“退款”“通知客户”“追缴收款”“查看费用”等入口。
* 对应原型文件：`order_detail.html`、`new_order.html`、`store_orders.html`

## 2. 费用管理总原则

### 2.1 费用项与资金记录分离
费用项表示客户应承担或应退还的业务金额，资金记录表示真实或模拟的资金动作。费用项和资金记录不能混用。

| 类型 | 说明 | 示例 |
| :--- | :--- | :--- |
| 费用项 `feeItem` | 记录应收、应退、减免、追缴的业务原因和金额。 | 车辆租金、超时费、清洁费、车损赔付、提前还车退费、违章追缴。 |
| 资金记录 `transaction` | 记录支付、退款、预授权、押金扣款、减免、企业月结等资金动作。 | 预授权冻结、取车尾款收款、还车补缴、原路退款、费用减免。 |

### 2.2 本期资金口径
* 客户自付订单在下单阶段按租赁业务规则收取预授权押金，包含个人预订和非月结企业预订。
* 预授权押金在下单阶段只完成冻结，不在下单阶段抵扣租金。
* 下单预授权押金本期仅支持线上预授权，不支持现金押金、线下押金或仅登记线下凭证。
* 取车阶段必须校验预授权押金有效，并在“履约担保”和“转租金”两种处理方式中选择一种。
* 履约担保模式下，预授权押金不抵扣租金；客户补足订单租金尾款并确认发车后，系统释放该笔预授权。
* 转租金模式下，系统将预授权扣款转为租金实收，取车应收尾款按订单总额扣除预授权扣款和其他已收款后计算。
* 企业预订分为企业月结和非月结企业预订。非月结企业预订使用企业资格可用行销方案或企业优惠资格，费用仍由客户本人支付，资金流程与个人预订一致；企业身份订单不支持优惠券和积分。
* 企业月结订单不进入客户支付链路，不生成预授权押金，不要求取车尾款收款；系统在还车结算确认、取消/爽约/拒绝履约确认、完成后追缴确认时生成 `monthly_bill`，相关费用统一进入企业月结账单。
* 企业月结订单的费用项统一使用 `monthly_billing` 状态，不使用客户待收款、待补缴、待退款状态。租期中产生的月结费用先进入费用中心，最终在还车结算时汇总挂账；取消费、爽约费、拒绝履约费用、已完成订单追加费用和已完成订单补录事故费用在确认时直接生成 `monthly_bill` 交易记录。
* 企业月结账单归属月份以月结挂账交易记录的入账时间判断。基础订单费用按还车结算确认并生成 `monthly_bill` 的时间入账；订单完成后的追加费用、补录事故费用、追缴、退款、费用调整或账单调整，按对应费用确认并生成 `monthly_bill` 或调整交易记录的时间入账，不使用原订单完成时间，也不回改已关闭账期。
* 还车阶段只处理租期结束后的差额结算，包括多退少补、车损、清洁、超时、超里程、油/电补差、异地还车等。门市租车异地还车费只按取车门店与订单当前还车门店是否一致判断，不按归还据点判断。
* 完成后追缴不改变订单主状态，主状态保持 `completed`，追缴状态在金额列独立展示。
* 全额支付保留为系统能力，本期不在门市租车页面启用。

企业月结入账示例：

| 场景 | 入账月份 |
| :--- | :--- |
| 订单 5 月 28 日还车完成，基础费用确认挂账 | 5 月账单 |
| 订单 5 月 28 日完成，6 月 3 日补录 ETC、车损或停车费并确认挂账 | 6 月账单 |
| 5 月账单已关闭，6 月发现需调增或调减 | 6 月生成调整项，不回改 5 月账单 |

### 2.3 24 小时自助取还车异常资金口径

24 小时自助租车的取车、还车由 APP 和车机指令驱动。费用中心必须保证资金动作与车机动作分离记录，避免因车机失败造成重复扣款、漏挂账或错误退款。

| 场景 | 资金处理 |
| :--- | :--- |
| 自助取车前整单支付 | 个人自付和企业非月结订单必须在自助取车开锁前结清整笔订单应收金额；待付金额为订单总额减已实际收款金额减预授权转租金扣款金额。下单预授权冻结未转租金时不抵扣订单总额。 |
| 自助取车支付成功后开锁失败 | 保留已成功的支付、预授权扣款或预授权处理记录；订单不得进入 `renting`；用户重试开锁时不得重复收款或重复扣款。 |
| 自助取车预授权转租金成功后开锁失败 | `pre_auth_capture` 记录保持有效，不重复发起预授权扣款；后续由开锁重试、人工交付或订单取消/拒绝履约流程决定费用处理。 |
| 自助取车支付失败 | 不写入有效收款记录，订单保持取车前状态；APP 提示重新支付或联系客服。 |
| 自助还车补/退租金结算 | 还车验车提交后，系统先计算最终 `balanceDue`。补缴完成、退款受理、企业月结挂账或账单平衡记录完成后，才允许下发车机落锁指令。 |
| 自助还车补缴成功后落锁失败 | 保留补缴交易记录；订单不得进入 `completed`；用户重试落锁时不得重复收款。 |
| 自助还车退款受理后落锁失败 | 保留退款交易记录或后台退款处理记录；订单不得进入 `completed`；重试落锁时不得重复创建退款。 |
| 自助还车企业月结挂账后落锁失败 | 保留 `monthly_bill` 挂账记录；订单不得进入 `completed`；人工确认前不得重复生成企业月结挂账。 |
| 车机指令结果未知 | 不新增资金动作；系统先查询指令最终结果。无法确认时进入人工处理，由人工处理结果决定是否继续履约、取消、拒绝履约或完成还车。 |

自助取还车资金动作必须具备业务幂等号。幂等号至少关联订单号、业务阶段、费用项、资金动作类型和外部交易请求号。同一幂等号重复提交时，系统返回原处理结果，不重新创建资金记录。

### 2.4 金额展示口径
| 口径 | 说明 |
| :--- | :--- |
| 订单总额 | 下单时根据租期、车组、增值服务和行销优惠方案计算的应收租金金额。个人预订可继续扣减积分和优惠券；企业预订不扣减积分和优惠券。 |
| 预授权押金 | 下单阶段按订单规则快照中的固定额度或租金比例冻结的金额，可在取车阶段作为履约担保释放，也可扣款转为租金实收；比例模式本期不超过 100%。 |
| 应收尾款 | 取车阶段应补齐的租金尾款；履约担保模式不扣减预授权金额，转租金模式扣减预授权扣款金额。 |
| 企业月结记账金额 | 企业月结订单最终进入企业账单的金额，不产生客户支付流水，不计入客户已付款。 |
| 还车待结金额 `balanceDue` | 还车结算阶段的最终差额，正数表示补缴，负数表示退款，0 表示平账。 |
| 追缴金额 | 订单完成后新增的违章、ETC、停车费、后发现车损等后续费用。 |

## 3. 费用阶段

| 阶段 | 阶段标识 | 触发节点 | 页面表现 | 状态影响 |
| :--- | :--- | :--- | :--- | :--- |
| 下单预授权 | `booking` | 新建订单提交时 | 新建订单弹窗展示“本次预授权金额” | 预授权成功后订单进入 `reserved`，支付状态为 `deposit_paid`。 |
| 待支付处理 | `pending_payment` | 下单选择暂不收款 | 列表和详情展示“支付”，打开下单预授权押金确认弹窗 | 预授权冻结成功后进入 `reserved`，支付状态为 `deposit_paid`；失败、关闭弹窗或未继续操作时保持 `pending_payment`。 |
| 企业月结记账 | `monthly_billing` | 企业月结订单还车结算、取消、爽约、拒绝履约或完成后追缴确认时 | 费用中心展示“企业月结” | 支付状态为 `monthly_billing`，不生成客户支付、退款和预授权入口。 |
| 取车尾款 | `pickup` | 取车 Step 2 | 展示订单总额、预授权押金、处理方式、已收款、应收尾款 | 履约担保模式尾款结清并确认发车后释放预授权；转租金模式预授权扣款后抵扣租金；尾款未结清时禁止发车。 |
| 还车结算 | `return_settlement` | 还车 Step 2 | 展示最终账单、补缴、退款、减免 | 根据 `balanceDue` 进入待结算、待补缴、待退款或完成。 |
| 完成后追缴 | `post_charge` | 订单 `completed` 后新增费用 | 费用管理展示“追缴”，列表金额列展示追缴状态 | 主状态保持 `completed`。 |

## 4. 费用项分类

### 4.1 按来源分类

| 来源 | 字段值 | 说明 | 是否可编辑 |
| :--- | :--- | :--- | :--- |
| 系统费用 | `system` | 系统按规则自动生成。 | 金额不可直接编辑，可通过规则重算或授权调整。 |
| 人工费用 | `manual` | 门店按实际情况录入。 | 待收款状态允许编辑或作废，已有关联资金记录的费用不允许直接改金额。 |

### 4.2 按阶段分类

| 阶段 | 字段值 | 说明 |
| :--- | :--- | :--- |
| 租期费用 | `rental` | 下单、取车、用车、还车过程中产生，进入订单结算。 |
| 追缴费用 | `post` | 订单完成后产生；客户自付订单独立通知客户支付，企业月结订单直接进入企业月结挂账，不改变主状态。 |

### 4.3 按金额方向分类

| 方向 | 字段值 | 说明 |
| :--- | :--- | :--- |
| 正向费用 | `charge` | 增加客户应付金额。 |
| 负向费用 | `refund` | 减少客户应付金额或形成应退金额。 |
| 减免费用 | `waiver` | 经授权后抵消客户应付金额。 |

### 4.4 费用类型

| 费用类型 | 来源 | 阶段 | 规则 |
| :--- | :--- | :--- | :--- |
| 车辆租金 | 系统 | 下单/取车 | 下单生成订单总额，取车阶段补足尾款。 |
| 续租租金 | 系统 | 用车/续租 | 延长用车确认成功后生成；立即收款时状态为已结清，还车时结算时状态为待收款。 |
| 增值服务 | 系统 | 下单/取车 | 按天或按次计费，进入订单总额。 |
| 优惠抵扣 | 系统 | 下单 | 行销优惠方案、优惠券、积分抵扣作为负向金额展示；企业身份订单仅支持企业资格行销优惠方案，不支持优惠券和积分。 |
| 取车行销方案变更差额 | 系统 | 取车 | 到店人工核验的行销方案未通过时，门店在取车流程内更换行销方案或改为不使用优惠，系统按取车时命中的方案版本重新试算订单总额和应收尾款，并生成事件级增量快照。 |
| 取消费 | 系统 | 取消 | 未取车前标准取消产生，客户责任时收取。 |
| 爽约费 | 系统 | 取消/爽约 | 逾期未取并由系统自动或门店手动处理爽约后产生。 |
| 拒绝履约费用 | 系统/人工 | 取车 | 取车资质不通过且客户责任时产生。 |
| 关闭退款 | 系统 | 关闭 | 关闭订单时按全额退款记录，不向客户收取费用。 |
| 改派差价 | 系统/人工 | 取车/还车 | 改派后新车辆价格高于原车辆，选择多退少补时生成。 |
| 改派退费 | 系统/人工 | 取车/还车 | 改派后新车辆价格低于原车辆，选择多退少补时生成负向费用。 |
| 超时费 | 系统 | 还车 | 实际还车时间超过预计还车时间和宽限期后生成。 |
| 提前还车退费 | 系统 | 还车 | 满足退费规则时生成负向费用。 |
| 超里程费 | 系统 | 还车 | 实际行驶里程超过订单包含里程后生成。 |
| 油/电补差 | 系统 | 还车 | 还车油量/电量低于取车标准时生成。 |
| 异地还车费 | 系统 | 下单/转单/还车 | 取车门店与订单当前还车门店不一致时生成；改还车门市/转单后按新旧门市差额生成；同一门店下不同归还据点不生成该费用。 |
| 车辆清洁费 | 人工 | 还车/追缴 | 车内吸烟、异味、污渍、宠物毛发等场景。 |
| 损伤赔偿 | 人工 | 还车/追缴 | 新增车损、配件损坏、钥匙遗失等场景。 |
| 事故车损费用 | 人工 | 出险/还车/追缴 | 事故登记后产生的定损、维修、折旧、停运等客户责任费用。 |
| ETC/停车费 | 人工 | 还车/追缴 | 账单已明确时创建费用。 |
| 违章追缴 | 人工 | 追缴 | 交管数据出账后创建追缴费用。 |
| 其他费用 | 人工 | 还车/追缴 | 需填写明确说明和凭证。 |

## 5. 资金记录类型

| 资金类型 | 字段值 | 金额方向 | 说明 |
| :--- | :--- | :--- | :--- |
| 预授权冻结 | `pre_auth_freeze` | 0 实收 | 下单阶段冻结客户信用额度。 |
| 预授权释放 | `pre_auth_release` | 0 实收 | 履约担保模式下，取车尾款结清并确认发车后释放冻结额度。 |
| 预授权扣款 | `pre_auth_capture` | 正向实收 | 转租金模式下，将预授权扣款转为租金实收。 |
| 订单收款 | `payment` | 正向实收 | 取车尾款、还车补缴、追缴收款等。 |
| 订单退款 | `refund` | 负向实收 | 还车退款、取消退款、关闭订单全额退款等。 |
| 费用减免 | `waiver` | 抵扣应收 | 授权减免，不产生外部支付流水。 |
| 企业月结 | `monthly_bill` | 记账 | 企业月结订单进入企业账单，本单不向客户收款。 |

### 5.1 支付渠道
| 渠道类型 | 渠道 |
| :--- | :--- |
| 线上支付 | 街口支付、LINE Pay、信用卡；下单预授权押金仅使用线上预授权。 |
| 线下收款 | 现金、POS 刷卡、银行转账；仅用于取车尾款、还车补缴、追缴等实际收款，不用于下单预授权押金。 |
| 内部记账 | 费用减免、企业月结、预授权释放。 |

## 6. 状态联动规则

### 6.1 支付状态

| 支付状态 | 字段值 | 触发条件 | 展示位置 |
| :--- | :--- | :--- | :--- |
| 未支付 | `unpaid` | 未产生有效收款或预授权。 | 金额列、详情页。 |
| 已收预授权 | `deposit_paid` | 下单阶段预授权押金冻结成功。 | 金额列、详情页。 |
| 企业月结 | `monthly_billing` | 企业月结订单不进入客户支付链路，最终费用按还车结算或终止费用结果挂账。 | 金额列、详情页。 |
| 部分支付 | `partial_paid` | 已有收款但订单当前应收未结清。 | 金额列、详情页。 |
| 已结清 | `paid` | 当前阶段应收款项已平账。 | 金额列、详情页。 |
| 待补缴 | `payment_due` | 还车结算后仍有应收余额。 | 金额列、详情页。 |
| 待退款 | `refund_pending` | 还车结算后存在应退余额。 | 金额列、详情页。 |
| 已退款 | `refunded` | 退款已完成。 | 金额列、详情页。 |

### 6.2 主状态联动

| 资金结果 | 主状态处理 |
| :--- | :--- |
| 下单暂不收款 | 主状态为 `pending_payment`。 |
| 待支付补收预授权成功 | 主状态从 `pending_payment` 更新为 `reserved`，支付状态更新为 `deposit_paid`。 |
| 下单预授权成功 | 主状态为 `reserved`。 |
| 下单企业月结 | 主状态为 `reserved`，支付状态为 `monthly_billing`。 |
| 取车尾款未结清 | 主状态保持 `inspecting`，禁止确认发车。 |
| 取车尾款结清并确认发车 | 主状态进入 `renting`。 |
| 还车 `balanceDue > 0` | 主状态进入 `payment_due`。 |
| 还车 `balanceDue < 0` | 主状态进入 `refund_pending`。 |
| 还车 `balanceDue = 0` 且确认还车 | 主状态进入 `completed`。 |
| 企业月结还车确认 | 生成 `monthly_bill` 挂账记录，主状态进入 `completed`。 |
| 完成后新增追缴费用 | 主状态保持 `completed`，追缴状态独立变化。 |

### 6.3 追缴状态

| 追缴状态 | 字段值 | 说明 |
| :--- | :--- | :--- |
| 追缴待通知 | `pending_notify` | 客户自付订单追缴费用已创建，尚未通知客户。 |
| 追缴已通知 | `notified` | 客户自付订单已通过 App、短信或人工方式通知客户。 |
| 追缴待支付 | `payment_due` | 客户自付订单已收到通知但尚未支付。 |
| 月结已挂账 | `monthly_billing` | 企业月结订单完成后产生追缴费用，系统已生成企业月结挂账。 |
| 追缴已结清 | `paid` | 客户自付订单追缴费用已完成收款或减免。 |

## 7. 计算规则

### 7.1 下单金额

```text
个人订单总额 = 标准租金 + 增值服务费用 - 行销优惠方案抵扣 - 积分抵扣 - 优惠券抵扣
企业订单总额 = 标准租金 + 增值服务费用 - 企业资格行销优惠方案抵扣
固定额度模式：本次预授权金额 = 固定预授权金额
租金比例模式：本次预授权金额 = 订单租金 × 预授权比例，预授权比例不超过 100%
```

* 原型按租金比例模式 30% 模拟预授权金额。
* 租金比例模式下，预授权金额不得超过订单租金。
* 车组押金标准单独展示，用于门店了解车辆担保要求，不直接等同于本次预授权金额。
* 企业月结订单不计算本次预授权金额。
* 待支付订单补收预授权押金时，仍按订单规则快照中的预授权额度模式计算本次预授权金额。预授权成功写入 `pre_auth_freeze` 资金记录；预授权失败、取消或关闭弹窗不写入有效资金记录。
* 待支付订单在资源保留时长内占用车组库存；保留时长到期仍未完成预授权押金确认时，订单取消并释放车组库存，不生成费用项。
* 订单创建成功后保存订单级基线计费快照和订单规则快照；不改变用户商业承诺的履约分配动作继续读取订单级基线快照。
* 涉及租期、车组、行销方案、价格规则、税费口径等商业条件变化的事件必须生成事件级增量快照，并按事件发生时命中的版本结算差额，不覆盖原订单快照。
* 订单计费快照包含标准价版本号、行销方案版本号、租赁规则版本号、车组标准价、行销优惠方案、增值服务价格、里程规则和创建时适用的客户/企业资格。个人订单同时包含优惠券和积分抵扣；企业订单不包含优惠券和积分抵扣。配置或行销方案后续调整不反向改写已创建订单。
* 费用中心展示订单号、客户、企业、取还门店、车组、车型、车牌和来源渠道时，读取订单历史信息快照，不使用当前主数据名称覆盖历史订单展示。
* 系统费用项需要记录费用来源快照，包括订单创建快照、续租快照、改派快照、还车结算快照或完成后追缴快照。费用管理弹窗查看费用明细时，可识别该费用按哪个版本和哪个事件生成。

### 7.2 取车尾款

取车尾款计算前必须完成身份与行销方案资格核验。需要到店人工核验的行销方案核验通过时，取车尾款继续读取订单创建时的行销方案快照；核验不通过时，原行销方案不得继续生效，门店必须更换行销方案或改为不使用优惠。

更换行销方案属于商业条件变化，费用中心生成“取车行销方案变更差额”费用来源记录，并关联取车事件级增量快照。该快照保存变更前方案、变更后方案、方案版本号、核验结果、证明文件、变更原因、操作人、操作时间和重算后的订单总额。客户自付订单按重算后的订单总额计算应收尾款；企业月结订单不向客户收款，重算后的最终应收进入企业月结账单。

```text
履约担保模式：
取车应收尾款 = 订单总额 - 已实际收款金额

转租金模式：
取车应收尾款 = 订单总额 - 已实际收款金额 - 预授权转租金扣款金额
```

* 预授权冻结不计入已实际收款金额。
* 履约担保模式下，预授权冻结只作为取车前担保，不抵扣租金；取车应收尾款结清并确认发车后，系统释放预授权。
* 转租金模式下，预授权扣款计入已处理租金，扣款成功后减少取车应收尾款。
* 非月结企业预订按客户自付订单计算取车应收尾款，由客户本人完成收款。
* 企业月结订单取车阶段不计算客户应收尾款，不生成客户收款记录，页面展示预计月结金额。
* 客户自付订单取车应收尾款大于 0 时必须完成收款。
* 取车应收尾款为 0 或订单结算方式为企业月结时，才允许进入验车发车。

### 7.3 还车结算

```text
客户自付还车差额 returnFeeTotal = 还车阶段正向费用合计 - 还车阶段退费项合计
还车已处理金额 returnTransactionTotal = 还车阶段已收款金额 + 还车阶段减免金额 - 还车阶段已退款金额
最终待结金额 balanceDue = returnFeeTotal - returnTransactionTotal

企业月结挂账金额 monthlyBillAmount = max(0, 原订单租金 + 续租费用 + 还车阶段正向费用 - 还车阶段退费项 - 减免金额)
```

客户自付订单取车阶段已补足订单租金，还车阶段不重复把取车已收租金计入 `returnTransactionTotal`。取车已收租金和预授权转租金记录仅用于退款渠道追溯。

| 结果 | 处理 |
| :--- | :--- |
| `balanceDue > 0` | 客户补缴，订单进入 `payment_due`。 |
| `balanceDue < 0` | 门店退款，订单进入 `refund_pending`。 |
| `balanceDue = 0` | 账单已平，允许确认还车。 |
| 企业月结 | 不进入客户补缴和门店退款，确认后按 `monthlyBillAmount` 生成企业月结挂账，订单进入 `completed`。 |

### 7.4 完成后追缴

```text
追缴待收金额 = 追缴费用合计 - 追缴已收金额 - 追缴减免金额
```

追缴费用不纳入订单主状态计算，不影响已完成订单的完成时间和完成状态。

企业月结订单产生完成后追缴时，费用项状态为 `monthly_billing`，系统同步生成 `monthly_bill` 交易记录，不展示客户收款、退款、补缴或原路退款入口。

## 8. 费用管理弹窗

### 8.1 页面结构
费用管理弹窗包含以下区域：
* 顶部：订单费用台账标题、订单结算方式说明、添加费用按钮。
* 汇总区：展示费用合计、客户已收或企业挂账、客户待收、已减免金额。
* 筛选区：费用阶段筛选、费用状态筛选。
* 费用列表：采用表格化布局，展示费用阶段、费用来源、费用名称、费用说明、创建时间、关联交易数、应收金额、已处理金额、待处理金额、费用状态和可执行操作。
* 交易展开区：点击费用行后展开该费用项关联的资金记录，包含支付渠道、交易类型、交易流水号、交易时间和金额。
* 底部操作区：展示当前筛选数量、全部费用数量、操作日志入口和关闭入口。
* 审计日志：展示费用相关操作记录。

### 8.2 筛选项

| 筛选类型 | 选项 |
| :--- | :--- |
| 阶段筛选 | 全部费用、租期费用、追缴费用。 |
| 状态筛选 | 全部、待收款、部分收款、企业月结、已结清、已退款、已减免、已释放、已取消。 |

### 8.3 操作按钮

| 按钮 | 显示条件 | 动作 |
| :--- | :--- | :--- |
| 添加费用 | 有费用新增权限。 | 新增人工费用。 |
| 通知客户 | 客户自付订单的追缴费用且 `notifyStatus=pending`。 | 发送客户追缴通知，通知后 `notifyStatus` 更新为 `notified`。 |
| 月结挂账 | 企业月结订单费用状态为 `monthly_billing`。 | 页面展示“月结挂账”标识，不展示客户通知、收款、退款和催收提醒入口。 |
| 收款 | 客户自付订单费用状态为 `pending` 或 `partial`。 | 打开收款弹窗。 |
| 退款 | 客户自付订单已存在可退款交易。 | 打开退款弹窗。 |
| 编辑 | 人工费用且待收款。 | 修改费用金额和说明。 |
| 作废 | 人工费用且待收款。 | 作废费用项，保留作废记录。 |
| 设置提醒 | 客户自付订单未结清费用。 | 设置催收提醒日期。 |
| 操作日志 | 始终显示。 | 展开费用审计日志。 |

### 8.4 收款弹窗
* 展示费用项目、待收金额。
* 支持线上支付和线下收款。
* 线下收款必须支持填写交易流水号和上传凭证。
* 收款金额不得大于当前待收金额。
* 收款成功后写入交易记录，费用状态按金额更新为 `partial` 或 `paid`。

### 8.5 退款弹窗
* 退款必须选择原交易。
* 可退款原交易按费用项关联字段匹配，兼容 `feeId` 单费用关联和 `feeIds` 多费用合并收款关联。
* 线上交易按原支付渠道退款。
* 线下交易支持现金退款或银行转账。
* 退款必须填写退款原因。
* 退款成功后写入交易记录，并更新费用状态。

### 8.6 减免弹窗
* 减免必须选择减免原因。
* 减免金额不得大于当前待收金额。
* 减免成功后写入 `waiver` 交易记录和审计日志。
* 超过门店授权额度时，需要具备权限的店长或管理员执行减免。

### 8.7 发票记录
* 发票记录展示已开票和待开票状态。
* 发票必须关联具体费用项。
* 已退款或已取消费用不允许重复开票。
* 发票状态不改变订单主状态。

### 8.8 客户自付与企业月结展示差异

| 订单类型 | 费用状态 | 页面展示 | 可执行动作 |
| :--- | :--- | :--- | :--- |
| 客户自付订单 | `pending`、`partial` | 展示待收金额、收款入口、提醒入口。 | 收款、设置提醒、追缴通知。 |
| 客户自付订单 | `paid` | 展示已处理金额和已结清状态。 | 存在可退款交易时允许退款。 |
| 客户自付订单 | `waived` | 展示已减免金额和已减免状态。 | 不展示收款和提醒入口。 |
| 企业月结订单 | `monthly_billing` | 展示企业挂账金额、客户应收 0、月结挂账标识。 | 不展示客户通知、收款、退款、提醒入口。 |
| 企业月结订单 | 完成后追缴 | 费用项状态为 `monthly_billing`，同步生成 `monthly_bill` 资金记录。 | 不改变订单主状态，不进入客户支付链路。 |

## 11. 验收标准

| 场景 | 验收标准 |
| :--- | :--- |
| 下单预授权 | 新建订单默认展示预授权押金，创建成功后支付状态为 `deposit_paid`。 |
| 待支付 | 暂不收款订单进入 `pending_payment`，列表展示“支付”。 |
| 企业月结 | 企业月结订单创建后支付状态为 `monthly_billing`，还车结算、终止费用确认或完成后追缴确认后生成 `monthly_bill` 记录，列表和详情不展示支付、通知客户、收款、退款、催收提醒入口。 |
| 取车履约担保 | 取车页选择履约担保时，预授权不抵扣租金；尾款结清并确认发车后生成预授权释放记录。 |
| 取车转租金 | 取车页选择转租金时，预授权扣款计入租金实收，并减少取车应收尾款。 |
| 取车尾款 | 取车页展示应收尾款，尾款未结清不能确认发车。 |
| 非月结企业预订 | 非月结企业预订使用企业资格可用行销方案或企业优惠资格，支付、预授权、尾款、还车补缴和退款规则与个人预订一致。 |
| 还车补缴 | 还车结算 `balanceDue > 0` 时展示收款入口，未收款时订单进入 `payment_due`。 |
| 还车退款 | 还车结算 `balanceDue < 0` 时展示退款入口，未退款时订单进入 `refund_pending`。 |
| 月结还车 | 企业月结订单还车结算展示企业挂账金额，确认后订单进入 `completed`，所有费用进入企业月结账单。 |
| 计费快照与规则快照 | 已创建订单的费用计算使用订单计费快照、规则快照和对应版本号，不受后续价格、行销方案或规则页面修改影响。 |
| 还车完成 | `balanceDue = 0` 后确认还车，订单进入 `completed`。 |
| 新增人工费用 | 人工费用新增后展示在费用管理列表，并写入审计日志。 |
| 收款 | 收款成功后生成交易记录，并更新费用状态和待收金额。 |
| 退款 | 退款必须选择原交易并填写原因，费用项通过 `feeId` 或 `feeIds` 均能匹配可退款原交易，成功后生成退款记录。 |
| 减免 | 减免必须填写原因，成功后生成减免记录并影响待收金额。 |
| 追缴 | 已完成订单新增追缴费用后，主状态保持 `completed`；客户自付订单金额列展示追缴状态，企业月结订单费用项展示“企业月结”并生成 `monthly_bill`。 |
| 通知客户 | 客户自付订单追缴待通知费用点击“通知客户”后，追缴状态更新为已通知；企业月结订单不展示通知客户入口。 |
| 操作日志 | 所有金额变动均能在费用管理弹窗查看审计日志。 |
