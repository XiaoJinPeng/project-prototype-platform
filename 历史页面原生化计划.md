# 历史页面原生化计划

## 1. 目标

历史页面原生化不是重新设计页面，而是在保持当前视觉、业务字段、模拟数据和交互不变的前提下，逐步使用工程统一的路由、外壳、公共组件与 Composables。模拟数据默认继续保留在各页面内部。

## 2. 当前基线

- 所有营运端与企业端页面已接入统一路由和 `AppShell`。
- 页面内旧菜单、旧顶栏、`legacy: true` 和非 scoped 样式白名单均已清空。
- 51 个迁移页面已纳入严格模板审计，关键路径已纳入 Playwright 回归。
- 历史页面不强制一次性重写，优先处理低风险重复结构。

## 3. 原生化层级

| 层级 | 标准 |
| --- | --- |
| N1 工程接入 | 使用统一路由、外壳、内容区滚动和 scoped 样式，不含旧 HTML 外壳 |
| N2 结构标准化 | 按风险使用 `PageHeader` 等公共组件，清理迁移冗余并规范页面结构；模拟数据仍保留在页面内部 |
| N3 模块化 | 大型页面按业务区块拆分子组件和 Composables，页面负责组合；模拟数据暂不做跨页面治理 |

目前全部历史页面已达到 N1；N2、N3 按以下批次推进。

## 8. 复杂页实施选择

复杂页面开始前需要确认：

1. 是否允许只调整内部组件边界和文件结构，页面最终视觉与交互必须完全不变。
2. 模拟数据暂不升级为共享状态，后续页面继续在各自页面内部维护。
3. 订单详情优先整体拆分，还是先拆弹窗和费用中心两个最重模块。
4. 是否接受拆分期间按页面逐批验收，而不是一次性处理全部大型页面。

## 9. 验收要求

- 每批必须通过生产构建、迁移审计、样式审计、相关路由和关键交互测试。
- 原型与 Vue 的弹窗、抽屉、按钮和业务字段数量不得减少。
- 不因组件化改变历史页面布局；发现视觉变化时优先恢复，不扩大重构。
- 新页面直接按 N2 标准开发，不再产生新的历史兼容页面。
