## 1. 文档概述

### 1.1 背景

租赁业务规则模块汇集门市租车全流程的核心控制点。不同于基础数据配置（如车型、价格），本模块定义的规则直接影响订单的流转逻辑、计费公式、资金处理和风控策略。

### 1.2 范围

本文件仅聚焦“门市租车”业务场景。本期门市租车规则按全局配置生效，不区分城市、门店和渠道；后续如需要门店差异化，再扩展规则适用范围和生效维度。

本规则中的支付、预授权、尾款、退款配置仅适用于客户自付订单，包括个人订单和非月结企业订单。企业月结订单不按本规则发起客户支付、预授权、取车尾款或退款，订单最终应收、终止有责费用和完成后追缴统一进入企业月结账单。

本期下单阶段的预授权押金统一采用线上预授权，不支持现金押金、线下押金或仅登记线下凭证。取车尾款、还车补缴、追缴等实际收款可按租户支付配置使用线上或线下收款方式。

### 1.3 默认值说明

租赁业务规则页面中的所有设置项都必须具备默认值。本期原型中展示的配置值即为默认值和当前设定值；新环境初始化时按默认值生成门市租车全局规则。配置保存后仅影响后续新建订单，已创建订单继续按订单生成时的规则快照执行。

### 1.4 规则版本记录

门市租车规则每次保存后生成新的全局规则版本，系统保留历史版本并禁止覆盖修改。订单创建成功时记录当时命中的 `rentalRuleVersionId`，后续订单流转按订单规则快照执行。

| 版本字段 | 规则 |
| :--- | :--- |
| 规则版本号 | 系统生成唯一 `rentalRuleVersionId`，用于订单规则快照追溯。 |
| 规则范围 | 本期固定为门市租车全局规则，不区分城市、门店和渠道。 |
| 交易授权 | 记录支付方式、预授权押金、取车资金处理口径、待支付资源保留规则。 |
| 流程管控 | 记录操作锁、取消、爽约、续租、超时还车、提前还车、还车结算等规则。 |
| 调度规则 | 记录车辆锁定、订单间隔、派车权重和调度评分参数。 |
| 操作信息 | 记录保存人、保存时间、变更说明和版本状态。 |

企业月结订单同样保存 `rentalRuleVersionId` 和完整规则快照；其中客户支付、预授权、尾款、补缴、退款类规则不执行，最终应收和追缴费用统一进入企业月结账单。

租赁业务规则页需要提供历史版本入口。点击当前规则版本或“历史版本”时，打开规则版本详情，版本详情沿用租赁业务规则页的分组结构，并以简化只读方式展示该版本内的完整设定值，包括交易授权、待支付资源保留、操作锁、取消、爽约、续租、超时还车、提前还车、还车结算和调度规则。历史版本只读展示，不支持在历史版本弹窗内修改；订单详情页点击 `rentalRuleVersionId` 时，展示订单创建时命中的规则版本详情。

### 1.5 目标

1. **交易风控**：配置客户自付订单的支付方式、预授权额度和取车资金处理口径。
2. **计费引擎**：定义时租转日租、异地还车、超时还车等核心算费规则。
3. **流程管控**：规范履约操作锁、未取车爽约、取消、提前取还车和续租处理。
4. **智能调度**：通过车辆锁定、订单间隔和派车权重控制车辆资源分配。

## 2. 核心规则板块

### 2.1 交易与授权

交易与授权用于配置客户自付订单的支付方式、预授权押金和取车资金处理方式。

#### 2.1.1 支付方式开关

| 配置项 | 规则 |
| :--- | :--- |
| 线上交易 | 支持 LINE Pay、街口支付、绿界金流等线上支付方式独立开启或关闭。 |
| 线下交易 | 支持现金、POS 刷卡、转账等线下支付方式独立开启或关闭。 |
| 企业月结 | 企业月结订单不读取客户支付方式开关，不发起客户支付。 |

#### 2.1.2 预授权押金策略

| 配置项 | 规则 |
| :--- | :--- |
| 固定额度模式 | 每单按固定金额冻结预授权押金。 |
| 租金比例模式 | 按订单租金金额的一定比例冻结预授权押金；本期比例不允许超过 100%，因此预授权金额不超过订单租金。 |
| 企业月结 | 企业月结订单不发起预授权，不生成预授权资金记录。 |

#### 2.1.3 取车资金处理口径

预授权处理支持同时启用以下口径。启用后，取车办理页面可按订单实际场景选择对应方式；仅启用一个口径时，系统默认使用该口径。

| 口径 | 规则 |
| :--- | :--- |
| 履约担保 | 下单阶段冻结预授权押金；取车时客户补足租金尾款；确认发车后释放预授权押金。 |
| 预授权转租金 | 下单阶段冻结预授权押金；取车时将预授权扣款转为租金实收，并抵扣取车尾款。 |

#### 2.1.4 待支付资源保留

待支付资源保留用于控制 `pending_payment` 订单占用车组库存的时长。该规则仅适用于客户自付订单，企业月结订单创建后直接进入 `reserved`，不进入待支付。

| 配置项 | 规则 |
| :--- | :--- |
| 启用待支付资源保留 | 开启后，暂不收款创建的待支付订单在保留时长内占用车组库存。 |
| 保留时长 | 默认 30 分钟。保留时长内完成预授权押金确认后，订单进入 `reserved` 并继续保留库存。 |
| 超时处理 | 保留时长到期仍未完成预授权押金确认时，订单自动进入 `cancelled`，释放车组库存和派车关系，不生成取消费。 |
| 派车限制 | `pending_payment` 订单不参与自动预分配、硬锁定和取车。 |
| 预授权渠道 | 待支付订单后续点击“支付”时仅支持线上预授权，不支持线下押金登记。 |

### 2.2 异地还车规则

异地还车规则用于配置门市租车跨门店或跨城市还车的基础费用。门市租车的异地还车费以取车门市和订单当前还车门市判断，不以车辆所在据点或归还据点判断。同一门市下不同据点取还或停放，不触发异地还车费。

| 配置项 | 规则 |
| :--- | :--- |
| 异地还车费 | 配置全局基础费用，单位为元/次。 |
| 计费判断口径 | 取车门市与订单当前还车门市不一致时触发；同一门市下不同归还据点不触发。 |
| 线路差异化 | 具体线路的差异化定价可在增值服务“异地还车”中覆盖。 |
| 改还车门市 | 转单确认前检查原还车门市后续预约占用；存在冲突时先快捷改派后续订单。转单后订单当前还车门市更新为新门市，并按取车门市到新还车门市重新计算异地还车费差额。 |

### 2.3 调度与派车

调度与派车规则用于控制车辆资源的占用、锁定和智能派车排序。

| 配置项 | 规则 |
| :--- | :--- |
| 调度锁定 | 订单开始前 N 小时锁定具体车辆；每笔订单到达硬锁定时间时，系统必须执行一次自动分派或重新校验后锁定。 |
| 订单间隔 | 同一辆车前后两单之间必须预留 N 分钟，用于清洁、检查和整备。 |
| 基准日均里程 | 用于智能调度在订单分配后估算车辆模拟新增里程，公式为订单租期 × 基准日均里程；该值不用于同车组订单排序权重。 |
| 里程权重 | 用于计算车辆评分中的已行驶里程 / 预计退役里程占比。 |
| 营收权重 | 用于计算车辆评分中的当月预计营收 / 车辆月成本占比。 |
| 车龄权重 | 用于计算车辆评分中的车辆年龄因子。 |

### 2.4 时租与日租转换

时租与日租转换用于定义不满整日、超过阈值和超时还车场景下的基础计费方式。

| 配置项 | 规则 |
| :--- | :--- |
| 日租阈值 | 租期超过 N 小时后，按 1 天计费。 |
| 按小时计费 | 不满日租阈值时，按每小时固定金额计费。 |
| 按日租百分比 | 不满日租阈值时，每小时按日租金百分比计费。 |
| 固定打包价 | 不满 1 天时统一收取固定金额。 |
| 最小计费单位 | 超过完整 24 小时后的剩余时长不足 1 小时按 1 小时计费。 |

### 2.5 取车、还车和操作锁规则

取还车规则用于管理履约过程中的时间偏差、未取车处理和履约操作锁。

#### 2.5.1 提前取车

| 配置项 | 规则 |
| :--- | :--- |
| 是否允许提前取车 | 关闭后，未到预计取车时间不得办理取车。 |
| 免费宽限期 | 在预计取车时间前 N 分钟内取车不加收费用，且不改变原预计还车时间。 |
| 超出宽限期 | 超过免费宽限期后办理提前取车，系统按实际起租时间重算租期和租金。 |

#### 2.5.2 履约操作锁

履约操作锁用于配置取车和还车办理过程中的操作接管、窗口关闭和超时释放规则。取车点击“取车”时创建操作锁；还车点击“还车”或“继续结算”时创建操作锁。取车锁和还车锁共用同一套有效期和释放权限。

| 配置项 | 规则 |
| :--- | :--- |
| 启用履约操作锁 | 开启后，点击取车或还车即创建操作锁。点击取车后订单进入 `inspecting`；点击还车时仅创建操作锁，不立即改变主状态，后续按验车和结算提交节点进入原用车状态、`settlement_pending`、`payment_due` 或 `refund_pending`。 |
| 锁有效期 | 操作锁超过配置时长后，原操作人可继续办理，具备权限的人员可释放操作锁。 |
| 关闭取车窗口处理 | 未确认发车时关闭取车窗口，订单固定保留 `inspecting`，取车草稿和操作锁继续保留。 |
| 关闭还车窗口处理 | 验车未提交时订单保持原用车状态；验车已提交后按当前待结金额保持 `settlement_pending`、`payment_due` 或 `refund_pending`，企业月结订单保持 `settlement_pending`。 |
| 释放锁权限 | 释放操作锁由角色权限配置控制，本页不选择角色；取车锁释放后取车草稿作废，订单先回到进入取车前状态，再按当前时间重新判定 `reserved`、`pickup_overdue` 或 `no_show`；还车锁释放后不回滚已提交验车、结算快照和待结状态。 |
| 确认发车 | 身份核验、验车、尾款或月结校验完成后，订单进入 `renting`。 |

#### 2.5.3 提前还车

| 配置项 | 规则 |
| :--- | :--- |
| 是否允许提前还车 | 关闭后，未到预计还车时间不得直接办理提前还车，需要门店授权处理。 |
| 退款规则 | 支持仅退还剩余完整天数租金或不退还任何费用。 |
| 不满整天 | 不满 1 天及其以内的剩余租期不退租金。 |

#### 2.5.4 未取车爽约

未取车爽约用于配置客户超过保留时长仍未到店取车后的资源释放、状态处理和违约金规则。

| 配置项 | 规则 |
| :--- | :--- |
| 保留车辆时长 | 当前时间超过预计取车时间 N 小时后，客户仍未取车，订单可判定为爽约。 |
| 自动释放资源 | 开启后，达到爽约条件时释放车辆、车组库存和派车关系，订单进入 `no_show`。 |
| 爽约违约金 | 支持固定金额或按预授权比例扣款；企业月结订单的有责费用进入企业月结账单。 |

### 2.6 取消与超时还车规则

取消与超时还车规则用于处理客户在取车前取消、逾期还车和超时费用。

#### 2.6.1 免责取消

| 配置项 | 规则 |
| :--- | :--- |
| 下单后限时取消 | 下单后 N 分钟或 N 小时内取消可免责。 |
| 取车前提前取消 | 预计取车时间前 N 小时以上取消可免责。 |
| 免责取消退款 | 客户自付订单按配置全额退款或扣除手续费后退款；企业月结订单取消费为 0 时仅释放资源。 |

#### 2.6.2 有责取消

| 配置项 | 规则 |
| :--- | :--- |
| 固定金额 | 不满足免责条件时，按固定金额收取取消费。 |
| 预授权比例 | 不满足免责条件时，按订单预授权押金的一定比例收取取消费。 |
| 阶梯扣费 | 可按距离取车时间配置不同扣费档位；选择固定金额时配置各档固定金额，选择预授权比例时配置各档预授权扣取比例。 |
| 企业月结 | 企业月结订单产生有责取消费时，费用进入企业月结账单，不向员工个人收款。 |

#### 2.6.3 超时还车

| 配置项 | 规则 |
| :--- | :--- |
| 超时免费宽限 | 预计还车时间后 N 分钟内还车，不收取超时费。 |
| 超时状态 | 当前时间超过预计还车时间加免费宽限期后，订单进入 `overdue`。 |
| 超时基础费用 | 超过宽限期后，按“时租与日租转换”规则计算超时用车费用。 |
| 超时罚款 | 可在超时基础费用之外加收固定金额或按超时费比例计算的罚款。 |
| 超时费封顶 | 可限制单次超时基础费用的最高金额；罚款是否计入封顶按配置执行。 |
| 还车后状态 | 超时订单提交还车验车后进入 `settlement_pending`，超时费用进入最终账单。 |

### 2.7 续租/延长用车规则

续租规则用于控制履约中延长用车、逾期补办续租和续租费用处理。

| 配置项 | 规则 |
| :--- | :--- |
| 允许延长用车 | `renting`、`renewing` 状态可发起延长用车；打开或关闭弹窗不改变订单状态。 |
| 允许补办续租 | `overdue` 状态可在权限校验通过后补办续租。 |
| 库存冲突校验 | 确认延长前必须校验当前车辆在延长时段是否已有后续订单；存在冲突时需改派后续订单或拒绝延长。 |
| 费用计算 | 续租费用按订单计费快照、车组标准价、行销优惠方案快照和新增租期计算，逾期补办续租成功后，原逾期时段按续租费用处理。 |
| 客户自付结算 | 支持立即收款或还车时结算；逾期补办、长时长、高金额或高风险客户必须立即收款。 |
| 企业月结结算 | 企业月结订单不发起续租收款，续租费用随还车最终应收进入企业月结账单。 |
| 状态处理 | 确认延长成功后，`renting` 或 `overdue` 更新为 `renewing`；`renewing` 再次延长后仍保持 `renewing`。 |
| 延长限制 | 本期不限制单次最大延长天数；只要库存冲突、费用和权限校验通过，即可确认延长。 |

## 4. 业务逻辑流程

### 4.1 预授权冻结流程

1. 客户自付订单下单租用“坦克300”，日租 ¥500，租期 3 天，租金合计 ¥1500。
2. 系统命中“租金比例模式 30%”。
3. 预授权金额 = ¥1500 × 30% = ¥450。
4. 下单支付环节通过线上预授权渠道发起 ¥450 预授权冻结。
5. 取车时按已启用的资金处理口径执行履约担保或预授权转租金。

### 4.2 爽约处理流程

1. 客户预计 10:00 取车。
2. 系统配置保留车辆时长 2 小时。
3. 12:00 后客户仍未办理取车，订单满足爽约条件。
4. 开启自动释放资源时，系统释放车辆和库存占用，订单进入 `no_show`。
5. 自动释放资源关闭或自动任务未执行前，门店可手动标记爽约。
6. 存在爽约违约金时，客户自付订单按资金规则处理扣款；企业月结订单生成月结费用。

### 4.3 超时还车计算

1. 客户预计 10:00 还车。
2. 系统配置超时免费宽限 30 分钟、超时罚款 ¥100、日租阈值 5 小时。
3. 10:25 还车时，仍在宽限期内，超时费用为 ¥0。
4. 10:40 还车时，已超过宽限期，剩余时长不足 1 小时按 1 小时计算超时基础费用，并叠加超时罚款。
5. 超时订单提交还车验车后进入 `settlement_pending`，超时费用参与最终结算。

### 4.4 续租处理流程

1. 订单处于 `renting`、`renewing` 或 `overdue`。
2. 门店打开延长用车弹窗，系统试算新增租期、续租费用和后续车辆占用冲突。
3. 客户自付订单按规则选择立即收款或还车时结算；企业月结订单直接生成月结费用项。
4. 确认延长成功后，更新预计还车时间、车辆占用时段和续租费用，订单进入或保持 `renewing`。
