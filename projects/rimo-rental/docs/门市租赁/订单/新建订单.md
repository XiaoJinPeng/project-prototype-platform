## 1. 文档概述

### 1.1 背景
新建订单是门市租车业务的订单创建入口，用于支持电话预订、柜台散客、企业客户代订等门店高频场景。门店员工需要在一个页面内完成客户锁定、行程录入、车组选择、增值服务选择、优惠抵扣、费用确认和收款处理。

### 1.2 目标
* **一站式创建**：在单页内完成客户、行程、车组、服务、优惠、费用和支付信息录入。
* **实时试算**：取还时间、车组、增值服务、行销优惠方案、优惠券、积分等信息变更后，费用明细实时刷新；企业身份订单不支持优惠券和积分。
* **库存校验**：按取还时间、取车门市及门市下可服务据点校验车组库存与增值服务库存。
* **状态明确**：订单创建成功后明确生成订单主状态、支付状态、派车状态和库存锁定结果。
* **门店高效操作**：页面布局采用左侧录入、右侧费用吸顶的结构，保证费用信息始终可见。

### 1.3 页面入口
* 后台侧边栏：订单中心 -> 新建门市订单
* 门市订单列表：点击“新建门市订单”
* 对应原型文件：`new_order.html`

## 2. 页面结构

页面由以下区域组成：

1. **客户信息**
   * 搜索并锁定客户。
   * 展示客户姓名、手机号、会员等级、剩余积分。
   * 支持个人预订和企业预订切换。

2. **用车信息**
   * 选择取车门市、还车门市。
   * 选择取车时间、还车时间。
   * 系统自动计算租期。

3. **车组选择**
   * 展示可售车组、车型示例、日租价、可用库存和车组押金标准。
   * 库存不足的车组置灰，不允许选择。

4. **增值服务**
   * 展示保险、设备、服务类增值项。
   * 支持勾选和数量调整。
   * 设备类服务受取车门市库存限制。

5. **优惠与抵扣**
   * 个人预订支持行销优惠方案、客户优惠券、兑换码、积分抵扣。
   * 企业预订仅支持企业资格可用行销优惠方案，不支持优惠券和积分。

6. **费用明细**
   * 实时展示租金、增值服务、优惠抵扣、订单总额、车组押金标准和本次预授权金额；企业月结订单展示月结记账金额，不展示本次预授权金额。

7. **支付/创建弹窗**
   * 本期支持线上预授权押金、暂不收款、企业月结。
   * 全额支付保留为后续能力，本期页面不展示全额支付入口。
   * 下单预授权押金仅支持线上预授权；取车尾款、还车补缴等实际收款可按租户支付配置支持在线支付和线下收款。
   * 创建成功后展示订单号、主状态、支付状态、派车状态。

## 5. 业务规则

### 5.1 默认值
* 取车时间默认次日 09:00。
* 还车时间默认取车时间 + 1 天。
* 预订类型默认个人预订。
* 收款模式默认预授权押金。
* 支付渠道默认在线支付。

### 5.2 租期计算
* 还车时间必须晚于取车时间。
* 租期按自然小时差计算，再根据门市租车计费规则转换为计费金额。
* 门市租车起租 24 小时，租期不足 24 小时按 24 小时计费。
* 超过完整 24 小时后，剩余时长不足 1 小时按 1 小时计；每个计费小时按车组标准价 10% 计费。
* 剩余计费小时达到第 5 小时起，按 1 天计费。
* 跨平假日订单按每一轮 24 小时的起始时间判断行销优惠方案适用平日折扣或假日折扣。超过完整 24 小时后的剩余时长视为新一轮 24 小时的开始，按该开始时间判断平日/假日。
* 调整取车时间或还车时间后，租期、租金、增值服务费用和订单总额实时刷新。

### 5.3 库存校验
* 车组库存按取车门市、门市下可服务据点、取车时间、还车时间和车组计算。
* 可用库存为 0 时，车组卡片置灰，不允许选择。
* 订单创建成功后锁定该车组在对应时间段的库存。
* 新建订单阶段只锁定车组库存，不锁定具体车牌。
* 门市租车订单客户面向取车门市和还车门市，不直接选择停车场据点。车辆仍挂在门市下的具体据点，系统按取车门市下具备门市租车服务能力的据点汇总可用库存。
* 24 小时自助租车订单需要客户选择具体自助取车据点，并允许客户选择同门市下已配置为允许还车且具备自助服务能力的其他据点还车。同门市下不同自助据点还车不产生调度费。
* 门市租车异地还车费只按取车门市和订单当前还车门市判断。同一门市下不同据点取还、备车或停放，不触发异地还车费。订单创建时，订单当前还车门市等于下单选择的还车门市；后续改还车门市/转单会更新订单当前还车门市。
* 创建成功后 `dispatchStatus=pending`。
* `pending_payment` 订单同样占用车组库存，但仅在待支付资源保留时长内占用，不参与具体车辆自动派车和硬锁定。
* 待支付资源保留时长默认 30 分钟，按订单规则快照保存。保留时长内完成预授权押金确认后，订单进入 `reserved`，车组库存继续保留并进入正常派车流程。
* 待支付资源保留时长到期仍未完成预授权押金确认时，系统将订单置为 `cancelled`，释放车组库存和派车关系，不生成取消费。

#### 5.3.1 车组可售库存计算

门市租赁车组库存按“时间窗 + 服务范围 + 车组”实时计算。库存只表达该时间段内还能承接多少笔同车组订单，不等同于已经锁定具体车辆。

```text
目标占用时间窗 = 取车时间 - 订单间隔时间 至 还车时间 + 订单间隔时间

门市租车可售车组库存 =
取车门市下可服务车辆数
- 有效订单占用数
- 内部不可售占用数

24 小时自助可售车组库存 =
取车据点下可服务车辆数
- 有效订单占用数
- 内部不可售占用数
```

| 计算项 | 规则 |
| :--- | :--- |
| 取车门市下可服务车辆数 | 取车门市下具备门市租车服务能力且服务状态启用的据点内，同车行、同车组、车辆启用状态为 `enabled` 的车辆数量。 |
| 取车据点下可服务车辆数 | 24 小时自助租车所选取车据点内，具备 24 小时自助服务能力、服务状态启用、同车行、同车组、车辆启用状态为 `enabled`，且满足车机绑定和可控要求的车辆数量。 |
| 有效订单占用数 | 与目标占用时间窗重叠，且仍占用该车组库存的订单数量。每笔有效订单固定占用 1 个车组库存名额。 |
| 内部不可售占用数 | 与目标占用时间窗重叠，且未被有效订单占用的维修、保养、清洁、事故停用、调度不可售、还车待整备等内部任务数量。 |
| 订单间隔时间 | 读取订单规则快照中的订单间隔时间，默认 60 分钟。 |

有效订单占用只按订单计算一次，不因订单已经预分配、硬锁或人工改派而重复扣减。派车状态用于表达该订单是否已经绑定具体车辆，不单独新增车组库存占用。

#### 5.3.2 订单状态与库存占用

| 订单状态 / 派车状态 | 是否占用车组库存 | 是否占用具体车辆 | 处理规则 |
| :--- | :--- | :--- | :--- |
| `pending_payment` | 是 | 否 | 仅在待支付资源保留时长内占用车组库存，不参与预分配和硬锁。超时未完成预授权时释放库存。 |
| `reserved + pending` | 是 | 否 | 已预约但未派具体车辆，占用 1 个车组库存名额。 |
| `reserved + pre_assigned` | 是 | 是，预分配占用 | 占用 1 个车组库存名额，并记录预分配车辆；预分配不额外增加第二次库存占用。 |
| `reserved + locked` | 是 | 是，硬锁占用 | 占用 1 个车组库存名额，并锁定具体车辆；硬锁不额外增加第二次库存占用。 |
| `manual_reassigned / upgraded` | 是 | 是 | 仍按该订单占用 1 个车组库存名额；原车辆释放，新车辆占用。 |
| `pickup_overdue` | 是 | 按当前派车状态判断 | 未爽约或取消前继续占用库存。 |
| `inspecting` | 是 | 是 | 取车办理中继续占用库存和当前锁定车辆；释放操作锁后按回退后的订单状态重新判断。 |
| `renting / renewing / overdue / accident_processing` | 是 | 是 | 车辆仍在履约或异常处理中，继续占用库存和具体车辆。续租成功后同步延长占用时间窗。 |
| `settlement_pending / payment_due / refund_pending` | 否 | 否，转为内部不可售任务 | 客户已归还车辆，但车辆在还车确认前不得进入新的候选池；不再按订单占用库存，改由车辆 `controlled` 或还车待整备任务减少可服务车辆数。 |
| `completed` | 否 | 否 | 订单库存释放；车辆是否重新可售，按还车确认后的车辆营运状态判断。 |
| `cancelled / no_show / rejected / closed` | 否 | 否 | 终态确认后释放车组库存、具体车辆绑定和派车关系；费用追缴不再占用库存。 |

#### 5.3.3 库存释放与变更

| 场景 | 库存处理 |
| :--- | :--- |
| 待支付资源保留超时 | 订单进入 `cancelled`，释放车组库存和派车关系。 |
| 客户取消、爽约、拒绝履约、关闭订单 | 终态确认后释放车组库存、具体车辆绑定和派车关系。 |
| 续租 / 延长用车 | 校验新时间窗内当前车辆和后续订单无冲突；确认成功后更新车组库存占用时间窗。 |
| 改租期 | 先校验新时间窗库存，校验通过后原子更新占用时间窗；失败时保留原订单时间窗。 |
| 改车组 | 先校验目标车组库存，校验通过后原子释放原车组库存并占用目标车组库存；失败时保留原车组。 |
| 同车组改派 / 换车 | 不改变车组库存占用，只释放原车辆并占用新车辆。 |
| 免费升等 | 原订单仍按实际履约车组占用库存，需记录免费升等原因和事件快照。 |
| 改还车门市 / 改还车据点 | 不改变原取车门市的车组库存占用；如影响后续车辆归位和后续订单冲突，按改派和转单规则处理。 |
| 还车验车提交 | 订单不再作为车组库存占用，但车辆进入还车待整备或内部处理，不得立即回到可售车辆数。 |
| 还车确认完成 | 车辆为 `enabled + idle` 且无内部任务时，重新计入可服务车辆数；车辆为 `controlled` 或 `disabled` 时，不计入可服务车辆数。 |

#### 5.3.4 并发校验

车组库存查询结果只用于页面展示。创建订单、补收待支付预授权、续租、改租期、改车组、改派和硬锁时，后端必须在提交时重新执行库存和时间窗校验，并在同一事务内完成库存占用、释放或更新。

同一车组、同一服务范围、同一目标占用时间窗内，系统不得仅依赖前端查询结果创建订单。并发提交时，以后端最终校验结果为准；库存不足时，创建或变更失败，并提示“当前车组库存不足，请重新选择时间或车组”。

### 5.4 企业预订
* 客户存在关联企业时，可切换为企业预订。
* 企业预订分为企业月结和非月结企业预订。
* 个人预订与企业资格预订可匹配的行销优惠方案不同。页面按预订类型展示当前可用方案。
* 企业月结订单使用企业月结资格和企业资格可用行销方案。下单阶段不发起支付、不发起预授权、不要求客户补交租金，订单创建后 `paymentStatus=monthly_billing`，结算方式记录为 `monthly`；最终应收金额在还车结算或终止费用确认后统一进入企业月结账单。
* 非月结企业预订使用企业资格可用行销方案、企业优惠资格或企业权益身份，费用仍由客户本人支付。下单预授权、取车尾款、还车补缴、退款、追缴等资金流程与个人预订一致，结算方式记录为 `customer_pay`。
* 企业预订不支持优惠券和积分，不展示优惠券、兑换码和积分抵扣入口，不生成优惠券或积分抵扣记录。
* 企业预订订单仍需锁定车组库存。

### 5.5 增值服务
* 保险类服务按租期天数计费，无库存限制。
* 设备类服务按配置计费，并校验取车门市库存。
* 服务类增值项按次或按配置计费。
* 超过最大可购数量或取车门市库存时，不允许提交订单。

### 5.6 优惠计算
费用计算顺序如下：

1. 标准租金 = 按车组标准价、起租 24 小时、超时折算和封顶规则计算得出。
2. 增值服务费用 = 按天服务费用 + 按次服务费用。
3. 行销优惠方案抵扣。系统展示所有可用行销优惠方案，默认选中最终价格最低的方案；订单最终只能使用一个行销优惠方案，不自动叠加多个行销优惠方案。行销优惠方案在车组标准价基础上打折，可区分平日折扣和假日折扣。
   * 需要到店人工核验的行销优惠方案允许在预约时选择，但仅代表客户承诺取车时提供对应证明文件。取车时核验不通过时，门店必须更换行销方案或改为不使用优惠，并按取车时命中的方案版本重新试算尾款。
4. 个人预订先执行积分抵扣。
5. 个人预订再执行优惠券抵扣。
6. 企业预订仅执行企业资格行销优惠方案，不执行积分抵扣和优惠券抵扣。
7. 个人预订最终订单金额 = 标准租金 + 增值服务费用 - 行销优惠方案抵扣 - 积分抵扣 - 优惠券抵扣。
8. 企业预订最终订单金额 = 标准租金 + 增值服务费用 - 企业资格行销优惠方案抵扣。

最终订单金额不得小于 0。

### 5.7 收款模式

| 收款模式 | 创建条件 | 创建后主状态 | 支付状态 |
| :--- | :--- | :--- | :--- |
| 预授权押金 | 本次预授权金额 > 0，金额按租赁业务规则的固定额度或租金比例生成；比例模式不超过 100% | `reserved` | `deposit_paid` |
| 暂不收款 | 允许保留待支付订单时使用 | `pending_payment` | `unpaid` |
| 企业月结 | 预订类型为企业预订，且该企业客户启用月结 | `reserved` | `monthly_billing` |
| 全额支付 | 收款金额 = 订单总额 | `reserved` | `paid` |

全额支付保留为完整业务能力，本期不在新建订单页面启用。

暂不收款创建成功后，订单进入 `pending_payment`。门店可在订单列表或详情页点击“支付”，补收下单阶段预授权押金。预授权押金确认成功后，订单进入 `reserved`，支付状态更新为 `deposit_paid`；支付失败、关闭弹窗或未继续操作时，订单保持 `pending_payment`，支付状态保持 `unpaid`。待支付资源保留时长到期仍未完成预授权押金确认时，订单自动进入 `cancelled`，车组库存和派车关系释放。

### 5.8 预授权
* 客户自付订单默认收取预授权押金，包含个人预订和非月结企业预订。
* 预授权金额严格执行租赁业务规则中的预授权额度模式：固定额度模式按配置金额冻结；租金比例模式按订单租金乘以配置比例冻结。
* 租金比例模式下，本期预授权比例不允许超过 100%，因此预授权金额不超过订单租金。
* 本期下单预授权押金仅支持线上预授权，不支持现金押金、线下押金或仅登记线下凭证。
* 新建订单页同时展示车组押金标准，用于取车阶段押金校验。
* 新建订单阶段完成预授权押金；取车流程中需确认预授权有效并补齐尾款后才能发车。
* 企业月结订单不生成预授权押金，不进入支付渠道选择，不在取车阶段要求客户补齐尾款，最终应收金额进入企业月结账单。

### 5.9 创建成功
订单创建成功后生成以下结果：

| 字段 | 生成规则 |
| :--- | :--- |
| 订单号 | 系统生成唯一订单号。 |
| 主状态 | 根据收款模式生成 `pending_payment` 或 `reserved`。 |
| 支付状态 | 根据收款模式生成 `unpaid`、`deposit_paid`、`monthly_billing` 或 `paid`。 |
| 派车状态 | 默认 `pending`。 |
| 车组库存 | 锁定对应车组在取还时间段内的库存；`pending_payment` 仅在待支付资源保留时长内占用库存。 |
| 预授权金额 | 客户自付订单记录本次按租赁业务规则生成的预授权押金金额；企业月结订单为空。 |
| 结算方式 | 记录 `customer_pay` 或 `monthly`，用于后续取车、还车、费用中心判断是否进入支付链路。 |
| 订单历史信息快照 | 保存下单时的客户、企业、取还门市、门市下可服务据点范围、车组、车型、行程、来源渠道和订单备注等展示信息。后续主数据改名或资料调整不反向影响已创建订单。 |
| 订单规则快照 | 保存创建订单时的门市租车全局规则版本号 `rentalRuleVersionId` 和规则明细，形成订单级基线快照。下单后不改变用户商业承诺的履约分配动作继续读取该基线快照；涉及租期、车组、行销方案、价格规则、税费口径等商业条件变化的事件，必须生成事件级增量快照。 |
| 订单计费快照 | 保存创建订单时的标准价版本号 `priceVersionId`、行销方案版本号 `marketingPlanVersionId`、车组标准价、行销优惠方案、是否需要到店核验、资格名称、证明文件说明、增值服务价格、里程规则和门市租车全局规则。个人预订同时保存优惠券、积分抵扣；企业预订不保存优惠券和积分抵扣。订单创建后的取消、爽约、提前还车退费、还车结算和完成后追缴优先读取订单级基线快照；续租、改还车门市、改车组、改行销方案等改变商业条件的事件读取对应事件级增量快照，并按事件发生时命中的版本结算差额，不覆盖原订单快照。 |
| 快照版本索引 | 记录订单可追溯的版本来源，包括标准价版本、行销方案版本、租赁规则版本、增值服务价格版本、里程规则版本和创建时间。订单详情页可直接查看快照摘要和完整版本明细。 |

订单创建时同时保存主数据 ID 和订单展示快照。ID 用于筛选、统计、权限和后续业务动作；订单列表、订单详情、打印单据、费用核对和历史审计展示时读取订单展示快照中的名称、地址和文本，不直接读取当前主数据名称。门店、车组、车型、企业、客户等级等主数据后续调整时，不改写已创建订单的历史展示信息。

## 6. 异常处理

| 场景 | 处理规则 |
| :--- | :--- |
| 未选择客户 | 禁止创建订单，提示“请选择客户”。 |
| 未选择车组 | 禁止创建订单，提示“请选择车组”。 |
| 还车时间早于或等于取车时间 | 禁止创建订单，提示“还车时间必须晚于取车时间”。 |
| 车组库存为 0 | 车组卡片置灰，不允许选择。 |
| 设备类增值服务库存不足 | 不允许选择超出可用库存的数量。 |
| 积分抵扣超过客户剩余积分 | 输入值自动限制到客户剩余积分。 |
| 优惠码无效 | 提示“无效的优惠码”。 |
| 企业预订选择优惠券或积分 | 页面不展示对应入口；通过接口传入时系统忽略并记录校验提示。 |
| 下单预授权选择线下收款 | 页面不展示线下收款入口；通过接口传入时校验失败，不创建有效预授权记录。 |
| 支付失败 | 订单不进入已预订状态，保留在支付弹窗，允许重新发起支付。 |
| 待支付补预授权失败 | 订单保持 `pending_payment`，支付状态保持 `unpaid`，不写入有效资金记录。 |
| 待支付资源保留超时 | 订单进入 `cancelled`，释放车组库存和派车关系，不生成取消费。 |

## 7. UI/UX 规则

* 页面采用左侧录入、右侧费用明细吸顶布局。
* 费用明细随页面滚动保持可见。
* 车组库存为 0 时，卡片置灰并禁止点击。
* 已选车组使用高亮边框和选中图标。
* 企业预订开关在客户无关联企业时置灰。
* 行销优惠方案在未选择客户时置灰；切换个人预订/企业预订后，方案下拉同步切换为对应资格可用方案。
* 优惠券、兑换码和积分抵扣仅个人预订展示；企业预订展示“企业身份不支持优惠券和积分”的提示。
* 支付弹窗中展示订单总额、车组押金标准、本次预授权金额。
* 非月结企业预订在支付弹窗中按客户自付订单处理，展示预授权押金、支付渠道和创建后的支付状态。
* 暂不收款模式下，支付弹窗展示无需本次收款的原因和订单创建后的状态。
* 企业月结模式下，支付弹窗展示“不发起支付、不发起预授权、最终费用进入企业月结账单”，创建成功后展示支付状态“企业月结”和预计月结金额。
* 创建成功后展示订单号、主状态、支付状态、派车状态，并提供跳转订单详情入口。

## 8. 验收标准

| 场景 | 验收标准 |
| :--- | :--- |
| 默认时间 | 进入页面时，取车时间默认为次日 09:00，还车时间默认为取车时间 + 1 天。 |
| 租期计算 | 修改取还时间后，租期、超时折算、计费金额和费用明细实时刷新；不足 1 小时按 1 小时计。 |
| 库存不足 | 可用库存为 0 的车组置灰，不允许选择。 |
| 个人预订 | 选择个人客户并完成预授权押金后，订单创建为 `reserved`，支付状态为 `deposit_paid`。 |
| 预授权押金 | 本次预授权金额按租赁业务规则的固定额度或租金比例生成，默认展示在支付弹窗中；比例模式不超过 100%。 |
| 暂不收款 | 选择暂不收款后，可 0 元创建订单，订单创建为 `pending_payment`，支付状态为 `unpaid`。 |
| 待支付补预授权 | 待支付订单点击“支付”后打开下单预授权押金确认弹窗；确认成功后进入 `reserved`，支付状态为 `deposit_paid`。 |
| 非月结企业预订 | 企业客户选择企业预订但不使用月结时，使用企业资格可用行销方案或企业优惠资格，费用由客户本人支付，创建后支付状态按所选收款模式生成。 |
| 企业月结 | 企业客户选择企业预订和企业月结后，可 0 元创建订单，不发起支付和预授权，支付状态为 `monthly_billing`，最终应收金额进入企业月结账单。 |
| 计费快照 | 订单创建成功后保存订单级基线计费快照、规则快照和对应版本号；后续价格、行销方案或规则页面调整只影响新建订单，不影响该订单。涉及商业条件变化的后续事件生成事件级增量快照，订单详情页可查看标准价版本、行销方案版本、租赁规则版本和事件快照版本。 |
| 增值服务 | 勾选按天服务后，费用随租期天数变化。 |
| 优惠抵扣 | 个人预订按行销优惠方案、积分、优惠券顺序抵扣，最终金额不小于 0；企业预订仅使用企业资格可用行销优惠方案，不支持优惠券和积分。 |
| 车组押金标准 | 选择不同车组时，车组押金标准同步刷新。 |
| 创建成功 | 创建成功弹窗展示订单号、主状态、支付状态、派车状态。 |
