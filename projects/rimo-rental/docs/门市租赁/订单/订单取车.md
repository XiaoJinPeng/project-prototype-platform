## 1. 文档概述

### 1.1 背景
取车是门市租车订单从预约阶段进入实际履约阶段的关键节点。门店员工需要在客户到店后完成身份核验、车辆锁定、预授权押金确认、租赁合约签署、验车留底和发车确认，确保客户资质、资金担保、车辆交付记录完整可追溯。

### 1.2 目标
* **流程标准化**：通过分步向导引导门店完成取车作业。
* **资金安全**：客户自付订单在发车前完成预授权押金校验、预授权处理方式确认和租金尾款结清；企业月结订单在发车前确认企业月结结算方式。
* **风险控制**：发车前完成驾驶人证件、黑名单、驾照有效期等风控校验。
* **资产留痕**：记录发车里程、油量/电量、车辆照片和已有损伤。
* **状态清晰**：取车开始后订单进入 `inspecting`，发车成功后进入 `renting`。

### 1.3 页面入口
* 门市租车订单列表：主状态为 `reserved`、`pickup_overdue` 的订单展示“取车”按钮，主状态为 `inspecting` 的订单展示“继续取车”按钮。
* 订单详情页：主状态为 `reserved`、`pickup_overdue` 的订单展示“取车”按钮，主状态为 `inspecting` 的订单展示“继续取车”按钮。
* 对应原型文件：`order_detail.html`

## 2. 状态与入口规则

| 场景 | 规则 |
| :--- | :--- |
| 允许取车 | 订单主状态为 `reserved`、`pickup_overdue`。 |
| 继续取车 | 订单主状态为 `inspecting`。 |
| 禁止取车 | 订单主状态为 `pending_payment`、`renting`、`renewing`、`overdue`、`settlement_pending`、`completed`、`cancelled`、`closed`。 |
| 待支付订单 | 必须先完成支付入口处理，不能直接进入取车。 |
| 取车开始 | 点击“取车”并创建操作锁成功后，订单立即进入 `inspecting`。 |
| 取车暂停 | 未确认发车前关闭向导，订单保持 `inspecting`，保留取车草稿和操作记录，列表和详情展示“继续取车”。 |
| 取车无后续操作 | 操作锁超过有效期后，订单仍保持 `inspecting`，并标记为取车中断；原操作人可继续办理，店长或管理员可释放操作锁。 |
| 操作锁释放 | 管理员释放操作锁并作废取车草稿后，订单先回到进入取车前主状态，再按当前时间重新判定 `reserved`、`pickup_overdue` 或 `no_show`。 |
| 发车成功 | 订单主状态更新为 `renting`，车辆营运状态更新为 `renting`，车辆启用状态保持原值。 |

### 2.1 状态切换口径

| 状态切换 | 触发时机 | 说明 |
| :--- | :--- | :--- |
| `reserved` -> `inspecting` | 门店点击“取车”，系统成功创建操作锁。 | 客户已到店并开始办理取车。该状态不是在拍照验车后才进入，而是在取车作业被正式接管时进入。 |
| `pickup_overdue` -> `inspecting` | 逾期未取客户到店后，门店点击“取车”，系统成功创建操作锁。 | 保留逾期未取的历史记录，当前主状态进入取车办理中。 |
| `inspecting` -> `renting` | 门店点击“确认发车”，且身份核验、行销方案资格核验、资金校验、合同签署、车辆锁定、验车数据全部通过。客户自付订单的资金校验包含预授权押金和应收尾款；企业月结订单的资金校验为确认企业月结结算方式。 | 车辆已经交付客户，订单进入实际用车阶段。 |
| `inspecting` -> `rejected` | 取车办理中触发风控、证件或资质拦截，门店确认拒绝履约。 | 订单终止履约，车辆资源释放，保留拒绝原因、费用结果和预授权处理记录。 |

### 2.2 取车窗口关闭与无后续操作

| 场景 | 订单状态 | 操作锁 | 页面表现 | 后续处理 |
| :--- | :--- | :--- | :--- | :--- |
| 点击取车后直接关闭窗口 | 保持 `inspecting` | 保持锁定 | 订单列表和详情页展示“继续取车” | 原操作人继续办理。 |
| 取车中途关闭窗口 | 保持 `inspecting` | 保持锁定 | 保留已录入的身份、押金尾款、合约、验车草稿 | 原操作人继续办理。 |
| 关闭窗口后无后续操作且未超时 | 保持 `inspecting` | 仅原操作人可继续 | 展示操作人和操作锁到期时间 | 等待原操作人继续。 |
| 关闭窗口后超过操作锁有效期 | 保持 `inspecting`，标记取车中断 | 锁进入可释放状态 | 列表和详情页展示“继续取车”，详情页展示“释放操作锁” | 原操作人继续办理，或店长/管理员释放操作锁。 |
| 打开拒绝履约弹窗后返回取车 | 保持 `inspecting` | 保持锁定 | 回到取车向导 | 继续办理取车，或重新触发拒绝履约。 |
| 店长/管理员释放操作锁 | 系统先回到进入取车前主状态，再按当前时间重新判定是否应进入 `pickup_overdue` 或满足爽约条件 | 操作锁解除 | 取车草稿作废，列表恢复“取车”或展示逾期/爽约处理入口 | 后续可重新点击取车、取消或按爽约处理。 |

系统不自动将 `inspecting` 回退为 `reserved` 或 `pickup_overdue`。取车作业一旦开始，代表门店已经接管订单，自动回退会造成门店交接和风控记录丢失。回退必须通过释放操作锁完成，并写入操作日志。
释放操作锁时，系统必须重新读取当前时间、预计取车时间和订单规则快照中的保留时长：未超过预计取车时间时回到 `reserved`；已超过预计取车时间但未超过保留时长时进入 `pickup_overdue`；已超过保留时长且自动释放资源开启时进入 `no_show` 并释放车辆资源；自动释放资源关闭时进入 `pickup_overdue` 并展示“标记爽约”。

### 2.3 车辆启用状态与营运状态校验

取车发车前，系统必须同时校验订单状态、派车状态、车辆启用状态和车辆营运状态，不能只判断订单已经进入 `inspecting`。

| 校验项 | 通过条件 | 不通过时处理规则 |
| :--- | :--- | :--- |
| 车辆启用状态 | 车辆为 `enabled` | 阻断发车。`reserved`、`pickup_overdue`、`inspecting` 且尚未发车的订单，需要重新校验是否可同车行改派。 |
| 车辆营运状态 | 车辆为 `idle` | `renting` 代表车辆仍被前序订单占用，`controlled` 代表车辆处于清洁、保养、调度、还车待整备等内部处理阶段，均不得发车。 |
| 派车状态 | 当前订单存在 `locked` 车辆 | 未硬锁时不允许发车。 |
| 车辆交付范围 | 车辆仍在订单取车门市可交付范围内，且车组、服务类型匹配订单要求 | 不满足时不允许发车。 |

已硬锁车辆在取车前变为 `disabled` 或进入 `controlled` 时，订单主状态保持 `inspecting`，系统应先在同车行范围内重新复核和改派；改派失败时，订单进入人工调度处理，门店不得直接确认发车。

## 3. 取车流程

```mermaid
flowchart LR
    A[进入取车] --> B[身份与优惠资格核验]
    B --> C[押金尾款]
    C --> D[合约验车发车]
    D --> E[发车完成]
```

### Step 1 身份与优惠资格核验

#### 1.1 驾驶人信息
页面展示并校验以下信息：

| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| 驾驶人姓名 | 文本 | 是 | 默认取订单客户姓名，支持按实际驾驶人调整。 |
| 手机号 | 文本 | 是 | 用于接收取车通知和合同签署通知。 |
| 身份证号 | 文本 | 是 | 用于实名校验。 |
| 驾照有效期 | 日期 | 是 | 用于判断驾驶资格是否有效。 |
| 身份证照片 | 图片 | 是 | 缺失时需现场拍摄上传。 |
| 驾照照片 | 图片 | 是 | 缺失时需现场拍摄上传。 |

#### 1.2 校验规则
* 驾照已过期时禁止进入下一步。
* 客户命中黑名单时禁止发车，根据风控结果进入 `rejected` 或人工审核。
* 证件信息缺失时禁止进入下一步。
* 门店员工必须勾选“确认人证合一”后才能进入下一步。

#### 1.3 行销方案资格核验
门店人员在取车时需要复核订单命中的行销方案是否符合客户证件、会员方案、企业资格和方案适用条件。复核不使用当前行销方案价格覆盖订单快照，只用于判断本次取车是否符合原方案资格。

行销方案的基础适用条件仍由系统在下单时判断，包括会员方案、企业资格、适用门店、适用车组、有效期、下单时间和用车时间等。到店人工核验不是与系统条件二选一的核验方式，而是在基础条件命中后额外判断“是否需要门店现场核验证明文件”。

| 到店人工核验配置 | 处理规则 |
| :--- | :--- |
| 未开启 | 取车时展示行销方案命中结果，门店无需额外上传证明文件。 |
| 已开启 | 用于学生、教师、军警、医护、长者等需要现场查看证明文件的优惠资格。取车第一步必须展示资格名称和证明文件说明，并由门店上传证明文件附件后记录核验结果。 |

到店人工核验的页面字段如下：

| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| 当前行销方案 | 文本 | 自动展示 | 展示订单创建时选择的行销方案名称和版本号。 |
| 到店核验 | 标签 | 自动展示 | 展示“需到店核验”或“无需到店核验”。 |
| 资格名称 | 文本 | 条件必填 | 到店人工核验时展示，例如学生身份、教师身份。 |
| 证明文件说明 | 文本 | 条件必填 | 到店人工核验时展示行销方案配置的文本说明，例如需查看学生证、教师证或在职证明。 |
| 证明文件附件 | 图片 | 条件必填 | 到店人工核验通过时至少上传 1 个证明文件附件，最多 3 个附件，不固定附件名称；核验不通过时可不上传附件，但必须填写不通过原因并更换行销方案或改为不使用优惠。 |
| 核验结果 | 单选 | 条件必填 | 到店人工核验时必须选择“通过”或“不通过”。 |
| 不通过原因 | 文本 | 条件必填 | 核验不通过时填写，例如无法提供有效学生证、证件过期、证件与本人不符。 |
| 更换后行销方案 | 下拉选择 | 条件必填 | 核验不通过时必须更换为客户当场符合资格的行销方案，或选择不使用优惠/默认方案。 |

处理规则：

* 核验通过时，系统保存核验结果、证明文件附件、核验人、核验时间、行销方案版本号和订单号，原行销方案继续生效。
* 核验不通过或客户无法提供证明文件时，原行销方案不得继续生效，门店必须在取车流程内更换行销方案或改为不使用优惠。
* 更换行销方案属于商业条件变化，系统必须生成取车事件级增量快照，保存调整前方案、调整后方案、调整原因、操作人、操作时间、证明文件记录和费用试算结果，不覆盖原订单级基线快照。
* 更换行销方案后，系统按取车时点命中的方案版本重新试算订单总额和应收尾款。客户自付订单在完成应收尾款收款后才能进入合约验车；企业月结订单不向客户收款，重算后的最终应收仍进入企业月结账单。
* 资格核验必须在押金尾款步骤之前完成。取车尾款、预授权转租金和履约担保处理均读取资格核验后的当前有效行销方案结果。
* 需要到店人工核验的行销方案本期仅适用于有人服务的门市租车；24 小时自助租车没有店员现场核验能力，本期不开放选择该类方案。

#### 1.4 24 小时自助取车仪表值

24 小时自助租车订单在订单详情页取车弹窗中，发车里程、发车油量/电量默认读取车机最新实时值，并展示“默认带入车机值”和同步时间。

处理规则如下：

* 已获取车机实时值时，页面默认带入车机最新里程和油量/电量，后台人员仍可根据现场核对结果手动修改。
* 未获取车机实时值时，页面提示“未获取车机实时值”，允许按现场仪表盘人工录入后继续。
* 确认发车后，以最终确认值写入取车记录和取车展示快照；车机默认值只作为默认带入来源，不单独覆盖人工确认结果。
* 该规则仅适用于 24 小时自助租车；有人服务的门市租车仍按店员现场录入为准。

### Step 2 押金与尾款

#### 2.1 预授权押金
客户自付订单在下单阶段默认按租赁业务规则收取预授权押金，包含个人预订和非月结企业预订。取车阶段必须确认预授权押金有效，并选择“履约担保”或“预授权转租金”处理方式。预授权处理方式和订单应收尾款均满足条件后，才能继续办理验车发车。

企业月结订单不生成预授权押金，不展示预授权处理方式，不要求客户在取车阶段补交租金。取车阶段校验订单已绑定企业月结结算方式，确认后视为资金校验完成；最终应收金额在还车结算确认后统一进入企业月结账单。

| 项目 | 规则 |
| :--- | :--- |
| 预授权金额 | 按订单规则快照中的预授权额度模式生成：固定额度模式按固定金额冻结，租金比例模式按订单租金乘以配置比例冻结；比例模式本期不超过 100%。 |
| 履约担保 | 预授权保持冻结，不抵扣租金；尾款结清并确认发车后释放预授权。 |
| 转租金 | 预授权扣款转为租金实收，系统按扣款金额抵扣取车应收尾款。 |
| 预授权状态 | 履约担保模式要求 `frozen`；转租金模式要求 `captured`。 |
| 待支付订单 | `pending_payment` 订单必须先完成支付/预授权处理。 |
| 非月结企业预订 | 使用企业资格可用行销方案或企业优惠资格，预授权和尾款规则与个人预订一致，费用由客户本人支付。 |
| 企业月结 | 不发起预授权，不校验押金，不进入客户支付渠道；系统确认订单结算方式为企业月结，并展示预计月结金额。 |
| 应收尾款 | 客户自付订单中，履约担保模式按订单总额扣除已支付租金计算；转租金模式额外扣除预授权转租金扣款。企业月结订单取车阶段不产生客户应收尾款，最终应收金额在还车结算确认后统一挂账到企业。客户自付尾款未结清时禁止进入下一步。 |

#### 2.2 尾款收取
* 取车时系统展示订单总额、已收款、预授权押金、处理方式和应收尾款。
* 履约担保模式下，下单时收取的预授权押金不等同于租金全款，不计入已收租金。
* 转租金模式下，预授权扣款成功后计入租金实收，并减少取车应收尾款。
* `balanceDue > 0` 时，必须通过在线支付、信用卡、现金、转账等方式补足尾款。
* 客户自付订单尾款结清后才允许进入验车发车步骤。
* 非月结企业预订的尾款由客户本人支付，支付方式、收款凭证、后续还车补缴和退款规则与个人预订一致。
* 企业月结订单取车阶段不生成客户收款和预授权记录，只记录预计月结金额；最终挂账金额以还车结算确认结果为准。

### Step 3 合约签署与验车发车

#### 3.1 租赁合约签署
* 支持推送至客户 App 签署。
* 支持柜台现场签名。
* 签署完成后展示签名来源、签署时间和签名图片。
* 系统故障时支持上传线下纸质合同照片作为兜底凭证。
* 未完成合约签署时禁止确认发车。

#### 3.2 车辆锁定
* 取车前必须锁定具体车辆。
* 未分配车牌时，门店需先完成派车或改派。
* 车辆锁定后 `dispatchStatus=locked`。
* 门市租车客户面向取车门市办理取车，不直接面向停车场据点。车辆被系统派给订单后，订单列表、订单详情和取车窗口需要展示车辆当前所在据点，门店人员根据该据点信息完成备车。订单列表和详情页的取还站点展示格式为“门市 - 据点 - 地址”，列表无地址时展示为“门市 - 据点”。
* 车辆所在据点由已派车辆带出，不由店员手动选择。该信息用于内部资产位置和车辆交接，不改变客户订单中的取车门市。

#### 3.3 发车数据

| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| 发车里程 | 数值 | 是 | 记录仪表盘当前里程。 |
| 发车油量/电量 | 数值 | 是 | 燃油车按 0/8 到 8/8 记录，电车按百分比记录。 |
| 实际取车时间 | 日期时间 | 是 | 默认当前时间，支持门店微调。 |
| 车辆所在据点 | 只读展示 | 自动带出 | 根据已派车辆的当前所属据点展示，记录车辆从哪个门市下据点备车，不允许店员手动选择。 |
| 交付位置 | 文本 | 是 | 记录门市交付区、临停车位或其他实际交付位置。 |
| 发车备注 | 文本 | 否 | 记录交付异常、客户说明等信息。 |

#### 3.4 验车影像
* 外观标准照：前方、后方、左侧、右侧、左前角、右前角。
* 内饰标准照：仪表盘、前排座椅、后排环境。
* 已有损伤需在车辆示意图或损伤列表中标记位置、类型、照片和说明。
* 发车照片作为还车定损基准，不允许在发车完成后覆盖原记录。

### Step 4 发车完成
点击“确认发车”后，系统执行以下动作：

| 动作 | 结果 |
| :--- | :--- |
| 订单状态 | `inspecting` -> `renting`。 |
| 车辆营运状态 | 更新为 `renting`，车辆启用状态保持原值。 |
| 派车状态 | 更新为 `locked`。 |
| 预授权处理 | 履约担保模式释放预授权；转租金模式保留预授权扣款对应的租金实收记录。 |
| 企业月结处理 | 企业月结订单不释放预授权、不扣取预授权，保留月结结算标识并进入用车中。 |
| 取车记录 | 写入发车时间、里程、油量/电量、照片、损伤记录、车辆所在据点、交付位置、经办人。 |
| 取车展示快照 | 保存确认发车时的实际驾驶人、取车门市名称和地址、车辆所在据点、交付位置、车辆 ID、车牌、车组名称、车型名称、车辆归属门市、发车里程和油量/电量。后续车辆换牌、车型改名、门市或据点改名不改写该取车记录。 |
| 合约记录 | 写入签署状态、签署方式、签署时间。 |
| 通知 | 向客户发送取车成功通知。 |

### 3.5 24 小时自助取车闭环

24 小时自助租车由用户在 APP 发起取车，不经过门店人工取车向导。系统仍复用门市租赁订单、派车、费用和取车记录，但需要单独记录自助取车状态和车机指令状态。

自助取车状态字段 `selfPickupStatus`：

| 字段值 | 页面显示 | 说明 |
| :--- | :--- | :--- |
| `not_started` | 未开始 | 用户尚未在 APP 发起自助取车。 |
| `precheck_failed` | 预检失败 | 订单、车辆、GPS、车机、支付或控车权限未通过取车预检。 |
| `payment_pending` | 待支付 | 客户自付订单整笔订单应收金额尚未结清，等待用户线上支付或预授权转租金处理。 |
| `unlock_pending` | 开锁处理中 | 取车验车记录已提交，系统已向车机下发开锁指令，等待车机返回结果。 |
| `unlock_failed` | 开锁失败 | 车机开锁失败、超时或返回未知结果，订单未进入用车中。 |
| `pickup_completed` | 取车完成 | 取车记录保存成功，车机开锁成功或人工确认交付，订单已进入用车中。 |
| `manual_required` | 需人工处理 | 预检、支付、车机或取车记录存在无法自动闭合的异常，需要门店或客服处理。 |

车机指令状态字段 `telematicsCommandStatus`：

| 字段值 | 说明 |
| :--- | :--- |
| `pending` | 指令待发送。 |
| `sent` | 指令已发送，等待车机回传。 |
| `success` | 车机确认执行成功。 |
| `failed` | 车机明确返回失败。 |
| `timeout` | 超过系统等待时间未收到明确结果。 |
| `unknown` | 车机返回结果无法判断是否执行成功。 |

自助取车处理规则：

1. 用户点击一键取车后，系统先校验订单主状态、派车状态、硬锁车辆、GPS 位置、车辆启用状态、车辆营运状态、车机在线状态、车门可控状态和资金状态。
2. 企业月结订单跳过个人支付和预授权处理；个人自付和企业非月结订单必须在自助取车开锁前结清整笔订单应收金额。
3. 自助取车应结清金额按订单当前计费快照计算：`自助取车待付金额 = 订单总额 - 已实际收款金额 - 预授权转租金扣款金额`。下单阶段的预授权冻结不等同于租金支付，未转租金时不得抵扣订单总额。
4. `自助取车待付金额 > 0` 时，APP 必须先完成线上支付或预授权转租金处理；自助取车阶段不支持线下收款、到店收款或仅登记收款凭证。
5. `自助取车待付金额 = 0` 后，系统才允许保存取车验车记录并下发车机开锁指令。待付金额未结清时，不得下发开锁指令，不得进入 `unlock_pending`。
6. 支付或预授权转租金成功后，系统保存资金记录；后续开锁失败时不得重复发起相同支付、重复扣款或重复释放预授权。
7. 用户提交取车照片、里程、油量/电量后，系统先保存取车验车记录，再下发车机开锁指令。
8. 车机开锁成功后，订单主状态从 `reserved` 更新为 `renting`，车辆营运状态更新为 `renting`，用户 APP 控车权限生效，`selfPickupStatus` 更新为 `pickup_completed`。
9. 车机开锁失败、超时或结果未知时，订单主状态保持取车前状态，不进入 `renting`；已成功的支付或预授权转租金记录保留，`selfPickupStatus` 更新为 `unlock_failed` 或 `manual_required`。
10. 开锁失败后，APP 展示“开锁失败，请重试或联系客服”。系统允许在同一订单、同一取车记录下重试车机开锁；重试必须使用幂等号，避免重复生成取车记录、重复扣款或重复改变订单状态。
11. 人工确认交付只能由具备权限的门店、客服或管理员操作。人工确认时必须记录原因、处理人、处理时间和车机异常记录；确认后订单才能进入 `renting`。
12. 自助取车期间所有车机指令必须记录指令编号、幂等号、指令类型、发送时间、回调时间、执行结果、失败原因、重试次数和关联订单。

## 4. 操作锁规则

* 点击“取车”后生成订单操作锁。
* 操作锁创建成功后，订单主状态立即变更为 `inspecting`。
* 操作锁期间，其他人员不能编辑订单金额、车辆、优惠、服务项和关键客户信息。
* 取车中改派只能由当前取车操作锁持有人在取车流程内执行；其他人员如需改派，必须先按权限释放或接管操作锁。
* 操作锁展示当前操作人、开始时间和锁定原因。
* 操作人关闭向导但未确认发车时，系统保存草稿，订单保持 `inspecting`，可继续办理取车。
* 操作锁有效期默认为 30 分钟，租赁业务规则支持配置。
* 超过操作锁有效期后，订单不自动回退，系统标记为取车中断并允许具备操作锁释放权限的人员释放操作锁。
* 释放操作锁后，取车草稿作废，订单先回到进入取车前状态，再按当前时间重新判定 `reserved`、`pickup_overdue` 或 `no_show`，并写入操作日志。

## 5. 异常处理

| 场景 | 处理规则 |
| :--- | :--- |
| 客户未支付 | 跳转支付入口，完成后再办理取车。 |
| 预授权失败 | 支持更换线上支付渠道或重新发起预授权；本期不支持现金押金、线下押金或仅登记线下凭证。预授权未成功时禁止发车，可保持取车中继续处理，或按拒绝履约/取消流程终止订单。 |
| 预授权转租金失败 | 保留履约担保模式，门店可改为补收尾款后继续发车；如需继续转租金，必须重新发起扣款并成功。 |
| 履约担保释放失败 | 订单仍可完成发车，系统保留释放失败记录并提示门店后续跟进；费用中心展示预授权待释放状态。 |
| 企业月结关系异常 | 禁止确认发车，门店需确认订单企业、员工身份和月结结算方式后继续办理。 |
| 驾照过期 | 禁止发车，订单进入人工处理。 |
| 行销方案资格核验不通过 | 禁止继续使用原行销方案。门店在取车流程内更换为客户当场符合资格的行销方案或不使用优惠，并重新试算尾款；客户拒绝补差或无可用方案时，按拒绝履约处理。 |
| 客户命中黑名单 | 禁止发车，订单进入风控处理。 |
| 车辆已禁用 | 禁止发车。订单尚未开始履约时，系统重新校验是否可同车行改派；改派失败进入人工调度。 |
| 车辆被管控中 | 禁止发车。车辆解除清洁、保养、调度、还车待整备等管控前，不允许继续取车。 |
| 未锁定车辆 | 禁止发车，需先完成派车或改派。 |
| 合同未签署 | 禁止发车。 |
| 验车照片缺失 | 禁止发车。 |
| 24 小时自助取车预检失败 | APP 不进入取车验车页，展示失败原因；订单主状态不变，`selfPickupStatus=precheck_failed`。 |
| 24 小时自助取车整单未结清 | APP 停留在支付页，必须完成整笔订单应收金额支付或预授权转租金后才能继续；系统不得下发开锁指令。 |
| 24 小时自助取车支付成功后开锁失败 | 保留已成功支付或预授权转租金记录，订单不得进入 `renting`，不得重复扣款；APP 提示重试或联系客服，后台展示自助取车异常。 |
| 24 小时自助取车开锁结果未知 | 订单不得直接进入 `renting`，系统先查询车机最终状态；仍无法确认时进入人工处理。 |
| 拒绝履约 | 取车办理中触发资质、证件、预授权或风控拦截时，门店填写拒绝原因并确认后，订单进入 `rejected`，车辆资源释放，费用和预授权按《取消与关闭订单》处理。 |
| 取车窗口被关闭 | 订单保持 `inspecting`，保留取车草稿，列表和详情页展示“继续取车”。 |
| 取车操作超时 | 订单保持 `inspecting`，标记取车中断，具备操作锁释放权限的人员可释放操作锁。 |

## 6. UI/UX 规则

* 取车向导使用全屏弹窗，顶部展示步骤进度。
* 每一步只展示当前步骤必要信息，减少门店操作干扰。
* 未满足前置条件时，“下一步”按钮置灰或点击后给出明确提示。
* 第一步展示身份核验和行销方案资格核验。需要到店人工核验的方案必须展示资格名称、证明文件说明、最多 3 个证明文件上传入口、核验结果和更换行销方案入口。
* 客户自付订单的押金与合约步骤展示订单总额、预授权金额、预授权状态、处理方式和应收尾款。
* 企业月结订单的押金与合约步骤展示订单总额、企业名称、结算方式和预计月结金额，不展示支付渠道、预授权处理方式和补收尾款按钮。
* 客户自付订单应收尾款大于 0 时，下一步按钮阻断并提示“请结清应收尾款”；企业月结订单结算方式异常时，下一步按钮阻断并提示“请确认企业月结结算方式”。
* 验车照片位使用固定标准位，保证门店拍摄口径统一。
* 取车向导在确认发车前展示“拒绝履约”入口，触发后弹窗填写拒绝原因、责任费用、预授权处理结果和备注。
* 移动端和平板端需保持拍照、签名、油量选择可操作。

## 7. 验收标准

| 场景 | 验收标准 |
| :--- | :--- |
| 已预订取车 | `reserved` 订单可点击“取车”，进入取车向导。 |
| 逾期未取继续办理 | `pickup_overdue` 订单可点击“取车”，进入取车向导。 |
| 验车中继续办理 | `inspecting` 订单展示“继续取车”，点击后回到取车向导。 |
| 取车窗口关闭 | 关闭取车窗口但未确认发车时，订单保持 `inspecting`，草稿保留。 |
| 取车超时释放 | 操作锁超时后，具备操作锁释放权限的人员释放操作锁，订单先回到进入取车前状态，再按当前时间重新判定 `reserved`、`pickup_overdue` 或 `no_show`。 |
| 待支付拦截 | `pending_payment` 订单不允许直接取车，需先完成支付入口处理。 |
| 操作锁 | 进入取车向导后，订单进入 `inspecting`，其他人员不能编辑关键字段。 |
| 身份与优惠资格核验 | 未勾选“确认人证合一”不能进入下一步。需要到店人工核验的行销方案未记录核验结果，核验通过但未上传证明文件，或核验不通过但未更换行销方案时，不能进入押金尾款步骤。 |
| 行销方案变更 | 取车时因资格核验不通过更换行销方案后，系统生成事件级增量快照，重新试算订单总额和应收尾款，并在订单记录中保存变更前方案、变更后方案、原因、操作人、操作时间和证明文件。 |
| 履约担保 | 选择履约担保时，预授权保持冻结，尾款结清并确认发车后释放预授权。 |
| 转租金 | 选择转租金时，预授权扣款成功后计入租金实收，并扣减应收尾款。 |
| 预授权押金 | 预授权状态未满足当前处理方式要求时不能进入验车发车。 |
| 应收尾款 | 应收尾款未结清时不能进入验车发车。 |
| 非月结企业预订 | 非月结企业预订按客户自付订单校验预授权押金和应收尾款。 |
| 企业月结 | 企业月结订单不展示预授权和补收尾款操作，确认企业月结结算方式后可进入验车发车。 |
| 合同签署 | 未完成租赁合约签署不能确认发车。 |
| 验车数据 | 发车里程、油量/电量、标准照片缺失时不能确认发车。 |
| 拒绝履约 | `inspecting` 订单确认拒绝履约后进入 `rejected`，车辆资源释放，费用中心生成拒绝履约费用或预授权处理记录。 |
| 发车成功 | 确认发车后订单状态由 `inspecting` 变为 `renting`，车辆状态变为用车中。 |
| 自助取车整单支付 | 个人自付和企业非月结自助订单在下发开锁指令前，订单总额必须已通过线上支付或预授权转租金结清。 |
| 自助取车开锁成功 | 24 小时自助订单保存取车记录并收到车机开锁成功后，订单进入 `renting`，用户获得 APP 控车权限。 |
| 自助取车开锁失败 | 支付已成功但开锁失败时，订单不进入 `renting`，不重复扣款，列表或详情可识别自助取车异常并允许重试或人工处理。 |
