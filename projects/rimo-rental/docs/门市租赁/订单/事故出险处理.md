## 1. 文档概述

### 1.1 背景
事故/出险处理用于记录车辆在租期内、还车结算中或订单完成后补录的事故、损伤、保险报案和客户责任费用。事故登记会影响订单风险状态、车辆整备、费用结算和后续追缴。

### 1.2 目标
* **事故可追溯**：记录事故时间、地点、描述、照片、责任归属、保险信息和处理状态。
* **状态可控**：履约中发生事故时，订单进入 `accident_processing`，避免直接完成或遗漏风控处理。
* **费用可结算**：事故车损、维修、折旧、停运、拖车、救援等费用进入费用中心。
* **车辆可管理**：事故记录同步影响车辆状态，便于后续维修、停运和排期调整。
* **完成后可追缴**：订单完成后补录事故费用时，订单主状态保持 `completed`，费用走追缴流程。

### 1.3 页面入口
* 订单详情页“更多操作 - 事故登记”。
* 订单列表 `accident_processing` 状态行展示“处理事故”。
* 还车验车发现新增损伤时，可从还车流程生成事故或车损记录。
* 已完成订单发现后续事故、违章关联损伤或保险追偿时，可在详情页补录。

## 2. 适用状态

| 主状态 | 是否允许登记事故 | 状态处理 |
| :--- | :--- | :--- |
| `pending_payment`、`reserved`、`pickup_overdue`、`inspecting` | 否 | 未进入实际用车或未交付，不登记租期事故。 |
| `renting`、`renewing`、`overdue` | 是 | 登记后订单进入 `accident_processing`。 |
| `accident_processing` | 是 | 追加事故记录或更新处理进度，状态保持不变。 |
| `settlement_pending`、`payment_due`、`refund_pending` | 是 | 订单保持当前结算状态，新增事故风险和费用线索。 |
| `completed` | 是 | 主状态保持 `completed`，事故费用进入追缴。 |
| `cancelled`、`no_show`、`rejected`、`closed` | 否 | 终态订单不新增事故处理，仅保留备注。 |

---

## 3. 状态流转

| 流转 | 触发时机 | 说明 |
| :--- | :--- | :--- |
| `renting / renewing / overdue` -> `accident_processing` | 履约中登记事故。 | 订单进入出险处理中，列表展示“处理事故”和“还车”。 |
| `accident_processing` -> `settlement_pending` | 车辆归还并提交还车验车。 | 事故费用纳入还车结算。 |
| `accident_processing` -> `closed` | 门店责任、重大客诉、不可抗力导致客户无责终止。 | 走关闭订单流程，全额退款或按具备权限人员的处理结果执行。 |
| `accident_processing` -> `completed` | 事故费用已平账、车辆已归还并确认还车。 | 订单完成；车辆根据事故处理结果进入维修、停运，或进入被管控中下的清洁/整备任务，任务完成后恢复可用。 |
| `completed` -> `completed` | 完成后补录事故费用。 | 主状态不变，追缴状态独立变化。 |

## 4. 事故登记字段

| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| 事故时间 | 日期时间 | 是 | 默认当前时间，支持门店调整。 |
| 事故地点 | 文本 | 否 | 记录事故发生地点或停车位置。 |
| 事故描述 | 多行文本 | 是 | 记录事故部位、程度、原因和客户说明。 |
| 责任归属 | 单选 | 是 | 客户责任、门店责任、第三方责任、责任待定。 |
| 处理状态 | 单选 | 是 | 待定损、保险处理中、客户确认中、已结案。 |
| 预估费用 | 金额 | 否 | 预估定损金额，后续可调整为最终费用。 |
| 保险报案号 | 文本 | 否 | 已报案时填写保险公司报案号。 |
| 现场照片 | 图片 | 否 | 支持上传事故现场、车辆损伤、交警单据、维修报价。 |
| 处理备注 | 多行文本 | 否 | 记录客户沟通、保险处理、门店处理说明。 |

## 5. 费用规则

### 5.1 费用类型

| 费用类型 | 字段值 | 适用场景 |
| :--- | :--- | :--- |
| 事故车损费用 | `accident_damage` | 定损、维修、折旧、停运、拖车、救援等事故相关费用。 |
| 事故减免 | `accident_waiver` | 门店责任、保险理赔或管理授权减免客户费用。 |
| 事故追缴 | `accident_post_charge` | 订单完成后才确认的事故或保险追偿费用。 |

### 5.2 金额处理
* 客户责任、第三方责任或责任待定且需要客户先行承担时，生成事故车损费用。
* 门店责任不向客户收取事故费用，可记录内部处理和车辆维修成本。
* 保险理赔金额确认后，可用减免或负向费用抵扣客户应付金额。
* 履约中或还车结算中产生的事故费用进入租期费用。
* `completed` 后补录的事故费用进入追缴费用，不改变订单主状态。
* 企业月结订单的事故相关应收费用统一进入企业月结账单，不拆分到员工个人支付。订单未完成时，事故费用以 `monthly_billing` 状态进入费用中心并参与还车月结挂账；订单已完成时，补录事故费用直接生成 `monthly_bill` 交易记录。

### 5.3 还车结算联动
* 事故费用在还车结算中参与 `balanceDue` 计算。
* 事故费用未确认时，订单可停留在 `settlement_pending`。
* 客户补缴完成且账单平衡后，订单可确认还车进入 `completed`。
* 企业月结订单的事故费用参与企业挂账金额计算，确认月结挂账后订单可进入 `completed`，不进入 `payment_due` 或 `refund_pending`。
* 事故费用存在争议时，保留事故记录、照片、责任说明和争议备注。

### 5.4 事故结案
事故记录支持在详情页标记结案。结案仅更新事故记录处理状态和履约事件，不直接改变订单主状态。订单主状态仍按还车、结算或追缴流程流转。

| 场景 | 结案结果 |
| :--- | :--- |
| 履约中事故结案 | 事故记录变为已结案，订单仍保持 `accident_processing`，客户还车后进入还车结算。 |
| 结算中事故结案 | 事故记录变为已结案，订单保持当前结算状态，费用继续参与账单。 |
| 已完成订单补录事故结案 | 主状态保持 `completed`，追缴状态按费用处理结果展示。 |
| 企业月结事故结案 | 事故记录变为已结案，相关应收仍统一进入企业月结账单。 |

## 6. 车辆状态联动

| 场景 | 车辆状态处理 |
| :--- | :--- |
| 轻微损伤且可继续运营 | 车辆进入被管控中，并创建清洁或整备任务；任务完成后恢复可用。 |
| 需要维修 | 车辆进入维修中，排期不可用。 |
| 严重事故 | 车辆进入停运/待定损，排期不可用；系统冻结该车后续预分配和硬锁关系，受影响订单进入人工调度范围。 |
| 第三方事故待处理 | 车辆根据可用性进入维修中，或进入被管控中下的整备任务，事故记录保持未结案。 |

事故登记导致车辆不可继续出租时，系统必须同步处理后续订单：
1. 当前事故车辆的后续 `pre_assigned`、`locked`、`manual_reassigned` 或 `upgraded` 绑定全部解除或标记不可交付。
2. 受影响订单优先在同取车门店、同车组、无冲突车辆中重新派车。
3. 无法自动改派时，订单保留原主状态，增加 `dispatch_manual_required` 风险标记，并进入人工调度队列。
4. 每个受影响订单写入调度告警、原车辆、事故记录编号、处理结果和操作日志。

## 8. 异常处理

| 场景 | 处理规则 |
| :--- | :--- |
| 未填写事故描述 | 禁止保存。 |
| 未选择责任归属 | 禁止保存。 |
| 责任待定 | 允许保存事故记录，费用可先暂存为待定损。 |
| 客户拒绝确认 | 订单停留在 `accident_processing` 或 `settlement_pending`，记录争议说明。 |
| 保险未定损 | 事故费用可先录入预估金额，最终金额确认后调整。 |
| 车辆不可继续出租 | 系统提示后续订单需改派。 |
| 完成后补录 | 生成追缴费用，主状态保持 `completed`；企业月结订单同步生成 `monthly_bill` 交易记录。 |

## 9. UI/UX 规则

* 事故弹窗展示已登记事故记录，支持继续新增。
* 新增事故时必须展示事故描述、事故时间、责任归属、处理状态、预估费用和照片上传。
* 事故记录在订单详情左侧业务流中展示，包含责任归属、处理状态、预估费用和照片。
* `accident_processing` 状态在列表中高亮，操作列展示“处理事故”和“还车”。
* 事故费用在费用中心以事故类型展示，并关联事故记录。
* 完成后追缴事故费用时，金额列展示追缴状态。
* 企业月结订单的事故费用展示“企业月结”，不展示客户收款、退款入口；完成后补录的事故费用在交易记录中展示为“企业月结挂账”。

## 10. 验收标准

| 场景 | 验收标准 |
| :--- | :--- |
| 履约中登记事故 | `renting`、`renewing`、`overdue` 订单登记事故后进入 `accident_processing`。 |
| 出险处理中处理 | `accident_processing` 订单列表展示“处理事故”和“还车”。 |
| 事故费用 | 客户责任且预估费用大于 0 时，费用中心生成事故车损费用。 |
| 结算中事故 | `settlement_pending`、`payment_due`、`refund_pending` 登记事故后主状态不回退，事故费用参与结算。 |
| 完成后补录 | `completed` 订单补录事故费用后主状态保持 `completed`，追缴状态独立展示。 |
| 企业月结事故费用 | 企业月结订单发生事故费用时，费用统一进入企业月结账单，不拆分个人支付责任；订单已完成后补录事故费用必须生成 `monthly_bill` 记录。 |
| 车辆不可用联动 | 事故导致车辆停运、维修、待定损或被管控中时，后续尚未开始履约的订单解除车辆占用，能自动改派则改派，不能改派则打上“人工调度”风险标记并进入人工调度队列。 |
| 必填校验 | 未填写事故描述或责任归属时不能保存。 |
| 操作记录 | 每次事故登记写入订单操作记录和费用审计日志。 |
| 权限控制 | 店员不能作废事故记录、事故减免、事故结案。 |
