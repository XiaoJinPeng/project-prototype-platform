## 1. 功能定位

服务站点管理用于维护营运平台的车行、门市和据点层级，并配置每个据点可承接的业务服务能力。

本次改造后，一个实体据点可以同时具备多个服务能力，不再通过重复建立多个相同据点来区分同站租还、甲租乙还、公务用车或后续门市租赁服务。

本期只定义改造后的新模型和新页面行为，不处理历史旧据点合并和旧订单回溯修正。上线时需要做最小数据初始化，将既有据点原服务类型转换为对应服务能力，确保新查询接口可以继续查到原有据点。

## 2. 组织层级

平台服务站点层级固定为：

```text
车行 -> 门市 -> 据点
```

| 层级  | 定义                                    |
|:--- |:------------------------------------- |
| 车行  | 车辆资产、组织权限和后续车行报表的上层归属。                |
| 门市  | 客户服务和门店作业层级，门店账号默认只处理所属门市范围内的数据。      |
| 据点  | 具体停车场、停车区域、自助取还车点或车辆实际停放位置。车辆挂在具体据点下。 |

页面菜单名称仍使用“服务站点管理”。页面字段和业务说明中，实际停车和取还车位置统一称为“据点”。

## 3. 据点基础资料

据点基础资料包含：

| 字段                      | 说明                                 |
|:----------------------- |:---------------------------------- |
| 所属门市                    | 必填。一个据点只能属于一个门市。                   |
| 据点名称                    | 必填。用于后台列表、车辆所在位置、订单快照和用户端取还车说明。    |
| 据点类型                    | 可配置选项，例如普通营运据点、旅行社团体专用、企业活动合作据点、机场取还车点、码头取还车点。 |
| 停车场类型                   | 记录露天、室内、立体、路侧等停车场类型。               |
| 停车场公司                   | 记录实际停车场或合作停车公司。                    |
| 联络人、电话、Email、传真         | 供营运人员处理现场事务。                       |
| 地址、经纬度、电子围栏             | 用于地图定位、导航、取还车判定和车机定位校验。            |
| 进出方式、固定车位、照片、说明、备注 | 用于现场运营和取还车指引。                      |
| 总车位数、停车费规则              | 用于车位容量管理和成本参考。                     |
| 租赁押金、月租成本、水电成本、人事成本     | 用于据点营运成本记录和参考。                     |

新增或编辑据点时，“所属门市”选择器只能选择到门市层级，不得选择车行层级保存。

据点类型由营运端维护，类型配置至少包含：

| 字段 | 说明 |
|:---- |:---- |
| 据点类型名称 | 租户自定义，例如普通营运据点、旅行社团体专用、企业活动合作据点。 |
| 一般预约开放 | 开启后，一般同站租还 / 甲租乙还预约入口可按服务能力和车辆可用性查询该类型据点；关闭后，一般预约入口不展示该类型据点。 |
| 状态 | 启用、停用。停用后不得再被新据点选择，已绑定据点保留历史展示。 |

合作通路租車券批次不再依赖旧用途字段。批次创建时选择适用的据点类型和具体据点；如需做旅行社团体专用场景，营运人员建立“旅行社团体专用”等据点类型，并关闭该类型的“一般预约开放”开关。

## 4. 服务能力模型

一个据点可同时配置多个服务能力。服务能力按业务大类分组，组内再配置具体服务模式。

| 业务大类 | 服务模式      | 说明                                 |
|:---- |:--------- |:---------------------------------- |
| 分时租赁 | 同站租还      | 用户选择具体车辆，在同一据点自助取还。                |
| 分时租赁 | 甲租乙还      | 用户选择具体车辆，取车据点和还车据点可以不同。            |
| 门市租赁 | 门市租车      | 客户面对门市，具体据点用于门市内部备车、停放、整备和归还登记。    |
| 门市租赁 | 24 小时自助租车 | 客户面对具体自助据点，APP 需要展示据点位置并引导用户自助取还车。 |
| 公务用车 | 公务用车      | 企业内部公务用车预约或企业车辆管理使用。               |

服务能力分为两层：

| 层级       | 说明                                                                |
|:-------- |:----------------------------------------------------------------- |
| 是否具备服务能力 | 表示该据点是否支持该服务模式。未具备时，新订单、新预约和车辆上架不应选择该据点作为该服务模式的服务据点。              |
| 服务状态     | 表示已具备的服务能力当前是否启用。启用时可被新订单或新预约选择；暂停时保留能力配置和历史数据，但新订单或新预约不再展示该服务能力。 |

据点整体停用和单个服务能力暂停需要区分处理：

| 状态     | 影响                                               |
|:------ |:------------------------------------------------ |
| 据点停用   | 该据点不再对新订单可见，原则上不能作为新订单取还车或车辆上架位置。已创建订单保留，历史数据不变。 |
| 服务能力暂停 | 只影响该据点下对应服务模式的新订单或新预约，其他已启用服务能力仍可继续使用。           |

## 5. 分时租赁配置

分时租赁大类启用后，可分别启用同站租还和甲租乙还。

### 5.1 同站租还

同站租还服务能力配置：

| 字段    | 说明               |
|:----- |:---------------- |
| 服务状态  | 启用或暂停。           |
| 预留车位数 | 用于分时租赁车辆停放和容量参考。 |
| 服务时间  | 默认继承分时租赁大类服务时间。  |

同站租还启用后，该据点可作为同站租还订单的取车和还车据点。

### 5.2 甲租乙还

甲租乙还服务能力启用后，默认同时支持作为取车据点和还车据点，不再单独配置取车或还车开关。

| 字段       | 说明                    |
|:-------- |:--------------------- |
| 服务状态     | 启用或暂停。                |
| 预留车位数    | 用于甲租乙还车辆调度和容量参考。      |
| 服务时间     | 默认继承分时租赁大类服务时间。       |

甲租乙还还车时，系统实时读取当前已启用的甲租乙还据点。某据点关闭或暂停甲租乙还服务能力后，用户还车可选列表不再展示该据点；已创建订单仍保留，需由营运人员按现有规则处理。

甲租乙还客户是否可以选择某个据点还车，不仅取决于该据点是否具备甲租乙还服务能力，还取决于该据点当前是否存在可还车空闲车位。只有同时满足以下条件时，系统才允许将该据点作为甲租乙还可还车据点返回给前台：

- 具备甲租乙还服务能力
- 甲租乙还服务状态为启用
- 当前可还车空闲车位大于 0

当前可还车空闲车位为 0 时，不得允许客户选择该据点还车。

同站租还预留车位数与甲租乙还预留车位数之和不得超过据点总车位数。新增或编辑据点保存时，若预留车位总数超过总车位数，系统需要拦截保存并提示用户调整。

## 6. 门市租赁配置

门市租赁大类启用后，可分别启用门市租车和 24 小时自助租车。

### 6.1 门市租车

门市租车是有人服务的门市租赁模式。客户预约和履约时面对的是门市，不直接选择门市下的具体据点。

门市租车服务能力配置：

| 字段     | 说明                 |
|:------ |:------------------ |
| 服务状态   | 启用或暂停。             |
| 柜台服务时间 | 用于门店办理取车、还车、验车和收款。 |
| 是否人工交车 | 默认开启。              |
| 是否人工验车 | 默认开启。              |

门市租车启用后，该据点可作为门市内部备车、车辆停放、整备和归还停放位置。订单列表和订单详情可以展示车辆当前所在据点，方便店员备车和还车后登记最终停放位置。

门市租车异地还车费和调度费按取车门市与订单当前还车门市判断，不按具体据点判断。

### 6.2 24 小时自助租车

24 小时自助租车是客户面对具体据点的自助取还车模式。APP 或官网下单时，需要展示具体自助据点的位置、导航、取还车方式和现场说明。

24 小时自助租车服务能力配置：

| 字段       | 说明                                |
|:-------- |:--------------------------------- |
| 服务状态     | 启用或暂停。                            |
| 是否允许夜间还车 | 用于控制非柜台时间的自助还车能力。夜间范围为 21:00–06:00。 |
| 自助取还方式   | 包含 App 扫码取还、自助机取还、密码柜取钥匙、远程审核开通等。 |

24 小时自助租车与门市租车共用门市租赁车组、价格、行销方案、增值服务和业务规则。区别只在客户面对的取还车层级和操作方式。

24 小时自助租车允许还到同门市下所有具备 24 小时自助租车能力且服务状态启用的据点。同门市下不同自助据点还车不产生调度费。

## 7. 公务用车配置

公务用车服务能力用于企业内部公务用车预约或企业车辆管理。

公务用车服务能力配置：

| 字段     | 说明                          |
|:------ |:--------------------------- |
| 服务状态   | 启用或暂停。                      |
| 服务时间   | 用于企业员工预约和现场管理参考。            |
| 签约企业客户 | 支持多选，一个公务用车服务能力可绑定多个签约企业客户。 |

一个据点可同时绑定多个公务用车企业客户。

公务用车可不绑定车机；是否绑定车机不影响公务用车预约管理能力。

## 8. 列表与筛选

服务站点管理列表展示当前车行或门市下的据点。

列表字段包含：

| 字段   | 说明                                      |
|:---- |:--------------------------------------- |
| 据点名称 | 展示据点名称和地址。      |
| 所属门市 | 展示据点归属门市。                               |
| 据点类型 | 展示据点类型，并标识该类型是否一般预约可用。                                 |
| 服务能力 | 展示该据点已配置的服务能力；暂停状态需要有明显标识。              |
| 车位概况 | 展示总车位、当前停放车辆数和剩余空位，用于门店查看合作停车场当前是否还有空位。 |
| 据点状态 | 控制据点整体启用或停用。                            |

筛选条件包含：

| 条件   | 说明                                                                                                                 |
|:---- |:------------------------------------------------------------------------------------------------------------------ |
| 关键词  | 支持按据点名称和地址搜索。                                                                                                      |
| 服务能力 | 支持筛选同站租还、甲租乙还、门市租车、24 小时自助、公务用车。该筛选支持多选；多选时按“命中任一已选服务能力”过滤。即使该服务能力当前为暂停，据点仍需出现在筛选结果中，并在服务能力列通过“暂停”标签展示状态。 |
| 服务状态 | 支持筛选启用中、已暂停。已选择服务能力时，只过滤该服务能力的状态；未选择服务能力时，按据点是否存在至少一项对应状态的服务能力过滤。                                                  |
| 据点状态 | 支持筛选启用中和已停用。                                                                                                       |

服务站点管理列表中的车位概况只表达停车场运营视角的空位参考信息，不在此处重复表达甲租乙还站点分布和车位查看能力，也不能替代甲租乙还对外 API 中的可还车空闲车位口径。

需要特别注意：甲租乙还旧版 APP 已经依赖既有可还车据点查询 API 中的空闲车位字段判断用户是否可以选择某个据点还车。本次服务站点管理改造为“一个据点多个服务能力”后，后端仍需保持该 API 的兼容口径，不得仅按服务能力和服务状态返回可还车据点，而忽略空闲车位判断；也不得改变空闲车位字段的含义、扣减时机、返回规则或空位为 0 时不可还车的判断逻辑，避免影响旧版 APP 的甲租乙还正常还车流程。

服务站点管理页面左侧组织树只展示车行和门市两级，用于切换查看范围。具体据点统一在右侧列表中展示、搜索、筛选和编辑，不在左侧组织树重复展示第三层据点节点。

左侧组织区域需要提供“全部”入口。点击“全部”后，右侧列表展示当前租户下全部据点，并保留右侧搜索、服务能力和状态筛选能力。

## 9. 车辆与订单影响

车辆仍挂在具体据点下。据点的服务能力用于判断车辆是否可以在对应业务模式中上架、派车或被用户选择。

| 场景           | 规则                                                                          |
|:------------ |:--------------------------------------------------------------------------- |
| 分时租赁同站租还新订单  | 一般预约入口只展示据点类型“一般预约开放”为开启、具备同站租还服务能力且服务状态启用的据点和车辆。                               |
| 分时租赁甲租乙还新订单  | 一般预约入口只展示据点类型“一般预约开放”为开启、具备甲租乙还服务能力且服务状态启用的据点和车辆。取车和还车据点可以为同一据点或不同据点。          |
| 甲租乙还还车       | 只展示具备甲租乙还服务能力、服务状态启用且当前可还车空闲车位大于 0 的据点。                                     |
| 合作通路租車券     | 批次按适用据点类型和适用据点控制可用范围。租車券模式可选择一般预约可用或不开放一般预约的据点类型；一般预约入口仍只按“一般预约开放”为开启的类型查询。              |
| 门市租车新订单      | 客户选择门市；后台派车、备车和归还登记使用门市下具备门市租车能力的据点。                                        |
| 门市租车还车       | 客户还给还车门市，门店验车后再登记最终停放据点。服务站点管理列表中的车位概况只用于门店判断停车场是否还有空位，不直接作为门市租车是否可还车的拦截条件。 |
| 24 小时自助租车新订单 | 客户选择具体自助据点；据点必须具备 24 小时自助租车能力且服务状态启用。                                       |
| 24 小时自助租车还车  | 客户可以还到同门市下所有具备 24 小时自助租车能力且服务状态启用的据点。同门市下不同自助据点还车不产生调度费。                    |
| 公务用车预约       | 只展示具备公务用车能力且服务状态启用的据点或车辆。                                                   |

服务能力关闭或暂停后，只影响新订单、新预约、新上架和实时可选列表。已创建订单继续保留，不自动取消、不自动改派、不自动改写订单快照。

若订单关联的门市或据点在订单创建后被关闭或暂停：

- 营运后台和 APP 需要展示对应据点状态。
- 关闭提示文案统一为“据点已关闭，请联系客服处理”。
- 暂停提示文案统一为“据点已暂停，请联系客服处理”。
- 本期不自动改门市、不自动改据点、不自动转单、不自动取消，由客服或门店人工处理。

甲租乙还车位与站点分布查看沿用既有独立功能，本页面列表不重复展示该能力。

## 10. 权限与操作记录

服务站点管理按车行、门市和据点数据权限控制可见范围。

以下操作需要写入业务操作记录：

- 新增、编辑、停用、启用据点。
- 新增、关闭、暂停或恢复某个服务能力。

- 修改门市租车或 24 小时自助租车服务配置。
- 修改公务用车签约企业客户。
- 修改据点车位容量、电子围栏、地址、经纬度和进出方式。

操作记录需要包含变更前后值、操作人、操作时间。

## 11. 原型对应

对应营运端：

- `02_原型和PRD/原型/营运端/stations.html`

原型表达：

- 左侧组织树只保留车行和门市两级，具体据点统一在右侧列表中展示。
- 左侧提供“全部”入口，支持从车行或门市范围快速回到全部据点列表。
- 一个据点可配置多个服务能力。
- 服务能力可启用或暂停。
- 甲租乙还启用后默认同时支持取车和还车。
- 门市租车和 24 小时自助租车在客户面对层级上的差异。
- 列表可筛选服务能力、服务状态和据点状态；服务能力筛选按是否具备能力过滤，暂停能力仍需出现在结果中并展示状态标签。
- 地址字段在原型中只示意台湾模式样式；实际开发时必须按 `国际化地址模型与地址组件规则.md` 中的租户地址模式渲染。

原型只用于展示页面结构、字段配置和列表口径，不实现历史旧据点合并、复杂联动校验或真实数据保存能力。

## 12. 据点模型改造影响清单

本次据点模型从“每种服务类型各建独立据点”改造为“一个据点可同时具备多种服务能力”。所有原本按“据点绑定的服务类型”查询据点的功能，都需要改为按“据点 + 据点类型一般预约开放属性 + 服务能力 + 服务状态”共同判断。

### 12.1 数据初始化与兼容处理

上线时不做旧据点合并，不回溯修改旧订单快照，但必须完成以下最小初始化：

1. 旧同站租还据点初始化为具备“分时租赁 / 同站租还”服务能力。
2. 旧甲租乙还据点初始化为具备“分时租赁 / 甲租乙还”服务能力。
3. 旧门市租车、自助租车、公务用车据点按原服务类型初始化对应服务能力。
4. 存量据点初始化为“普通营运据点”或原有同义据点类型，并默认开启“一般预约开放”。
5. 如存在需要隔离一般预约入口的合作通路租車券据点，初始化或人工调整为对应专用据点类型，并关闭该类型的“一般预约开放”。
6. 初始化不合并同名、同地址或同停车场据点，不修改旧订单、旧车辆位置和历史统计快照。

### 12.2 统一查询条件

所有新订单、新预约、新上架、车辆调拨、行销方案、优惠券、票券批次和后台选择器查询据点时，至少需要同时判断：

| 判断项 | 规则 |
| --- | --- |
| 据点状态 | 据点整体必须为启用。 |
| 据点类型一般预约开放属性 | 一般预约入口只能查询“一般预约开放”为开启的据点类型；租車券模式按批次适用据点类型和适用据点查询。 |
| 服务能力 | 据点必须具备当前业务模式对应服务能力。 |
| 服务状态 | 当前业务模式对应服务状态必须为启用。 |
| 车辆条件 | 车辆服务类型、启用状态、营运状态和时间窗可用性仍按原业务规则判断。 |

仅用于后台历史查询、订单详情展示、操作记录和报表追溯时，可以展示已停用据点或已暂停服务能力，但需要清楚标记当前状态。

### 12.3 受影响功能

| 功能 / 场景 | 需要调整的点 |
| --- | --- |
| APP 同站租还据点与车辆查询 | 改为查询“一般预约开放 + 同站租还能力启用”的据点，再按车辆实时可用性返回车辆。 |
| APP 甲租乙还取车据点查询 | 改为查询“一般预约开放 + 甲租乙还能力启用”的据点，再按车辆实时可用性返回车辆。 |
| APP 甲租乙还还车据点查询 | 继续同时判断“甲租乙还能力启用 + 当前可还车空闲车位大于 0”，不得只按服务能力返回。 |
| 营运后台新建分时租赁订单 | 据点选择器按当前服务模式过滤，不能选择未具备对应服务能力或已暂停的据点。 |
| 车辆调拨 / 服务类型切换 | 目标据点必须具备目标服务类型对应服务能力且服务状态启用；同一据点可在多个目标服务类型下出现。 |
| 门市租车派车、备车、还车登记 | 客户面对门市，后台登记具体停放据点时，只能选择该门市下具备门市租车能力的据点。 |
| 24 小时自助租车预约与还车 | 用户面对具体自助据点，只能选择具备 24 小时自助租车能力且服务状态启用的据点。 |
| 合作通路租車券批次 | 批次选择适用据点类型和适用据点；可选择一般预约可用或不开放一般预约的据点类型，保存时校验所选据点具备同站租还能力且服务状态启用。 |
| 公务用车据点 | 公务用车只查询具备公务用车能力且服务状态启用的据点；企业自建公务车据点按公务车 PRD 单独处理。 |
| 行销方案、优惠券、折扣码 | 适用据点选择器按适用业务模式过滤据点；保存时校验所选据点至少具备对应服务能力。 |
| 统计报表 | 当前统计按新据点和服务能力查询；历史订单、历史报表和订单快照不因据点配置变化回写。 |

### 12.4 甲租乙还车位与旧版 APP 兼容

甲租乙还服务能力启用后，默认同时支持作为取车据点和还车据点，不再单独配置取车或还车开关。

甲租乙还可还车空闲车位计算口径保持不变：

```text
甲租乙还可还车空闲车位 = 甲租乙还预留车位数 - 甲租乙还当前停放车辆数
```

旧版 APP 已依赖可还车据点查询 API 中的空闲车位字段。本次改造后，后端不得改变该字段含义、扣减时机、返回规则和空位为 0 时不可还车的判断逻辑。

甲租乙还车位与站点分布查看按“具备甲租乙还服务能力的据点”展示，每个据点一行。列表中的总车位为据点级字段，甲租乙还可还车空闲车位按甲租乙还预留车位独立计算，不与服务站点管理列表的停车场空位概况混用。

### 12.5 车位口径边界

| 口径 | 用途 | 是否拦截业务 |
| --- | --- | --- |
| 服务站点管理列表车位概况 | 给门店查看停车场当前停车情况。 | 不作为门市租车是否可还车的拦截条件。 |
| 甲租乙还可还车空闲车位 | 判断甲租乙还用户是否可选择某据点还车。 | 空位为 0 时必须拦截。 |
| 租車券车辆库存提醒 | 创建批次时提示固定取还时间段内车辆是否可能不足。 | 仅提示，不锁车，不阻止保存。 |

### 12.6 权限与操作记录

以下操作必须记录变更前后值、操作人和操作时间：

1. 据点类型调整。
2. 据点启用、停用。
3. 服务能力新增、暂停、恢复。
4. 甲租乙还预留车位数调整。
5. 据点总车位、地址、经纬度、电子围栏和进出方式调整。

据点类型、据点类型一般预约开放开关、甲租乙还预留车位数和服务能力状态属于高影响配置，需要按营运端权限控制，仅允许具备服务站点管理权限的账号操作。

### 12.7 验收用例

1. 同一一般预约可用据点同时启用同站租还和甲租乙还后，两个入口都能查到该据点，但各自只返回符合车辆服务类型和时间窗的车辆。
2. 暂停同站租还能力后，同站租还新订单不可选该据点，甲租乙还仍可正常使用。
3. 甲租乙还可还车空闲车位为 0 时，APP 可还车据点列表不得返回该据点。
4. 关闭“一般预约开放”的据点类型不出现在一般预约入口；用户进入租車券模式后，系统按批次适用据点类型和适用据点查询车辆。
5. 旧同站租还、旧甲租乙还据点完成初始化后，原 APP 查询入口仍能查到对应据点。
6. 行销方案和优惠券选择适用据点时，只展示具备对应业务模式服务能力的据点。
7. 门市租车还车登记最终停放据点时，只能选择当前还车门市下具备门市租车能力的据点。
8. 服务站点管理列表车位概况变化不影响门市租车还车办理；甲租乙还还车仍按可还车空闲车位判断。
