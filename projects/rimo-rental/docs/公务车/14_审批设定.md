## 1. 页面范围

原型页面：`02_原型和PRD/原型/企业端/approval_setting.html`

页面用途：配置分公司公务车预约是否需要审批。

## 2. 数据范围

1. 企业总管理员可配置全部分公司的审批设定。
2. 分公司管理员可配置本分公司的审批设定。
3. 审批设定按员工所属分公司生效。

## 3. 配置项

| 字段 | 说明 |
| --- | --- |
| 分公司 | 配置所属分公司 |
| 是否启用预约审批 | 开启 / 关闭 |
| 审批人 | 部门主管、分公司管理员 |
| 状态 | 启用、停用 |

## 4. 审批规则

1. 未开启预约审批时，员工提交预约后直接进入已预约。
2. 开启预约审批时，员工提交预约后进入待审批。
3. 待审批订单不应被视为最终有效预约，除非当前实现已明确采用提交即暂占车辆；如采用暂占，需在订单状态和排程中明确标识。
4. 审批通过后订单进入已预约，并占用车辆排程。
5. 审批驳回后订单进入已驳回，不占用车辆排程。
6. 管理员代员工新增公务订单时，也按员工所属分公司的审批设定执行。

## 5. 通知规则

1. 员工提交申请后，推送给员工部门主管。
2. 分公司管理员可在后台审核。
3. 审批通过或驳回后，通知预约员工。

## 6. 验收标准

1. 分公司开启审批后，新建订单进入待审批。
2. 分公司关闭审批后，新建订单直接进入已预约。
3. 审批通过后车辆排程显示占用。
4. 审批驳回后车辆排程不显示占用。
5. 管理员代建订单仍遵循员工所属分公司审批规则。
