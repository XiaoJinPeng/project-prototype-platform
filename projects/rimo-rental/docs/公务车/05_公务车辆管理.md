# 企业端公务车后台功能 PRD - 公务车辆管理

| 版本 | 日期 | 调整概括 |

## 1. 页面范围

原型页面：`02_原型和PRD/原型/企业端/official_vehicles.html`

页面用途：管理企业公务车辆资料、状态、来源、所属分公司、所在据点、合约绑定和车辆使用数据。

## 2. 数据范围

1. 企业总管理员可查看和管理企业全部公务车辆。
2. 分公司管理员可查看本分公司资产车辆，并管理本分公司资产车辆基础资料。
3. 分公司管理员不可修改车辆资产归属，不可将车辆调拨给其他分公司。

## 3. 列表字段

| 字段 | 说明 |
| --- | --- |
| 车辆 | 车牌号码、车款、车辆类型 |
| 资产归属 | 车辆签约企业 / 资产归属分公司 |
| 所在据点 | 车辆当前停放据点 |
| 车辆来源 | 企业自有、营运商提供、其他租车公司等 |
| 里程数 | 当前车辆里程数 |
| 上架里程 | 车辆进入公务车服务时的里程 |
| 累计用车里程 | 当前里程数 - 上架里程 |
| 合约 / 配额 | 当前绑定合约和配额占用情况 |
| 启用状态 | 未启用、启用、封存 |
| 操作 | 详情、编辑、调拨、停用等 |

## 4. 累计用车里程

计算公式：

`累计用车里程 = 当前里程数 - 上架里程`

规则：

1. 当前里程数和上架里程均存在时计算。
2. 计算结果小于 0 时展示为 0，并提示数据异常。
3. 当前里程数缺失时展示“待补资料”。
4. 累计用车里程用于观察车辆投入公务车服务后的使用量，不等同于车辆总里程。

## 5. 新增车辆入口

1. 公务车辆管理页提供“新增车辆”按钮。
2. 新增车辆优先使用弹窗方式完成。
3. `official_vehicle_add.html` 可作为独立原型入口暂时保留，字段规则与新增车辆弹窗一致。
4. 新增 / 编辑车辆的清洁保养资料区需要提供保养母版、清洁母版、上次保养里程、上次保养时间、上次清洁时间。
5. 保养母版和清洁母版不是新增车辆必填项；未选择时，车辆在清洁保养提醒中按对应类型标记为资料待补。
6. 保养母版和清洁母版可选范围必须按车辆资产归属分公司过滤，只允许选择企业通用母版或当前资产归属分公司的专用母版。
7. 车辆资产归属分公司变更后，如原已选母版不属于新资产归属分公司的可选范围，系统清空对应母版并提示重新选择。

## 6. 车辆调拨规则

1. 只有企业总管理员可以调整车辆资产归属分公司。
2. 调拨后，车辆原合约配额绑定失效。
3. 调拨到新分公司后，必须由新分公司重新选择并绑定可用合约配额。
4. 调拨前已存在的预约继续有效，按原订单快照执行。
5. 调拨后不再开放原分公司员工创建该车辆的新预约，除非新分公司据点权限和用车范围允许。

## 7. 验收标准

1. 分公司管理员看不到其他分公司资产车辆的编辑入口。
2. 总管理员可以调整车辆资产归属。
3. 车辆累计用车里程按公式自动计算。
4. 车辆来源可用于列表筛选和通知设定范围匹配。
5. 新增车辆入口打开的字段与新增车辆 PRD 一致。
6. 新增 / 编辑车辆时选择的保养母版、清洁母版与清洁保养提醒页面“车辆套用”页签展示一致。
