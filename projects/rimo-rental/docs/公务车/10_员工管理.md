## 1. 页面范围

原型页面：`02_原型和PRD/原型/企业端/employee_management.html`

页面用途：查看和维护已加入企业并通过审核的员工资料。

## 2. 数据范围

1. 企业总管理员可查看企业全部员工。
2. 分公司管理员可查看本分公司员工。
3. 员工所属分公司决定订单归属、审批归属和统计归属。

## 3. 列表展示规则

员工管理列表应合并可合并信息，减少列数。

| 列 | 展示内容 |
| --- | --- |
| 员工 | 员工姓名、身份证号 |
| 联系方式 | 手机号、Email |
| 组织 | 分公司、部门 |
| 角色 / 权限 | 员工、主管等 |
| 状态 | 启用、停用 |
| 最近记录 | 最近登录时间、更新时间 |
| 操作 | 详情、编辑、停用等 |

页面不展示员工账号。姓名下方仅保留身份证号，不展示“账号：”“证号：”等标签前缀。

## 4. 操作规则

1. 企业总管理员可编辑企业全部员工组织关系和状态。
2. 分公司管理员可编辑本分公司员工基础资料和部门归属。
3. 员工停用后，不允许新增公务车预约。
4. 员工已有未完成订单时，停用不自动取消订单，企业内部自行处理。

## 5. 与其他页面关系

1. 新增公务订单选择员工时，读取员工管理中的已通过且未停用员工。
2. 员工所属分公司用于判断可用据点、审批规则和订单归属。
3. 员工所属部门用于预约审批、用车范围和部门用车统计。

## 6. 验收标准

1. 企业端员工管理不显示员工账号。
2. 员工姓名下方展示身份证号。
3. 联系方式列展示手机号和 Email。
4. 分公司管理员不能查看或编辑其他分公司员工。
