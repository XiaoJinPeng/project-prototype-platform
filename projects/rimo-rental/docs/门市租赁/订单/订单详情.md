## 1. 文档概述

### 1.1 背景

订单详情页是门市租车订单的统一作业台，承载订单全生命周期中的信息查看、业务处理、费用处理、异常登记和操作追溯。详情页不单独定义各子流程的业务规则，子流程规则以取车、还车、延长用车、改派、取消与关闭订单、费用管理等专项文档为准。

### 1.2 目标

- **状态清晰**：详情页头部展示订单主状态、支付状态、派车状态、追缴状态和风险标记。
- **入口统一**：所有订单动作从详情页进入，按钮展示规则与订单列表一致。
- **流程闭环**：取车、还车、续租、改派、费用、取消、爽约、拒绝履约、关闭订单、事故登记均在详情页形成履约事件和操作日志。
- **资金可追溯**：费用项、资金记录、预授权、退款、减免、发票和追缴记录在详情页统一查看。
- **终态只读**：订单进入 `completed`、`cancelled`、`no_show`、`rejected`、`closed` 后，不再展示履约处理入口。

### 1.3 页面入口

- 门市租车订单列表点击“详情”。
- 订单列表点击业务按钮后进入详情页并自动打开对应弹窗。
- 新建订单成功后点击“查看订单详情”。
- 费用、追缴、通知、事故等消息入口跳转订单详情页。

### 1.4 列表跳转入参

订单列表进入详情页时，需要通过 URL 参数带入当前行的模拟数据和目标动作，详情页按参数恢复订单基础信息、企业信息、结算方式、支付状态、追缴状态和默认弹窗。

| 参数 | 说明 |
| --- | --- |
| `id` | 订单号。 |
| `status` | 订单主状态。 |
| `action` | 进入详情后自动打开的动作入口，支持 `pay`、`pickup`、`return`、`extend`、`cancel`、`no_show`、`close`、`accident`、`notify`、`collect`、`fees`。 |
| `paymentStatus` | 支付/预授权状态。 |
| `billing` | 结算方式，`monthly` 表示企业月结，`customer_pay` 表示客户自付。 |
| `clientType`、`corporateName`、`corporateShortName` | 企业客户识别信息。 |
| `postChargeStatus`、`postChargeAmount`、`postChargeCount` | 完成后追缴状态、金额和项数。 |

详情页顶部需要根据入参展示主状态、支付状态、企业月结标签、追缴标签和风险标签；操作按钮只展示当前状态可执行的动作，不展示不可执行的禁用按钮。

## 2. 页面结构

### 2.1 顶部摘要区

| 区域 | 展示内容 | 规则 |
| --- | --- | --- |
| 订单基础信息 | 订单号、下单时间、来源渠道、业务线 | 业务线为门市租车时展示门市订单操作集合。 |
| 主状态 | `orderStatus` 中文状态标签 | 状态码和颜色使用《门市租车订单状态》。 |
| 金额摘要 | 订单总额、已收金额、待收金额、待退金额 | 与费用中心汇总保持一致。 |
| 客户摘要 | 客户姓名、手机号、会员等级、企业简称、风险标签 | 企业客户展示企业简称；企业月结订单额外展示“企业月结”标签；黑名单、证件异常、自助异常需高亮。 |
| 车辆摘要 | 车组、车型、车牌、派车状态 | 未派车时展示“未分配”。 |
| 行程摘要 | 预计取车时间、预计还车时间、实际取车时间、实际还车时间 | 逾期未取、逾期未还时高亮对应时间。 |

顶部摘要区和详情页所有历史业务信息默认读取订单历史信息快照，包括客户姓名、企业简称、取还门店名称、门店地址、车组名称、车型名称、车牌号和订单来源。主数据 ID 仅用于跳转当前资料、权限校验和业务动作，不用于覆盖订单快照展示。若当前主数据与订单快照不一致，可展示“当前资料已变更”提示，并保留订单下单时信息作为主展示。

### 2.2 主操作区

详情页顶部保留高频业务按钮和更多操作下拉。按钮由订单主状态、子状态和权限共同决定。

| 入口 | 展示条件 | 点击结果 |
| --- | --- | --- |
| 支付 | `pending_payment` | 打开下单预授权押金确认弹窗，预授权成功后订单进入 `reserved`。 |
| 取车 | `reserved`、`pickup_overdue` | 打开取车向导，创建取车操作锁，订单进入 `inspecting`。 |
| 继续取车 | `inspecting` | 回到取车向导，保留已录入草稿。 |
| 延长用车 | `renting`、`renewing` | 打开延长用车弹窗。 |
| 补办续租 | `overdue` | 打开延长用车弹窗，并执行逾期续租风控校验。 |
| 还车 | `renting`、`renewing`、`overdue` | 打开还车向导。 |
| 继续结算 | `settlement_pending`、`payment_due`、`refund_pending` | 打开还车向导费用结算步骤。 |
| 费用 | 非终态、`completed` | 打开费用管理弹窗。终态订单仅允许查看，追缴费用按费用管理权限处理。 |
| 改派 | `reserved`、`pickup_overdue`、`inspecting`、`renting`、`renewing`、`overdue` | 打开改派弹窗。存在未结清费用时先处理费用。 |
| 改还车门市 | `reserved`、`pickup_overdue`、`inspecting`、`renting`、`renewing`、`overdue`、`accident_processing` | 打开改还车门市/转单弹窗。确认前检查原还车门市后续预约冲突，存在冲突时先快捷改派后续订单。 |
| 改还车据点 | `reserved`、`pickup_overdue`、`inspecting`、`renting`、`renewing`、`overdue`、`accident_processing` | 打开改还车据点弹窗。适用于 24 小时自助订单或需要指定具体还车据点的场景；可选择到具体据点。若目标据点所属门市变化，先更新订单当前还车门市，再按门市口径校验冲突和计算差额。 |
| 取消订单 | `pending_payment`、`reserved`、`pickup_overdue` | 打开取消订单弹窗。 |
| 标记爽约 | `pickup_overdue` 且超过保留时长 | 打开爽约处理弹窗。 |
| 关闭订单 | `pending_payment`、`reserved`、`pickup_overdue`、`inspecting`、`renting`、`renewing`、`overdue`、`accident_processing` | 打开关闭订单弹窗。 |
| 事故登记 | `renting`、`renewing`、`overdue`、`settlement_pending`、`payment_due`、`refund_pending`、`completed` | 打开事故/出险登记弹窗。 |
| 订单备注 | 全部状态 | 打开备注弹窗。 |
| 打印单据 | 全部状态 | 打印或下载订单相关单据。 |

### 2.3 信息分栏

订单详情页采用左侧业务信息、右侧财务信息的结构。

#### 左侧业务信息

- 客户与驾驶人信息。
- 派车信息与车辆实物信息。
- 预约资料：取车门店、订单当前还车门店、取还据点展示、取还站点状态、租期时间轴、车组、优惠方案、增值服务、客户备注。
- 取车记录：身份核验、行销方案资格核验、证明文件、押金尾款、合约签署、发车验车、发车时间、发车里程、发车油量/电量、照片和签字。
- 还车记录：订单当前还车门店、归还据点、还车时间、还车里程、还车油量/电量、车况查验、损伤记录、客户签字。
- 履约事件：统一展示订单关键动作、改派、改还车门市、改还车据点、事故、异常和完整操作日志。

#### 右侧财务信息

- 账务总览：展示合计应收、客户实收或企业挂账、待处理金额、完成后追缴金额。
- 收费明细：展示车辆租金、增值服务、优惠抵扣、续租费用、还车结算费用、取消费、爽约费、拒绝履约费用、追缴费用。
- 资金记录：展示预授权冻结、预授权释放、预授权扣款、订单收款、订单退款、企业月结、减免。
- 发票记录：展示普票、专票、开票状态、开票金额、关联费用项、已开票金额、待开票金额和未开票金额。
- 费用审计：展示费用新增、编辑、删除、收款、退款、减免、通知、提醒、导出等记录。
- 企业月结订单展示 `monthly_bill` 记账记录，不展示客户支付、预授权、尾款收款和原路退款操作；非月结企业预订展示企业简称和企业资格优惠，资金操作与个人预订一致。

### 2.4 履约事件模块

履约事件模块用于承载订单生命周期中的关键业务事件，替代原来分散展示的改派记录、事故记录和订单操作记录。模块按时间倒序展示事件，默认展示紧凑摘要，完整审计信息通过“操作日志”查看。

#### 2.4.1 分类 Tab

| Tab | 展示范围 | 展示规则 |
| --- | --- | --- |
| 全部 | 订单关键动作、改派、改还车门市、事故、异常、费用相关事件 | 使用紧凑事件卡片展示，避免重复铺开展示详情。 |
| 异常 | 事故、取消、爽约、拒绝履约、关闭订单、自助取还车异常等风险事件 | 未结案事故和未处理自助异常优先展示，并在模块顶部展示风险提示。 |
| 改派 | 所有改派和车辆分配事件 | 展示原车辆、新车辆、改派原因、差价金额、差价处理方式。 |
| 事故 | 所有事故/出险事件 | 展示事故描述、责任归属、处理状态、预估/最终费用、照片和关联费用入口。 |
| 操作日志 | 完整订单操作记录 | 以表格展示时间、操作、详情、操作员和备注，用于审计追溯。 |

#### 2.4.2 事件卡片字段

| 事件类型 | 字段 | 规则 |
| --- | --- | --- |
| 通用事件 | 类型标签、标题、摘要、时间、操作人、备注 | 所有事件按时间倒序排列。 |
| 改派事件 | 原车辆、新车辆、原因、差价、差价处理方式、操作人、操作时间 | 差价处理方式包含取车补收、还车结算、企业月结挂账、免费升等、不调整费用。 |
| 改还车门市事件 | 原还车门市、新还车门市、调度费差额、变更原因、后续预约冲突处理结果 | 存在后续预约冲突时，展示被改派的后续订单号和新车辆。 |
| 改还车据点事件 | 原还车门市/据点、新还车门市/据点、调度费差额、变更原因、后续预约冲突处理结果 | 24 小时自助订单优先展示具体据点；若目标据点跨门市，同时展示门市变化。 |
| 事故事件 | 事故描述、责任归属、处理状态、费用金额、照片、关联费用入口 | 未结案事故展示风险色；已结案事故降为普通记录。 |
| 自助异常事件 | 自助阶段、车机指令类型、指令状态、失败原因、重试次数、资金处理结果、人工处理结果 | 用于展示 24 小时自助取车开锁失败、还车落锁失败、车机结果未知和人工确认记录。 |
| 操作日志 | 时间、操作、详情、操作员、备注 | 操作日志保留完整原始记录，履约事件列表中不重复展示已独立形成改派事件或事故事件的同类操作记录。 |

#### 2.4.3 高度与密度控制

- 履约事件模块的事件列表固定最大展示高度，超出后在模块内部滚动，不拉长订单详情整页。
- “全部”和“异常”只展示紧凑摘要；进入“改派”Tab 后展开车辆前后对比；进入“事故”Tab 后展开责任、状态、费用和照片。
- “操作日志”表格固定最大展示高度，超出后表格内部滚动。
- 模块头部展示关键计数，包括未结案事故数、改派数和关键事件总数。
- 事件列表为空时展示空状态，不占用过高页面空间。

### 2.5 右侧财务区

右侧财务区用于快速查看订单的账务结果、计费来源和资金流向，完整费用处理仍从费用管理弹窗进入。右侧财务区按照账务总览、计费快照、收费明细、资金记录、发票记录、费用审计的顺序展示。

#### 2.5.1 账务总览

| 字段 | 客户自付订单 | 企业月结订单 |
| --- | --- | --- |
| 合计应收 | 展示订单全部有效费用合计，排除已取消、已释放、已退款和已减免费用。 | 展示订单全部有效费用合计，作为企业最终挂账依据。 |
| 客户实收/企业挂账 | 客户自付订单展示客户实收金额。 | 企业月结订单展示企业挂账金额。 |
| 待处理金额 | 展示待收或待退的未平金额。 | 固定展示客户应收为 0。 |
| 完成后追缴 | 展示完成后新增且未结清的追缴费用金额。 | 展示完成后新增且进入企业月结的追缴费用金额。 |

#### 2.5.2 计费快照

计费快照分为两层展示：

- **订单级基线快照**：展示订单创建时锁定的价格、行销方案和业务规则版本，后台营运可在详情页直接确认订单按哪一套配置生成。
- **事件级快照链**：仅在后续事件改变订单商业条件、计费结果或结算结果时生成，用于展示“哪一次事件改了什么、差额怎么算、差额进哪里”。

纯履约动作，例如派车、改派、同车组换车、换据点、备车、验车、改还车门市、改还车据点，只写入履约事件和操作日志，不进入计费快照链。

| 快照项 | 展示规则 |
| --- | --- |
| 标准价版本 | 展示 `priceVersionId`、车组名称、标准日租金、里程规则和生效时间。 |
| 行销方案版本 | 展示 `marketingPlanVersionId`、方案名称、适用客户资格、是否需要到店核验、证明文件说明、平日折扣、假日折扣和优惠结果。未使用方案时展示“未使用行销方案”。 |
| 租赁规则版本 | 展示 `rentalRuleVersionId`、起租规则、不足 1 小时处理、5 小时转日租阈值、预授权和待支付资源保留规则。 |
| 增值服务与里程 | 展示增值服务价格版本、里程规则版本和订单创建时的服务明细。 |
| 企业月结 | 企业月结订单同样展示快照版本；客户支付、预授权、补缴、退款类规则标记为不执行，最终费用进入企业账单。 |

事件级快照链至少支持以下示例：

| 事件类型 | 是否生成事件级快照 | 展示重点 |
| --- | --- | --- |
| 取车时更换行销方案 | 是 | 原方案、新方案、资格核验原因、优惠差额、取车补收或减收结果。 |
| 续租 | 是 | 原还车时间、新还车时间、新增租期、事件差额、结算去向。 |
| 改租期 | 是 | 原租期、新租期、影响的费用项和差额。 |
| 改车组 | 是 | 原车组、新车组、价格差额、差额处理方式。 |
| 改派导致改车组/升等/降等 | 是 | 原车组、目标车组、原车辆、目标车辆、差价和差价处理方式；若免费升等导致差额为 0，仍需保留事件快照。 |
| 改还车门市或改还车据点且产生异地还车费/调度费差额 | 是 | 原还车门市/据点、新还车门市/据点、异地费或调度费差额、结算去向。 |
| 逾时重计费/还车重计费 | 是 | 实际触发的逾时时长或重计费结果、差额金额、待补缴/待退款结果。 |
| 同车组派车/改派/换车/换据点 | 否 | 只在履约事件展示，不进入计费快照链。 |

事件级快照详情至少展示以下字段：

| 字段 | 展示要求 |
| --- | --- |
| 快照编号 | 使用独立事件快照编号，和订单基线快照区分。 |
| 事件类型 | 明确是续租、改租期、改车组、改还车门市、改还车据点、行销方案调整或还车重计费。 |
| 触发时间 | 展示事件实际确认时间。 |
| 触发原因 | 展示客户要求、资格不符、车辆不可交付、门店协调、站点关闭等业务原因。 |
| 命中版本 | 展示本次事件命中的标准价版本、行销方案版本、租赁规则版本、服务价格版本。 |
| 变更前后 | 用“变更前 -> 变更后”形式展示受影响的商业条件。 |
| 差额结果 | 展示原金额、事件差额、当前应收/应退或企业挂账结果。 |
| 结算去向 | 明确进入取车补收、还车补缴、还车退款、企业月结或不调整费用。 |
| 操作人 | 展示确认该事件的人。 |
| 附件/证明 | 行销方案核验、客服协商、站点异常等场景可挂证明文件或备注。 |

计费快照默认展示摘要，支持查看完整快照明细。版本号必须可点击，点击后以弹窗展示该版本内的完整设定值；版本详情需要沿用对应设定页面的字段命名和展示结构。标准价版本按门市标准价表展示，行销方案版本按方案详情和规则详情展示，租赁规则版本按业务规则分组展示，增值服务和里程版本按对应配置项展示。具备独立管理页面的版本可同时提供跳转入口。事件级快照详情按“触发原因、商业条件变化、差额结果、结算去向”展示，并按时间顺序挂在订单级基线快照下方。

ZT2405060003：改车组 / 免费升等

ZT2405040003：取车更换行销方案

ZT2405020008：续租

ZT2405010022：逾时重计费

#### 2.5.3 收费明细

- 右侧收费明细只展示关键费用摘要，不承载完整费用处理。
- 费用项展示费用名称、费用说明、费用阶段、费用状态、金额、待收金额或企业入账标识。
- 费用项数量较多时，右侧收费明细在模块内部滚动，不拉长详情页。
- 点击“管理费用”进入费用管理弹窗，费用新增、编辑、删除、收款、退款、减免、通知、提醒均在费用管理弹窗内完成。

#### 2.5.4 资金记录

- 资金记录来源为订单交易流水，不再只读取页面上的已收款记录。
- 客户自付订单展示预授权冻结、预授权释放、预授权扣款、客户收款、客户退款和费用减免。
- 企业月结订单只展示 `monthly_bill` 企业挂账记录和内部减免记录，不展示客户支付、预授权、尾款收款和原路退款记录。
- 每条资金记录展示交易类型、渠道、关联费用、交易时间、金额和交易流水号。
- 点击“查看全部”进入完整资金流水弹窗。

#### 2.5.5 发票记录

- 发票记录展示已开票金额、待开票金额和未开票金额。
- 每张发票展示发票号、发票类型、开票状态、开票时间、开票金额和关联费用项。
- 客户自付订单按客户已结算费用申请开票。
- 企业月结订单按企业月结账单申请开票，发票金额以企业账单确认金额为准。
- 完成后追缴费用开票时，需关联对应追缴费用项。

#### 2.5.6 费用审计

- 费用审计只记录费用相关动作，包含费用新增、编辑、删除、收款、退款、减免、通知、提醒、导出。
- 履约事件中的操作日志记录订单主流程动作；费用审计记录费用域内的明细变更。
- 右侧费用审计只展示最近记录，完整日志在费用管理弹窗内查看。

## 3. 状态驱动规则

### 3.1 主状态与入口

| 主状态 | 详情页主按钮 | 更多操作 | 说明 |
| --- | --- | --- | --- |
| `pending_payment` | 支付 | 取消订单、关闭订单、备注、打印 | 预授权押金确认成功后进入 `reserved`。 |
| `reserved` | 取车、改派 | 取消订单、关闭订单、备注、打印 | 等待客户到店。 |
| `pickup_overdue` | 取车、改派 | 取消订单、标记爽约、关闭订单、备注、打印 | 仍可继续取车，也可取消或标记爽约。 |
| `inspecting` | 继续取车、改派 | 关闭订单、拒绝履约、备注、打印 | 取车作业已被接管，操作锁生效。 |
| `renting` | 延长用车、还车、改派 | 关闭订单、事故登记、备注、打印 | 车辆已交付。 |
| `renewing` | 延长用车、还车、改派 | 关闭订单、事故登记、备注、打印 | 已进入续租周期。 |
| `overdue` | 补办续租、还车、改派 | 关闭订单、事故登记、备注、打印 | 需优先联系客户和处理风控。 |
| `accident_processing` | 事故处理、费用 | 关闭订单、备注、打印 | 出险未闭环前不进入完成。 |
| `settlement_pending` | 继续结算、费用 | 事故登记、备注、打印 | 车辆已还，等待账单确认。 |
| `payment_due` | 继续结算、收款、费用 | 事故登记、备注、打印 | 应收余额未平。 |
| `refund_pending` | 继续结算、退款、费用 | 事故登记、备注、打印 | 应退余额未处理。 |
| `completed` | 费用、追缴、发票 | 事故登记、备注、打印 | 已归档，后续费用走追缴。 |
| `cancelled` | 详情 | 备注、打印 | 终态，只读。 |
| `no_show` | 详情 | 备注、打印 | 终态，只读，费用中心保留爽约费。 |
| `rejected` | 详情 | 备注、打印 | 终态，只读，保留拒绝原因。 |
| `closed` | 详情 | 备注、打印 | 终态，只读，保留关闭原因和退款记录。 |

### 3.2 终态只读规则

- `cancelled`、`no_show`、`rejected`、`closed` 不展示取车、还车、续租、改派、取消、爽约、拒绝履约和关闭订单入口。
- `completed` 不展示取车、还车、续租、改派、取消和关闭订单入口。
- `completed` 允许新增追缴费用、查看费用、查看发票、补充事故记录和打印单据。
- 终态订单允许新增备注，但备注不改变订单主状态。

## 4. 强流程弹窗规则

### 4.1 待支付弹窗

- `pending_payment` 点击“支付”时，打开下单预授权押金确认弹窗。
- 弹窗展示订单总额、本次预授权金额、支付渠道、线下凭证号和确认按钮。
- 预授权成功后写入 `pre_auth_freeze` 资金记录，支付状态更新为 `deposit_paid`，订单主状态更新为 `reserved`，并写入操作记录。
- 支付失败、关闭弹窗或未继续操作时，订单保持 `pending_payment`，支付状态保持 `unpaid`，不写入有效资金记录。
- 企业月结订单不展示待支付弹窗，不展示客户支付和预授权入口。

### 4.2 取车弹窗

- 从 `reserved`、`pickup_overdue` 点击取车时，订单立即进入 `inspecting` 并创建取车操作锁。
- 取车窗口关闭且未确认发车时，订单保持 `inspecting`，详情页展示“继续取车”和操作锁信息。
- 操作锁超时后，具备操作锁释放权限的人员可释放操作锁；释放后订单先回到进入取车前状态，再按当前时间重新判定 `reserved`、`pickup_overdue` 或 `no_show`。
- 取车办理中触发资质、证件、行销方案资格、资金校验或风控拦截时，可在取车弹窗内确认拒绝履约，订单进入 `rejected`。客户自付订单的资金校验包含预授权和尾款；企业月结订单的资金校验为确认企业月结结算方式。
- 24 小时自助租车订单的取车弹窗中，发车里程、发车油量/电量默认展示车机最新实时值，并展示同步时间提示；后台人员可按现场核对结果手动修改，确认发车后以最终确认值写入取车记录和取车展示快照。未获取车机实时值时，页面提示后允许人工录入继续。

### 4.3 还车弹窗

- 从 `renting`、`renewing`、`overdue` 点击还车时，进入还车验车步骤。
- 验车数据提交后，订单进入 `settlement_pending`，车辆视为已归还。
- 客户自付订单取车时已补足原订单租金，还车弹窗的费用结算页只展示和计算还车阶段差额费用；取车已收租金只作为退款渠道依据，不在还车差额中重复抵扣。
- 客户自付订单费用结算后 `balanceDue > 0` 时进入 `payment_due`；`balanceDue < 0` 时进入 `refund_pending`；`balanceDue = 0` 时保持 `settlement_pending`，由门店确认还车。
- 企业月结订单不展示客户收款和退款入口，还车结算页按原订单租金、续租费用、还车差额费用、减免后的最终应收展示企业挂账金额。
- 客户自付账单平衡并确认还车后，订单进入 `completed`；企业月结挂账确认后，订单进入 `completed`。
- 24 小时自助租车订单的还车弹窗中，还车里程、还车油量/电量默认展示车机最新实时值，并展示同步时间提示；后台人员可按现场核对结果手动修改，提交验车后以最终确认值写入还车验车记录和还车结算快照。未获取车机实时值时，页面提示后允许人工录入继续。

### 4.4 费用弹窗

- 费用中心展示订单全部费用项和资金记录。
- 费用处理结果驱动 `pending_payment`、`payment_due`、`refund_pending`、`completed` 的状态流转。
- 追缴费用只改变追缴子状态，不改变 `completed` 主状态。
- 企业月结订单新增费用时，费用项状态为 `monthly_billing`。租期中费用随还车结算汇总挂账；已完成订单新增追缴费用时，系统直接生成 `monthly_bill` 交易记录，不展示客户通知支付、收款或退款入口。

### 4.5 改派弹窗

- 改派不直接改变订单主状态。
- 发车前改派影响订单待收尾款，统一在取车补收。
- 用车中改派影响还车结算，统一在还车结算处理。
- 免费升等必须填写原因并写入操作记录。

### 4.6 事故登记弹窗

- 事故登记用于记录租期内、还车结算中或完成后补录的事故/出险信息。
- 履约中订单登记事故后进入 `accident_processing`。
- 还车结算中登记事故时，订单保持当前结算状态，同时增加事故风险标记和事故费用线索。
- 已完成订单补录事故时，主状态保持 `completed`，后续费用通过追缴处理。
- 企业月结订单补录客户责任事故费用时，费用项状态为 `monthly_billing`；订单已完成时同步生成 `monthly_bill` 交易记录，订单未完成时纳入后续还车月结挂账。

## 7. UI/UX 规则

- 详情页顶部状态与操作按钮保持吸顶，便于门店作业时随时查看当前状态。
- 主按钮只展示当前状态可执行的业务动作，不可执行动作隐藏或置灰。
- 强流程弹窗使用大尺寸弹窗，步骤进度固定展示。
- 取车、还车、改派、事故、费用、取消/关闭弹窗关闭时，必须给出状态影响提示。
- 取还站点需要展示门市状态和据点状态；关联站点关闭或暂停时，在站点信息下方展示“站点已关闭，请联系客服处理”或“站点已暂停，请联系客服处理”。
- 费用相关金额统一使用正负方向和颜色表达：应收为红色，应退为绿色，已平为绿色或灰色。
- 终态订单头部状态降低视觉权重，但保留费用、记录和单据查看能力。
- 履约事件按时间倒序展示，重要状态变化使用状态色标记。
- 履约事件列表固定高度并在模块内滚动，不能因事件数量增加导致详情页整体被明显拉长。
- 操作日志作为履约事件的子分类展示，保留完整审计追溯，不作为主流程信息平铺展示。

## 8. 验收标准

| 场景 | 验收标准 |
| --- | --- |
| 状态入口 | 不同主状态下，详情页只展示该状态允许执行的按钮。 |
| 站点关闭提示 | 取还门市或据点关闭、暂停后，详情页取还站点信息处明确展示状态标签与联系客服提示文案。 |
| 列表跳转 | 从列表点击取车、还车、续租、收款、退款、取消、爽约、通知客户、追缴收款、查看费用时，进入详情页并打开对应入口。 |
| 取车中断 | 取车窗口关闭后订单保持 `inspecting`，详情页展示“继续取车”和操作锁信息。 |
| 还车中断 | 验车提交后关闭还车窗口，订单按待结金额进入 `settlement_pending`、`payment_due` 或 `refund_pending`。 |
| 待支付处理 | `pending_payment` 订单完成下单预授权押金后进入 `reserved`；失败、关闭弹窗或未继续操作时保持 `pending_payment`。 |
| 企业月结展示 | 企业月结订单金额摘要展示“企业月结”，还车结算展示最终企业挂账金额，取消/爽约/拒绝履约和完成后追缴展示 `monthly_bill` 记录，详情页不展示支付、补收尾款、预授权处理、收款和退款入口。 |
| 右侧财务区 | 账务总览、计费快照、收费明细、资金记录、发票记录、费用审计均可在右侧快速查看；客户自付订单展示客户实收和待处理金额，企业月结订单展示企业挂账金额且客户应收为 0。 |
| 资金记录 | 资金记录读取完整交易流水，客户自付订单展示预授权、收款、退款和减免，企业月结订单只展示 `monthly_bill` 企业挂账和内部减免。 |
| 终态只读 | `cancelled`、`no_show`、`rejected`、`closed` 不展示履约处理入口。 |
| 完成后追缴 | `completed` 订单新增追缴费用后，主状态保持 `completed`；客户自付订单金额列展示追缴待通知、已通知、待支付或已结清，企业月结订单金额列展示月结已挂账。 |
| 权限控制 | 店员不能执行关闭订单、标记爽约、拒绝履约、退款、减免和免费升等。 |
| 履约事件 | 详情页统一通过履约事件展示关键动作、改派、事故和异常；事件数量较多时列表在模块内滚动，不拉长详情页。 |
| 操作日志 | 每次状态变化、费用处理、预授权处理、事故登记和改派确认均写入操作日志。 |
| 费用一致 | 详情页金额摘要、费用中心汇总、订单列表金额状态保持一致。 |
