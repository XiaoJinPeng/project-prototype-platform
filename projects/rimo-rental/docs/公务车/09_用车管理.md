## 1. 页面范围

原型页面：`02_原型和PRD/原型/企业端/car_use_management.html`

页面用途：配置指定车辆的可使用员工或部门范围。

## 2. 业务口径

1. 用车管理用于限制部分车辆只允许特定员工或部门预约。
2. 未配置用车范围的车辆，按员工所属分公司可用据点、车辆状态、合约配额和排程冲突正常开放。
3. 用车范围限制不覆盖车辆启用状态、所在据点、合约配额、时间冲突等基础规则。

## 3. 数据范围

1. 企业总管理员可管理企业全部车辆用车范围。
2. 分公司管理员可管理本分公司资产车辆的用车范围。
3. 分公司管理员不可管理其他分公司资产车辆。

## 4. 列表字段

| 字段 | 说明 |
| --- | --- |
| 规则名称 | 用车范围规则名称 |
| 适用分公司 | 规则所属分公司 |
| 适用车辆 | 规则绑定车辆 |
| 可用对象 | 可预约该车辆的员工或部门 |
| 状态 | 启用、停用 |
| 操作 | 编辑、停用、启用 |

## 5. 新增 / 编辑弹窗

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 规则名称 | 是 | 便于管理员识别规则 |
| 适用分公司 | 是 | 选择规则所属分公司 |
| 适用车辆 | 是 | 可多选车辆 |
| 可用对象类型 | 是 | 员工、部门 |
| 可用对象 | 是 | 按对象类型选择员工或部门 |
| 状态 | 是 | 启用、停用 |

## 6. 匹配规则

1. 车辆存在启用状态的用车范围规则时，仅允许规则内员工或部门预约。
2. 选择部门时，部门下新增员工自动继承该部门车辆使用资格。
3. 员工同时命中多个规则时，只要命中任一规则即可预约该车辆。
4. 规则停用后，该规则不再参与预约校验。
5. 车辆无启用规则时，按普通公务车可用规则判断。

## 7. 验收标准

1. 配置指定车辆只给某部门使用后，其他部门员工不可预约该车辆。
2. 配置指定车辆只给某员工使用后，其他员工不可预约该车辆。
3. 停用规则后，车辆恢复按普通公务车规则判断。
4. 分公司管理员不能配置其他分公司资产车辆。
