# 页面导入导出说明

## 定位

页面导入导出是本地工程工具，用来衔接需求阶段的 HTML 原型与工程阶段的 Vue 页面。

- 导入：只接收由 `templates/html-prototype-page.html` 创建的规范 HTML，并生成工程内的 Vue 页面、路由和菜单登记。
- 导出：从当前页面注册表选择一个或多个页面，构建成可直接打开的独立 HTML；公共运行依赖通过 CDN 加载。
- 导出的运行包用于演示和交付，不作为 Vue 源码重新导回。
- HTML 直读是独立的第二种页面来源，不改变导入导出流程；直读页面只进入原型路由，不参与 Vue 页面导出。
- HTML 直读目录在项目配置中按客户端设置；每个客户端可以使用不同的外部 HTML 文件夹。
- 历史 HTML 页面不纳入自动清理、自动迁移或批量重构范围。

## 使用入口

启动工程开发服务后，打开首页，在底部“工程工具”中进入“页面导入导出”，或直接访问：

```text
/tools/page-transfer
```

```powershell
npm run dev
```

导入和导出都使用当前 Vite 开发服务提供的本地 Node 能力，不需要额外启动第二个服务。Node 只在本机工程侧使用，导出的演示包不要求接收方安装 Node，但打开导出 HTML 时需要能够访问 CDN。

## 导入流程

1. 在 HTML 原型阶段复制 `templates/html-prototype-page.html`，只编辑模板规定的 `[AI-EDIT]` 区域。
2. 页面定案后，在页面导入导出工具中上传 HTML。
3. 工具检查 Manifest、页面内容区、业务内容区、覆盖层、页面逻辑和页面专属样式。
4. 选择目标项目，并选择“新增页面”或“替换页面”。
5. 新增时填写客户端、路由片段、菜单分组、菜单名称和图标；替换时先选择原页面，系统会自动带出这些信息，仍允许修改。
6. 点击导入。新增会生成新 Vue 页面；替换会先把原 Vue 文件和原页面定义保存到项目包的 `.backups/pages`，再写入新页面。
7. 重新载入开发服务后，页面会出现在对应路由和菜单中。被替换页面的备份可在“路由菜单管理”中还原。

导入时页面内容、弹窗、抽屉、页面逻辑、模拟数据和专属样式作为一个整体处理，不允许只选择部分业务区块。原因是区块之间可能共享状态、计算属性和事件方法，任意拆选容易造成页面可编译但交互失效。

## 导入校验边界

必须满足：

- `templateVersion` 为 `1`。
- `pageKey` 使用小写 kebab-case。
- `client` 使用小写 kebab-case，并且必须存在于所选项目的 `project.json.clients`。
- `routePath` 使用 `/{client}/{path}`。
- 页面只能有一个实际的 `data-page-content` 节点。
- 页面包含内容区、覆盖层和页面逻辑标记。
- 不使用 `onclick` 等内联事件。
- 菜单分组和图标必须属于当前工程已有配置。

外部图片、外部脚本和不建议使用的浏览器跳转逻辑会产生警告。模板要求的页面状态、方法和模拟数据仍然保留在生成页面内部；不会自动抽取到公共 Mock 文件。

## 导出页面

1. 选择要导出的项目，再按客户端筛选页面。
2. 勾选一个或多个页面。
3. 填写导出名称。
4. 只选一页时点击“导出单页 HTML”；选择多页时点击“生成多页 ZIP”。

导出只编译本次选中的页面，以及这些页面实际依赖的页面代码、专属样式和公共外壳。Vue、Element Plus、图标、Tailwind、Vue Router、Vue I18n 和 Noto Sans SC 通过 CDN 加载，不会打入 HTML；导出运行时只保留简中基础词库，不会带入登录页和后台全量多语言词库。不会把整个工程、PRD 文档或未选页面打入导出文件。菜单会根据选择结果裁剪，只展示本次导出的页面。

只选择一个页面时，直接输出一个独立 HTML，不生成 ZIP：

```text
页面名称.html
```

选择多个页面时，每个页面生成一个可独立打开的 HTML，并统一压缩为 ZIP：

```text
页面演示包.zip
├─ operation-page-a.html
├─ operation-page-b.html
└─ enterprise-page-c.html
```

多页文件中的侧栏菜单使用相对文件名互相跳转。页面专属 CSS、JavaScript 和工程图片会随 HTML 输出，不生成 `assets` 文件夹；公共依赖通过 CDN 加载，因此接收方不需要安装 Node，但需要网络连接。页面运行文件不会连接真实接口，也不会包含真实登录、权限或数据持久化。

导出完成后，服务端会先将生成文件写入 `exports/{project-id}`，同时返回预览地址和下载地址。点击下载按钮时由浏览器通过下载接口接收文件；`exports` 只是本地预览和下载所需的服务端输出目录，可定期清理旧文件。

## 路由菜单管理

首页“工程工具”中的“路由菜单管理”用于逐个项目管理客户端页面登记：

- 新增路由时填写客户端、路由片段、页面名称、菜单分组和图标，系统会同时生成一个可运行的占位 Vue 页面。
- 编辑路由时可以修改页面名称、路由片段、菜单分组和图标；如果修改路由片段，系统会同步调整页面文件路径，但不会改动页面内容。
- 编辑菜单分组时可以新增分组、重命名已有分组或删除分组；已有分组 ID 和页面归属关系保持不变，仍被页面使用的分组不能删除。分组修改会单独生成备份。
- 删除路由时会从页面定义和菜单中移除，并将原 Vue 文件及页面定义保存到 `projects/{project-id}/.backups/pages/{backup-id}`。
- 编辑和替换导入产生的原页面备份也会出现在这里，可点击“还原”恢复原页面和原路由登记。
- 备份还原后仍保留记录并标记为“已还原”，不会自动清理历史备份。
- 删除客户端默认页面会被阻止，需要先在项目配置中更换默认页面，避免项目包进入无效状态。

## 版本和来源关系

HTML 原型是需求定案前的页面来源；导入完成后，工程中的 Vue 页面成为工程维护来源。导出包是某一时点的运行快照，不是新的原型来源。

如果导出包需要修改，应回到 Vue 页面修改后重新导出。如果需求仍在变化，应回到原 HTML 原型修改，确认定案后重新导入，并处理路由冲突。不要同时把导出 HTML 和 Vue 页面当成两套长期维护源。

## 当前限制

- 规范 HTML 可以导入任意项目包中已登记的后台客户端，不支持移动端多画面 HTML 自动拆分。
- 导入暂不自动复制 HTML 文件旁边的任意资源目录；模板页面应优先使用工程已有资源或内联资源。
- 单个独立 HTML 只内嵌页面专属代码和样式，公共依赖不内嵌，文件体积明显小于完整工程包。
- 如果客户现场无法访问 CDN，页面无法加载 Vue、Element Plus 或字体；此时应使用开发服务或另行制作离线资源包，不要直接把 CDN 版 HTML 当作离线包。
- 页面注册表写入后需要重新载入开发服务，才会重新生成路由和菜单模块。
