# 工作台（营运端）

## 1. 文档定位

本文定义营运平台新版工作台的产品方案、信息架构、任务聚合规则、角色模板、业务线展示口径、权限规则和开发验收标准。

本次工作台按 RIMO 平台最新业务架构重新设计，不沿用旧版 `workbench.html` 原型，也不沿用旧版 `Dashboard.md` 口径。旧页面仅作为历史文件保留，不作为新版工作台的设计依据。

新版工作台的定位是：

1. 营运平台登录后的默认首页。
2. 多业务线、多角色、多组织范围下的日常工作入口。
3. 面向“今天要处理什么、先处理什么、从哪里进入处理”的任务中心。
4. 连接订单、车辆、站点、调度、会员、企业、账单、公务车等模块的聚合入口。

工作台不是经营分析大屏，不是报表中心，不是广告式门户。长周期趋势、复杂图表、经营复盘、财务分析和报表导出仍放在独立统计分析页面中。

---

## 2. 产品结论

新版工作台采用“统一入口 + 四类角色模板 + 统一任务模型”的设计。

| 层级 | 结论 |
| --- | --- |
| 页面入口 | 营运平台保留一个“工作台”入口 |
| 页面定位 | 首页优先展示待办、异常、履约节奏和快捷入口 |
| 角色模板 | 营运总控、车行协同、门市履约、据点作业 |
| 数据范围 | 严格按账号的数据范围和功能权限渲染 |
| 展示重点 | 只展示当前账号有权查看、需要关注、能跳转处理的内容 |
| 业务线覆盖 | 分时租赁、门市租赁、公务车全部纳入，但只展示与当前角色有关的工作事项 |
| 交互原则 | 首页只做聚合、提醒、跳转，不在首页塞入复杂业务表单 |

## 3. 平台背景

RIMO 是 SaaS 化租车平台。平台按客户端和职责分为：

1. 客服系统：管理租户、实例、业务线授权、渠道和车机分配。
2. 营运平台：营运商、车行、门市、据点人员日常使用的主体后台。
3. 企业端：企业管理员工、企业订单、企业月结账单和公务车。
4. 前台应用：公开消费者 APP、官网下单入口、公务车 APP。

营运平台当前和规划中的业务线包括：

| 业务大类 | 业务模式 | 当前状态 | 工作台关注点 |
| --- | --- | --- | --- |
| 分时租赁 | 同站租还 | 已上线 | 自助流程异常、车辆异常、逾期、支付和风控 |
| 分时租赁 | 甲租乙还 | 已上线 | 自助流程异常、可还车位、站点状态、调度和逾期 |
| 门市租赁 | 门市租车 | 规划中 | 待交车、待收车、备车、验车、结算、支付和资格核验 |
| 门市租赁 | 24 小时自助租车 | 规划中 | 自助取还车异常、车机值、据点状态、改还车据点和人工介入 |
| 公务车 | 企业公务车 | 已上线 | 合约、配额、账单、续费、车辆上架和到期下架 |

营运平台账号按数据范围分为：

| 账号范围 | 说明 |
| --- | --- |
| 营运角色 | 营运商范围内，按功能权限查看和操作 |
| 车行角色 | 绑定车行范围内，按功能权限查看和操作 |
| 门市角色 | 绑定门市范围内，按功能权限查看和操作 |
| 据点角色 | 绑定据点范围内，按功能权限查看和操作 |

工作台必须严格遵循上述数据范围。车行、门市、据点角色不能在工作台看到超出自身范围的数据汇总、订单明细、车辆明细或经营指标。

## 4. 设计目标

### 4.1 用户目标

1. 用户登录后 10 秒内知道今天最重要的工作是什么。
2. 用户不需要先理解整个平台结构，也能从工作台进入正确页面处理问题。
3. 门市和据点人员能快速看到今天要交的车、要收的车、要备的车和现场异常。
4. 车行人员能快速看到车行范围内的履约压力、车辆整备、跨门市调度和资产异常。
5. 营运人员能快速看到全租户范围内的高优任务、异常、资金账单和业务风险。

### 4.2 系统目标

1. 通过统一任务模型聚合不同模块的待办和异常。
2. 通过角色模板降低首页复杂度，避免所有角色看到同一套大杂烩内容。
3. 通过统一跳转规则确保每条事项都有明确处理入口。
4. 通过权限和数据范围控制，避免数据越权。
5. 通过清晰的排序和优先级规则，避免开发各模块自行拼凑首页数据。

## 5. 产品原则

### 5.1 任务优先

工作台先展示“需要处理的事”，再展示“可参考的数据”。纯统计指标不能占据首屏核心区域。

### 5.2 角色优先

不同角色登录后看到不同工作台模板。营运总控关注全局异常和业务风险；车行关注车行资产与履约；门市关注当日交收车；据点关注现场停车、车辆和自助异常。

### 5.3 范围优先

页面所有数据先按账号可见范围过滤，再按业务线、状态和时间聚合。前端不能拿到无权限数据后再自行隐藏。

### 5.4 异常优先

正常业务流只在需要人工参与时进入工作台。无人值守、自助完成、没有异常的流程，不占用工作台注意力。

### 5.5 跳转闭环

工作台中的卡片、数字、任务、异常和快捷入口都必须能跳转到对应页面，并自动带入筛选条件。

### 5.6 少图表

工作台不做复杂图表。需要表达趋势、排行、复盘时，跳转到统计分析页面。首页最多使用轻量进度、时间轴、数量和状态标签。

### 5.7 不制造新业务状态

工作台不新增订单状态、车辆状态、账单状态或站点状态。工作台只读取各业务模块已有状态，按规则聚合成待办和提醒。

## 6. 页面整体结构

新版工作台采用以下结构：

1. 顶部上下文栏
2. 今日重点卡片
3. 主工作清单
4. 今日履约时间线
5. 资源健康与异常摘要
6. 快捷入口
7. 公告、通知和最近访问

### 6.1 首屏优先级

首屏内容按以下优先级展示：

1. 当前工作范围、日期和搜索。
2. 今日最重要的 4 个数字。
3. 主工作清单。
4. 今日履约时间线。
5. 关键异常和快捷入口。

### 6.2 桌面布局

桌面端按 12 栏布局：

| 区域 | 宽度 | 说明 |
| --- | --- | --- |
| 顶部上下文栏 | 12 栏 | 页面标题、范围、搜索、通知、刷新 |
| 今日重点卡片 | 12 栏 | 固定 4 张卡片 |
| 主工作清单 | 7 栏 | 页面核心区域 |
| 今日履约时间线 | 5 栏 | 今日交收车、取还车、关键时段 |
| 资源健康与异常摘要 | 7 栏 | 车辆、站点、车机、车位、资金异常 |
| 快捷入口与公告 | 5 栏 | 高频入口、公告、最近访问 |

### 6.3 移动和窄屏适配

营运平台以桌面使用为主。窄屏下按以下方式处理：

1. 顶部上下文栏折叠为两行。
2. 今日重点卡片改为横向滑动或两列。
3. 主工作清单置顶。
4. 今日履约时间线、资源健康、快捷入口按顺序向下排列。
5. 不允许出现横向滚动条。

## 7. 顶部上下文栏

### 7.1 展示字段

| 字段 | 说明 |
| --- | --- |
| 页面标题 | 固定显示“工作台” |
| 当前日期 | 显示日期、星期和当前服务器时间 |
| 当前工作范围 | 显示营运商、车行、门市或据点名称 |
| 业务线筛选 | 营运和车行模板展示；门市和据点模板默认不展示 |
| 组织范围筛选 | 营运模板支持筛选车行、门市、据点；车行模板支持筛选门市、据点 |
| 全局搜索 | 搜索订单、车辆、会员、企业、车行、门市、据点 |
| 通知中心 | 展示系统通知、业务提醒、公告提醒 |
| 刷新按钮 | 手动刷新工作台数据 |
| 个人菜单 | 个人信息、退出登录 |

### 7.2 工作范围显示规则

| 模板 | 默认范围 |
| --- | --- |
| 营运总控工作台 | 当前租户 |
| 车行协同工作台 | 当前账号绑定车行 |
| 门市履约工作台 | 当前账号绑定门市 |
| 据点作业工作台 | 当前账号绑定据点 |

同一账号存在多个数据范围时，系统默认进入最高数据范围对应模板。若权限系统允许该账号切换范围，只能切换到该账号已授权的数据范围。

### 7.3 全局搜索规则

全局搜索只返回当前账号有权查看的数据。

| 搜索对象 | 返回内容 |
| --- | --- |
| 订单 | 订单号、客户、业务模式、状态、取还站点 |
| 车辆 | 车牌、车型、车组、所属车行、当前位置、营运状态 |
| 会员 | 姓名、手机号、会员方案、企业身份 |
| 企业 | 企业名称、月结状态、企业方案 |
| 车行 | 车行名称、状态 |
| 门市 | 门市名称、所属车行、状态 |
| 据点 | 据点名称、所属门市、服务能力、状态 |

无权限对象不返回搜索结果。后端不能返回完整数据后依赖前端过滤。

## 8. 角色模板

新版工作台定义四类模板。四类模板共用同一页面框架，但默认卡片、任务分类、时间线和快捷入口不同。

### 8.1 营运总控工作台

适用对象：

1. 营运商管理员。
2. 客服。
3. 调度。
4. 财务。
5. 风控。
6. 运营人员。

页面目标：

1. 发现全租户范围内的高优任务。
2. 发现跨车行、跨门市的履约风险。
3. 发现资金、账单、资格、事故和站点异常。
4. 快速进入订单、调度、车辆、站点、会员、企业和账单页面。

默认今日重点卡片：

| 卡片 | 统计口径 | 主跳转 |
| --- | --- | --- |
| 高优任务 | 当前账号权限范围内 P0 + P1 任务数 | 工作清单，筛选高优 |
| 今日待交付 | 今日需要人工关注的取车、交车、自助取车异常数量 | 订单列表 |
| 今日待归还 | 今日需要人工关注的还车、收车、自助还车异常数量 | 订单列表 |
| 资金账单待处理 | 待支付、待补缴、待退款、企业月结、公务车账单待处理数量 | 费用/账单页面 |

无资金权限账号不展示“资金账单待处理”，第四张卡片替换为“异常待处理”。

默认任务分类：

| 分类 | 包含内容 |
| --- | --- |
| 订单履约 | 待派车、硬锁失败、待交车、待还车、待结算、取还车中断、转单待处理 |
| 资金账单 | 待支付、预授权异常、待补缴、待退款、企业月结、公务车账单 |
| 风控客服 | 会员审核、资格核验异常、事故待处理、投诉待跟进 |
| 车辆站点 | 车辆被管控、车机异常、站点暂停、车位不足、调度待处理 |
| 公务车 | 合约到期、配额异常、账单逾期、续费待处理 |

默认快捷入口：

| 入口 | 说明 |
| --- | --- |
| 新建订单 | 后台代下单入口 |
| 门市租赁订单 | 门市租车和 24 小时自助订单 |
| 分时租赁订单 | 同站租还和甲租乙还订单 |
| 车辆排程 | 查看车辆占用与调度 |
| 服务站点管理 | 管理车行、门市、据点和服务能力 |
| 车辆档案库 | 查看车辆状态和资产归属 |
| 会员管理 | 会员与企业身份 |
| 企业与账单 | 企业月结、公务车合约和账单 |

### 8.2 车行协同工作台

适用对象：

1. 车行管理员。
2. 车行调度。
3. 车行运营。
4. 车行财务或结算查看人员。

页面目标：

1. 管理当前车行范围内的车辆、门市、据点和订单履约。
2. 发现当前车行内跨门市的车辆调拨、备车和还车归位需求。
3. 关注车辆整备、维保、事故、停放、可用库存和履约压力。

默认今日重点卡片：

| 卡片 | 统计口径 | 主跳转 |
| --- | --- | --- |
| 今日待交车 | 当前车行范围内今日待交车、待取车、自助取车异常数量 | 订单列表 |
| 今日待收车 | 当前车行范围内今日待收车、待还车、自助还车异常数量 | 订单列表 |
| 待整备车辆 | 当前车行范围内清洁、保养、补能、维修、归位车辆数量 | 车辆档案库/调度页 |
| 可用库存 | 当前车行范围内启用、空闲、无时间冲突、可销售车辆数量 | 车辆档案库 |

默认任务分类：

| 分类 | 包含内容 |
| --- | --- |
| 今日履约 | 今日交车、收车、取还车异常、订单逾期 |
| 车行内调度 | 同车行跨门市备车、还车归位、改还车门市、改还车据点 |
| 整备维保 | 清洁、保养、补油/补电、维修、事故处理 |
| 站点车辆 | 门市/据点状态异常、车位不足、车机异常、车辆被管控 |
| 经营关注 | 当前车行订单量、车辆使用、未来车行结算相关摘要 |

车行协同工作台不展示营运商全范围经营指标，不展示其他车行订单和车辆。

### 8.3 门市履约工作台

适用对象：

1. 门市店长。
2. 门市店员。
3. 门市客服。

页面目标：

1. 帮助门市人员完成当天交车、收车、验车、收款、资格核验和结算。
2. 让门市人员看到本门市下各据点车辆所在位置，方便备车和归还登记。
3. 让门市人员优先处理会影响客户到店体验的事项。

默认今日重点卡片：

| 卡片 | 统计口径 | 主跳转 |
| --- | --- | --- |
| 待交车 | 当前门市今日待交车、待取车、取车中断数量 | 订单列表 |
| 待收车 | 当前门市今日待收车、待还车、还车中断数量 | 订单列表 |
| 待备车 | 当前门市下各据点需备车、整备、归位车辆数量 | 车辆速查/调度页 |
| 现场异常 | 支付失败、资格核验失败、车辆异常、站点异常数量 | 工作清单 |

默认任务分类：

| 分类 | 包含内容 |
| --- | --- |
| 交车办理 | 待交车、取车验车、资格核验、取车收款、预授权处理 |
| 收车办理 | 待收车、还车验车、归还据点登记、待结算、补缴、退款、月结挂账 |
| 备车整备 | 备车、洗车、保养、补能、车辆归位 |
| 现场异常 | 证件缺失、资格核验失败、支付失败、取还车窗口中断、事故登记 |

门市履约工作台默认不展示其他门市订单。改还车门市转单完成后，新还车门市才在工作台看到该订单的还车事项。

### 8.4 据点作业工作台

适用对象：

1. 据点管理员。
2. 停车场协作人员。
3. 自助据点现场处理人员。

页面目标：

1. 管理当前据点的车辆停放、车位、车机、自助取还车异常。
2. 支撑 24 小时自助租车和分时租赁的现场问题处理。
3. 支撑门市租车内部备车、归还停放和车辆整备。

默认今日重点卡片：

| 卡片 | 统计口径 | 主跳转 |
| --- | --- | --- |
| 当前停放 | 当前据点停放车辆数 | 车辆列表 |
| 今日进出 | 今日预计取车、还车、调拨进出数量 | 时间线 |
| 车位余量 | 当前总车位与剩余车位 | 服务站点/车位页 |
| 据点异常 | 车机异常、门禁异常、车位不足、服务暂停数量 | 工作清单 |

默认任务分类：

| 分类 | 包含内容 |
| --- | --- |
| 自助异常 | 开锁失败、还车失败、车机值异常、车机离线 |
| 车辆停放 | 待归位、待移车、待整备、待确认停放位置 |
| 车位容量 | 甲租乙还可还车位不足、据点车位紧张 |
| 据点状态 | 据点关闭、暂停服务、服务能力异常 |

## 9. 统一任务模型

工作台中的“任务”不是新的业务单据。任务是从订单、车辆、站点、调度、会员、企业、账单、公务车等模块实时聚合出来的待处理事项。

### 9.1 任务字段

后端返回工作台任务时，至少需要包含以下字段：

| 字段 | 说明 |
| --- | --- |

### 9.3 任务状态口径

工作台任务状态不单独落业务状态。任务是否仍然待处理，以来源对象的当前状态实时判断。

| 工作台展示状态 | 判断方式 |
| --- | --- |
| 待处理 | 来源对象仍满足任务进入条件 |
| 已处理 | 来源对象已不满足任务进入条件 |
| 已超时 | 当前服务器时间超过任务 deadlineAt |
| 即将到期 | 当前服务器时间距离 deadlineAt 小于系统配置阈值 |
| 已读 | 仅用于通知型提醒，不代表业务已处理 |

工作台不能因为用户点击“查看”就把业务任务标记为已处理。只有来源业务状态变化后，任务才从待处理列表移除。

## 10. 优先级与排序

### 10.1 优先级定义

| 优先级 | 含义 | 示例 |
| --- | --- | --- |
| P0 | 已经影响客户履约、资金安全或系统继续流转，必须立即处理 | 客户到店但硬锁失败、取车收款失败、还车无法结算、车机无法开锁 |
| P1 | 今日必须处理，不处理会影响当天履约或客户体验 | 今日待交车、今日待收车、今日待备车、今日账单到期 |
| P2 | 需要跟进，但不阻断当前履约 | 车辆保养即将到期、合约 7 天内到期、低油低电 |
| P3 | 信息提醒或低风险关注 | 公告、统计摘要、未来到期事项 |

### 10.2 默认排序

工作清单默认排序规则：

1. P0 在前。
2. P1 在后。
3. P2 在后。
4. P3 默认不进入主工作清单，只在通知或摘要区域展示。
5. 同优先级内，已超时在前。
6. 同优先级且同 SLA 状态内，按 deadlineAt 升序。
7. deadlineAt 相同，按计划取车/还车/账单截止时间升序。

### 10.3 任务聚合去重

同一来源对象同时触发多个任务时，按以下规则展示：

1. 同一订单同时存在待交车和支付失败，主清单展示支付失败任务，订单详情中展示关联待交车状态。
2. 同一车辆同时存在低油和待清洁，若该车辆已关联今日订单，任务优先展示为待备车异常。
3. 同一据点同时存在服务暂停和车位不足，按服务暂停优先。
4. 同一企业账单同时存在逾期和待确认，按逾期优先。

工作台卡片统计可以累计多个任务数量，但主工作清单需要避免同一对象重复刷屏。

## 11. 今日重点卡片

今日重点卡片固定展示 4 张。卡片只展示行动型数字，不展示纯经营累计数。

### 11.1 卡片字段

| 字段 | 说明 |
| --- | --- |
| 标题 | 今日待交车、今日待收车、待整备车辆等 |
| 数字 | 当前筛选范围内的数量 |
| 变化说明 | 与昨日或上一刷新相比的变化，可为空 |
| 最高优先级 | 当前卡片内最高优任务等级 |
| 主入口 | 点击进入对应列表 |

### 11.2 卡片点击规则

用户点击卡片后，跳转到对应业务列表，并自动带入：

1. 当前工作范围。
2. 当前业务线筛选。
3. 卡片对应状态或任务类型。
4. 今日时间范围。

例如，门市店员点击“待交车”，进入门市租赁订单列表，并默认筛选当前门市、今日取车、待交车/验车中/取车中断等状态。

### 11.3 空状态

卡片数量为 0 时，不隐藏卡片。卡片显示 `0` 和简短空状态，例如“今日暂无待交车”。工作台不因全部为 0 而展示大面积空白。

## 15. 业务线任务来源规则

### 15.1 分时租赁：同站租还

同站租还是用户自助取还车业务。正常预约、正常取车、正常还车不进入门市或据点工作台。

进入工作台的事项：

| 场景 | 任务类型 | 说明 |
| --- | --- | --- |
| 预约后超时未取车 | order_pickup_overdue | 按分时租赁订单有效期规则判断 |
| APP 自助取车失败 | device_command_failed | 开锁、车机、车辆状态异常 |
| 逾期未还 | order_return_overdue | 按实际还车规则判断 |
| APP 自助还车失败 | device_command_failed | 锁车、里程、油量/电量、照片或位置异常 |
| 扣款失败 | order_payment_due | 后付费扣款失败 |
| 车辆异常 | vehicle_controlled / device_offline | 影响后续预约或取还车 |

### 15.2 分时租赁：甲租乙还

甲租乙还需要额外关注可还车据点和车位。

进入工作台的事项：

| 场景 | 任务类型 | 说明 |
| --- | --- | --- |
| 可还车位不足 | station_capacity_risk | 空闲车位为 0 或低于预警阈值 |
| 还车据点关闭或暂停 | store_or_station_closed / station_service_paused | 已有订单按客服人工处理口径提示 |
| 预约后超时未取车 | order_pickup_overdue | 甲租乙还是实时订单，需要快速处理 |
| 逾期未还 | order_return_overdue | 影响车辆和后续调度 |
| 自助取还车失败 | device_command_failed | 车机、门禁、车辆状态异常 |
| 还车后待调度 | vehicle_prepare_due | 车辆需要归位、整备或调度 |

### 15.3 门市租赁：门市租车

门市租车是有人服务业务，工作台需要承接门市日常作业。

进入工作台的事项：

| 场景 | 任务类型 | 说明 |
| --- | --- | --- |
| 待支付 / 预授权待完成 | order_payment_due | 个人自付和企业非月结订单 |
| 待派车 | order_dispatch_due | 订单已创建但未预分配车辆 |
| 硬锁失败 | order_hard_lock_failed | 到硬锁时间点系统未锁定可履约车辆 |
| 今日待交车 | order_pickup_due | 门市到店交车 |
| 取车流程中断 | order_pickup_interrupted | 店员打开取车窗口后未完成提交 |
| 行销资格待核验 | order_qualification_check_due | 开启到店人工核验的行销方案 |
| 取车收款异常 | order_payment_due | 尾款、预授权转租金或履约担保相关 |
| 今日待收车 | order_return_due | 门市收车和验车 |
| 还车流程中断 | order_return_interrupted | 还车窗口未提交 |
| 待结算 | order_settlement_due | 验车完成后等待费用结算 |
| 待补缴 | order_payment_due | 还车后应补缴 |
| 待退款 | order_refund_due | 还车后应退款 |
| 企业月结挂账 | order_monthly_bill_due | 企业月结订单还车后进入企业账单 |
| 改还车门市 | order_return_store_change_due | 同车行范围内转单 |
| 事故登记 | order_accident_due | 事故和改派本期不做系统联动，但事故待处理需进入任务 |

### 15.4 门市租赁：24 小时自助租车

24 小时自助租车与门市租车共用计费和规则，但取还车由用户在 APP 自助完成。

进入工作台的事项：

| 场景 | 任务类型 | 说明 |
| --- | --- | --- |
| 待派车 / 硬锁失败 | order_dispatch_due / order_hard_lock_failed | 按门市租赁派车规则处理 |
| 自助取车失败 | device_command_failed | 车机、门禁、开锁、车辆状态异常 |
| 车机里程/油量/电量未获取 | device_offline | 页面提示人工处理或核对 |
| 自助还车失败 | device_command_failed | 锁车、照片、位置、车机值异常 |
| 改还车据点 | order_return_station_change_due | 24 小时自助可指定具体还车据点 |
| 据点关闭或暂停 | store_or_station_closed / station_service_paused | 后台和 APP 提示联系客服人工处理 |
| 待补缴 / 待退款 | order_payment_due / order_refund_due | 还车结算后的资金任务 |

开启到店人工核验的行销方案本期不适用于 24 小时自助租车，因此 24 小时自助订单不产生到店资格核验任务。

### 15.5 公务车

公务车不涉及员工个人支付，工作台关注合约、配额、账单和上架状态。

进入工作台的事项：

| 场景 | 任务类型 | 说明 |
| --- | --- | --- |
| 合约即将到期 | official_contract_due | 按合约到期提醒规则 |
| 合约已到期 | official_contract_due | 到期后强制下架合约内车辆 |
| 配额不足或异常 | official_quota_due | 企业车辆绑定和上架受影响 |
| 公务车账单待客户确认 | official_bill_due | 账单流程任务 |
| 待提供汇款账户 | official_bill_due | 公务车账单状态 |
| 待汇款 / 汇款确认中 | official_bill_due | 公务车账单状态 |
| 账单逾期 | official_bill_due | 逾期提醒 |

### 15.6 企业月结

企业月结订单不走个人支付、预授权、退款链路。

进入工作台的事项：

| 场景 | 任务类型 | 说明 |
| --- | --- | --- |
| 月结订单还车完成 | order_monthly_bill_due | 订单最终应收进入企业账单 |
| 企业账单待处理 | enterprise_bill_due | 企业 + 自然月账单 |
| 企业账单逾期 | enterprise_bill_due | 只提醒，不限制员工继续下单 |
| 企业账单付款记录待确认 | enterprise_bill_due | 线下公对公转账记录确认 |

## 17. 公告、通知和最近访问

### 17.1 公告

公告区域展示最新 3 条。

字段包括：

1. 标题。
2. 发布时间。
3. 发布范围。
4. 摘要。

公告只展示当前账号可见范围内的公告。

### 17.2 通知中心

通知中心承接系统通知和业务提醒。

工作清单与通知中心边界：

| 类型 | 展示位置 |
| --- | --- |
| 需要处理且有明确处理入口 | 工作清单 |
| 只需要告知或查看 | 通知中心 |
| 系统公告 | 公告区和通知中心 |
| 已过期但仍需追溯的通知 | 通知中心历史 |

### 17.3 最近访问

最近访问展示最近 5 个页面。

规则：

1. 只记录当前账号有权限访问的页面。
2. 同一页面重复访问只保留最近一次。
3. 访问失效页面时不展示。
4. 最近访问不记录具体客户隐私字段。

## 18. 权限与数据范围

### 18.1 数据过滤顺序

工作台数据按以下顺序过滤：

1. 租户。
2. 业务线授权。
3. 服务模式权限。
4. 账号数据范围。
5. 菜单权限。
6. 操作权限。
7. 字段权限或隐私规则。

后端必须在接口层完成权限和范围过滤。

### 18.2 模块显示规则

| 情况 | 处理 |
| --- | --- |
| 有权限但无数据 | 显示空状态 |
| 无页面权限 | 不展示模块和入口 |
| 有查看权限无操作权限 | 展示查看入口，不展示处理按钮 |
| 无资金权限 | 不展示资金卡片、资金任务和金额字段 |
| 无调度权限 | 不展示改派、调度、硬锁失败处理入口 |
| 无会员审核权限 | 不展示会员审核任务 |
| 无公务车权限 | 不展示公务车模块 |

### 18.3 金额和敏感字段

金额、证件、手机号、发票、支付记录、退款记录等字段按当前平台权限体系处理。后续如新增字段级脱敏或隐私权限，工作台必须同步遵守。

## 19. 数据接口责任

工作台后端需要提供聚合接口。接口名称可由开发按项目规范确定，但职责必须覆盖以下能力。

### 19.1 上下文接口

返回当前账号的工作台上下文：

| 内容 | 说明 |
| --- | --- |
| templateType | workbench_operation、workbench_car_company、workbench_store、workbench_station |
| tenantInfo | 当前租户 |
| scopeInfo | 当前车行、门市或据点 |
| enabledBusinessLines | 当前授权业务线 |
| permissions | 当前工作台可用权限 |
| filterOptions | 当前可选业务线、车行、门市、据点 |

### 19.2 摘要接口

返回今日重点卡片和资源摘要。

摘要接口必须按当前筛选条件和权限返回结果，不允许前端自行聚合无权限明细。

### 19.3 任务接口

返回主工作清单。

支持参数：

| 参数 | 说明 |
| --- | --- |
| templateType | 当前模板 |
| scopeType | 营运、车行、门市、据点 |
| scopeId | 当前范围 ID |
| businessCategory | 业务大类 |
| serviceMode | 业务模式 |
| priority | 优先级 |
| taskType | 任务类型 |
| dateRange | 今日、未来、全部 |
| onlyMine | 只看当前账号可处理 |

### 19.4 时间线接口

返回今日履约时间线。

时间线不返回无权限客户资料。门市和据点账号只返回当前范围内的事项。

### 19.5 快捷入口接口

返回当前账号可见快捷入口及用户排序配置。

### 19.6 通知公告接口

返回公告、通知和未读数。

通知接口与任务接口分开，避免通知型消息污染工作清单。

## 20. 刷新与时效

### 20.1 时间基准

所有超时、即将到期、今日、未来判断均使用服务器时间。

前端展示本地时间时，需要以服务器返回时间为准进行格式化。

### 20.2 刷新策略

| 模块 | 刷新方式 |
| --- | --- |
| 顶部上下文 | 登录后加载，切换范围后刷新 |
| 今日重点卡片 | 进入页面加载，支持手动刷新 |
| 工作清单 | 进入页面加载，支持 60 秒轮询或手动刷新 |
| 今日履约时间线 | 进入页面加载，支持手动刷新 |
| 资源健康 | 进入页面加载，支持 60 秒轮询或手动刷新 |
| 公告通知 | 进入页面加载，支持通知中心更新 |
| 最近访问 | 页面访问时写入 |

### 20.3 任务消失规则

任务来源对象状态变化后，下一次刷新时从工作台移除。

例如：

1. 待交车订单完成取车后，从待交车任务移除。
2. 待退款订单退款成功后，从待退款任务移除。
3. 硬锁失败订单完成人工改派后，从硬锁失败任务移除。
4. 据点暂停恢复后，从站点服务暂停任务移除。

## 21. 页面状态

### 21.1 加载状态

页面首次加载时，今日重点卡片、工作清单、时间线和资源摘要使用骨架屏。

### 21.2 空状态

空状态必须说明当前范围暂无事项，例如：

1. 今日暂无待交车。
2. 当前车行暂无待整备车辆。
3. 当前据点暂无自助异常。

空状态不使用营销文案，不出现夸张插图。

### 21.3 部分失败

某个模块加载失败时，只在该模块区域展示失败提示和重试按钮，不阻断整个工作台。

### 21.4 无权限状态

账号无某类权限时，不展示对应模块，不展示“无权限”的大面积提示。只有用户直接访问无权限页面时，才展示无权限页。

## 22. 操作记录

工作台本身不生成业务操作记录。用户从工作台跳转到来源业务页面并完成处理后，由来源业务模块记录操作记录。

以下工作台行为需要记录用户行为日志：

1. 点击任务处理入口。
2. 点击卡片跳转。
3. 修改快捷入口排序或显示状态。
4. 切换工作范围。
5. 全局搜索并打开对象。

用户行为日志用于产品分析，不等同于业务操作记录。

## 25. 原型要求

原型必须体现：

1. 四类角色模板的差异，至少提供营运总控和车行协同两种完整示例。

3. 今日重点卡片固定 4 张。
4. 主工作清单为页面核心，不被图表抢占。
5. 卡片和任务点击后的目标页面和筛选口径需要在原型中表达。
6. 无权限、空状态、异常加载状态需要有示例。
7. 页面整体风格简洁、克制、可读，不使用花哨装饰。
8. 不使用复杂渐变、过度阴影、大面积炫彩统计图。
9. 文字优先表达业务动作，避免使用空泛概念词。

## 26. 验收标准

### 26.1 业务验收

1. 营运角色登录后，只看到当前租户范围内且有权限的数据。
2. 车行角色登录后，只看到绑定车行范围内的数据。
3. 门市角色登录后，只看到绑定门市范围内的数据。
4. 据点角色登录后，只看到绑定据点范围内的数据。
5. 无资金权限账号不展示资金和账单任务。
6. 无调度权限账号不展示调度处理入口。
7. 今日待交车、待收车、待备车、待结算、待支付、待退款、硬锁失败等任务能正确进入工作清单。
8. 正常完成且无需人工处理的自助取还车事项不进入工作清单。
9. 24 小时自助租车的自助异常、车机异常和改还车据点任务能进入工作清单。
10. 公务车合约、配额和账单任务能按权限展示。

### 26.2 交互验收

1. 用户点击任一卡片后，能进入对应业务页面并自动带入筛选条件。
2. 用户点击任一任务主操作后，能进入对应处理页面。
3. 来源业务状态处理完成后，工作台下一次刷新不再展示该任务。
4. 工作清单排序符合 P0、P1、P2 和超时优先规则。
5. 页面无权限模块不展示，有权限但无数据模块展示空状态。
6. 某个模块加载失败时，不影响其他模块展示。

### 26.3 开发验收

1. 后端接口完成权限和数据范围过滤，前端不接收无权限明细后再隐藏。
2. 工作台不定义新的订单、车辆、站点或账单业务状态。
3. 工作台任务来源均能追溯到来源模块和来源对象。
4. 每类任务均有明确 `taskType`、优先级、跳转页面和权限点。
5. 所有超时判断使用服务器时间。
6. 任务数据、摘要数据、时间线数据和通知数据分接口返回，避免互相污染。
7. 重要任务类型新增或调整时，需要同步更新本文和业务决策记录。
